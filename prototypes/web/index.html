灌溉控制器 Web 原型

按实际使用频率组织页面：总览看状态，手动紧跟首页，近期计划和计划配置分开。

原型说明

对应导航总览 / 手动 / 近期计划 / 计划配置 / 记录 / 设置 / 调试
首页职责只回答设备是否正常、是否正在浇水、能否立即停止、有没有异常
交互边界所有改变状态的操作均需确认
实现方式静态 HTML/CSS，不接入固件

设计原则

不做复杂 SPA，不堆装饰。顶部导航负责去向，总览页不再重复放快捷入口。

页面清单

页面功能原型
总览设备状态、浇水状态、停止操作和异常提示打开
手动查看浇水状态、按路选择时长并开始手动浇水打开
近期计划昨日、今日、明日、后天的计划执行状态，可跳过某次或某天计划打开
计划配置计划列表和修改入口打开
编辑计划执行时间、循环规则、执行模式和每路时长打开
记录浇水记录和系统事件追踪打开
设置灌溉业务参数，每次只修改一个参数打开
调试展示当前 JSON 状态结构打开
