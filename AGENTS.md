# ESP32 灌溉项目协作规则

## 沟通规则

- 默认使用中文沟通。
- 先依据当前硬件和当前方案判断，不从历史方案继承结论。
- 信息不足时先核实；无法核实时明确说明假设和风险。
- 用户讨论想法时先判断目标、约束和成功标准，不直接执行。
- 用户明确要求修改、实现或整理时，默认直接执行。

## 当前版本依据

当前只有一个有效版本。

- 旧工程归档在 `上一版/V6/`，只能作为需求背景参考。
- 当前硬件事实以 `pcb/` 下 2026-07-06 的 BOM 和网表为准。
- 当前产品与软件方案以 `docs/当前方案/` 为准。
- ESP32 固件必须基于同级目录 `../Esp32Base` 基础库搭建，优先使用基础库已经提供的启动、WiFi、Web、认证、OTA、文件系统、日志、事件、配置、健康、看门狗、时间和 RTC 能力。

## 当前产品边界

当前系统是单板 12V DC 智能浇水控制器：

- 12V DC 电源。
- 6 路 12V DC 电磁阀输出。
- 1 路必配流量计输入，固定位置。
- 1 路可选低液位/缺水干接点输入，固定位置，可配置常开/常闭。
- 1 路可选自吸泵控制干接点输出，固定位置，只控制外部继电器模块信号。
- RTC 本地定时。
- WiFi / Web 本地管理。

当前不做：

- RS485。
- 主控/分站/从站。
- 多水源并行。
- 主阀。
- 输入/输出任意映射。
- 云端、多用户、天气联动、通用 IO 平台或复杂权限系统。

## 工作方式

- 修改前先理解影响范围。
- 只能修改本项目工作区 `/Users/tyg/dir/claude_dir/Esp32_Irrigation` 内的文件。
- 本项目外的任何文件都不允许修改，包括但不限于同级目录、基础库、工具库、全局配置和系统配置。
- 尤其禁止修改 `../Esp32Base` 基础库。即使发现基础库存在问题，也不能直接改基础库文件。
- 如果判断基础库需要修改，只能向用户提供清晰的问题说明、建议修改点和可直接交给基础库维护者使用的提示词，等待用户明确授权或另行处理。
- 不删除、覆盖或回滚用户已有改动，除非用户明确要求。
- 代码或文档任务完成后，提交并推送本次改动；用户明确要求不提交或不推送时除外。
- 不提交密钥、token、账号、隐私数据和敏感配置。

## 基础库优先规则

- 项目业务代码只实现智能浇水系统的领域逻辑，不重复实现 `Esp32Base` 已经提供的通用基础能力。
- 如果发现 `Esp32Base` 缺少可复用的通用能力，或基础库能力存在缺陷，只能记录问题并给用户提供基础库修改建议或提示词，不能直接修改 `../Esp32Base`。
- 不在本项目里为基础库问题写临时补丁、旁路实现或重复框架，除非用户明确要求在本项目内做一次性验证。
- 使用 `Esp32Base` 的能力不能改变当前产品边界：本项目仍然不启用 RS485、主从站、分站或历史兼容架构。
