# ESP32 Irrigation

当前项目处于灌溉系统 Web 原型重建阶段。

目标不是修补旧原型，也不是延续某个历史版本，而是在当前系统方案基础上重新做一版最终给用户看的原型。方案已经基本确定，但仍允许为了更清晰、更好用、更可落地而优化。

## 当前入口

```text
AGENTS.md
docs/README.md
docs/当前方案/
pcb/
```

## 当前依据

- 硬件事实只以 `pcb/` 下当前 BOM 和网表为准。
- 当前方案只以 `docs/当前方案/` 为准。
- `docs/历史参考/` 只作为历史参考，不作为当前结论。
- 旧方案、旧原型、旧固件和历史过程文件不作为当前原型依据。

## 当前阶段

当前只做产品方案核对和 Web 原型设计：

1. 核对硬件边界、产品边界、对象模型、状态模型、配置模型、任务、历史和信息架构。
2. 用 Web 原型验证页面数量、页面职责、信息层级、任务流和视觉表达。
3. 原型确认后再进入后续实现设计。

当前不做：

- 固件实现。
- API、存储、通信协议。
- 工程目录、类名或具体实现方案。
- 云端、多用户、天气联动、复杂统计或通用调试后台。

## 原型要求

原型必须重新设计，不能参考失败原型。

页面应像真实产品界面，能让用户清楚理解当前状态、能否浇水、为什么不能浇水以及下一步该做什么。无真实数据时使用空状态、待配置状态、默认值和不可操作原因，不编造设备、计划、历史或运行数据。
