# 文档05：软件架构设计说明书
## 1. 软件分层架构（四层结构，低耦合高内聚）
```
应用层：业务逻辑实现
├─ 浇水控制模块
├─ 浇水模式调度模块（并行/串行模式控制）
├─ 定时任务模块
├─ 流量计量模块
├─ 漏水检测模块
├─ 交互逻辑模块
├─ 网络服务模块
├─ API鉴权模块
└─ OTA升级模块
服务层：通用公共服务
├─ RTC时间服务
├─ 日志记录服务
├─ 参数存储服务（含磨损均衡）
├─ 通知推送服务
└─ 校准计算服务
驱动层：硬件驱动
├─ LCD1602驱动
├─ 编码器驱动
├─ 电磁阀驱动
├─ 流量传感器驱动
└─ RTC驱动
硬件抽象层：屏蔽硬件差异
└─ ESP32硬件接口封装
```
## 2. 核心模块划分
| 模块名称 | 职责 | 接口 |
|----------|------|------|
| 浇水控制 | 电磁阀开关、异常检测 | `water_on(road), water_off(road), stop_all()` |
| 定时任务 | 任务调度、循环处理 | `add_timer(), del_timer(), run_timer()` |
| 流量计量 | 脉冲计数、水量计算 | `get_volume(road), reset_count(road)` |
| Web服务 | 网页服务、API处理 | `web_init(), web_handle()` |
| 物联网平台 | 点灯科技对接、推送 | `iot_init(), iot_push(msg)` |
| 存储服务 | 参数、记录读写 | `save_config(), load_config(), save_record()` |

## 3. 核心数据结构（紧凑二进制，最小化存储体积）
### 3.1 浇水记录结构（每条固定12字节）
```c
typedef struct {
    uint32_t timestamp; // 4字节 Unix时间戳（开始时间）
    uint8_t  road_info; // 1字节 低4位对应路1-4，bit5=0=定时/1=手动
    uint16_t duration;  // 2字节 浇水时长，单位秒
    float    volume;    // 4字节 浇水量，单位L
    uint8_t  status;    // 1字节 0=正常完成/1=手动停止/2=异常停止
} WaterRecord;
```
### 3.2 系统日志结构（每条固定8字节）
```c
typedef struct {
    uint32_t timestamp; // 4字节 时间戳
    uint8_t  type;      // 1字节 0=操作日志/1=系统日志/2=告警日志
    uint8_t  event;     // 1字节 事件枚举（映射表转义）
    uint16_t param;     // 2字节 附加参数（路数/错误码等）
} LogRecord;
```
### 3.3 定时任务结构
```c
typedef struct {
    uint8_t id;
    uint8_t hour;
    uint8_t minute;
    uint8_t road;
    uint16_t duration; // 秒
    uint8_t mode; // 0=每天 1=每周 2=间隔
    uint8_t week_mask; // 每周模式bit0=周日 bit6=周六
    uint16_t interval_day; // 间隔模式天数
    uint8_t enable;
} timer_t;
```
### 3.4 系统配置结构（新增浇水模式配置）
```c
typedef struct {
    char wifi_ssid[32];
    char wifi_pass[64];
    char admin_pass[16];
    char api_key[16]; // API鉴权密钥
    float calib_coef[4];
    uint8_t road_enable[4];
    uint8_t water_mode; // 0=并行/1=串行
    uint8_t flow_detect_time; // 水流检测时长，默认30秒
    uint8_t dnd_enable;
    uint8_t dnd_start_hour;
    uint8_t dnd_end_hour;
    uint16_t active_time; // 活跃时长秒
    uint8_t lock_flag;
} config_t;
```
