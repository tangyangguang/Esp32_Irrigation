# 02 配置模型与参数边界

## 配置分层

| 配置类别 | 保存位置 | 用途 |
|---|---|---|
| 主控系统配置 | 主控 | 设备显示信息、LCD、SHT30、RS485、记录策略 |
| 主控通知配置 | 主控 | 手机通知与轻操作的启用状态、通道参数和通知类别 |
| 主控运行状态 | 主控 | 自动浇水暂停状态、暂停范围、恢复时间和最近通知状态 |
| 从站业务配置 | 主控 | 从站名称、水源名称、水路名称、启用状态 |
| 从站安全配置 | 主控 + 从站 | 输入/输出映射、本地安全参数、阀门参数 |
| 单次运行参数 | 主控下发，从站临时使用 | 运行时长、标准流速、流量保护参数 |

主控保存完整业务配置。从站只保存足够保证本地安全执行和安全停机的配置。

## 主控系统配置

主控系统配置包含：

- 设备名称。
- LCD2004 显示策略，第一版由固件默认策略固定，不在 Web 中暴露首页内容选择项。
- SHT30 温湿度显示策略，第一版默认显示可用温湿度，不在 Web 中暴露隐藏/显示选择项。
- RS485 波特率，默认 `19200 bps`。
- 从站轮询周期。
- 事件和运行记录保留策略，第一版采用固定策略，不在设置页暴露保留天数配置。

时间来源、时区、网络校时和基础 Web/API 访问能力属于 `Esp32Base` 提供的基础能力，不在灌溉业务系统设置页中作为可选配置。灌溉业务只读取系统时间有效状态；系统时间无效时禁止自动计划，但不影响手动查看、配置和记录查看。

主控板不启用 FRAM。配置、计划、校准参数和记录由主控侧负责生命周期管理；具体存储介质属于实施阶段设计。

## 从站业务配置

```text
station_id
rs485_address
enabled
station_name
water_source_name
notes
```

规则：

- `rs485_address` 来自拨码开关，范围 `1..15`。
- 地址 `0` 保留。
- 主控需要检测疑似地址冲突。
- 未启用从站不参与手动运行和计划运行。
- `water_source_name` 只作为从站对应真实水源的显示名称，不形成独立调度对象。

## 水路配置

每个从站最多 8 路阀输出，对应 `VALVE1..VALVE8`。

每一路阀输出角色：

- `disabled`
- `branch_zone`
- `master_valve`

水路配置：

```text
zone_index             1..8
role
enabled
name
lock_state
baseline_ml_min
baseline_status
```

规则：

- 一个从站最多 1 路主阀。
- 主阀不能同时作为分支水路。
- 主阀不作为用户可直接运行水路。
- 至少 1 路分支水路启用后，从站才具备可运行水路。
- 标准流速属于分支水路，不属于主阀。
- 第一版不单独配置手动运行权限或计划运行权限；启用的分支水路默认可被手动运行和计划选择，实际能否启动仍由运行准入、锁定、阻断、配置同步和在线状态决定。
- 第一版不保存分支水路运行默认时长；手动运行时长在运行入口临时填写，计划运行时长在计划表单中填写。

## 输入配置

从站 `IN1..IN4` 可配置为：

- `disabled`
- `flow_meter`
- `low_level`
- `reserved`

输入配置：

```text
input_index             1..4
role
active_level            low / high
debounce_ms
```

规则：

- 一个从站必须且只能配置 1 路流量计。
- 低液位可选，最多配置 1 路。
- 同一路输入不能同时承担多个角色。
- `reserved` 只表示 `IN1..IN4` 中该路暂不参与当前灌溉逻辑。
- `reserved` 不生成保护、不参与运行准入、不要求传感器参数。
- 从站 `H6/H8/H9`、`SPARE_*` 不进入当前 Web 输入配置。

## 流量计配置

```text
input_index
edge                    rising / falling
pulses_per_liter
sample_window_sec
display_refresh_ms
first_flow_timeout_sec
no_flow_timeout_sec
stabilize_sec
standby_leak_confirm_sec
low_flow_action          disabled / alarm_only / stop
high_flow_action         disabled / stop
standby_leak_enabled
standby_leak_action      alarm_block_station
```

规则：

- `pulses_per_liter` 来自流量计校准。
- 计量未校准时仍允许灌溉运行，但水量估算不可用或仅供参考。
- 只要能检测脉冲，仍可做无水判断。
- `stabilize_sec` 只屏蔽低/高流量判断，不屏蔽无水和低液位保护。
- `low_flow_action=alarm_only` 时，从站不执行低流量停机，主控根据轮询状态生成告警。
- `low_flow_action=stop` 时，主控在运行命令中启用低流量停机，由从站本地执行。
- `high_flow_action=stop` 是高流量保护的唯一停机策略；`disabled` 表示不启用高流量保护。
- `standby_leak_enabled=0` 时，待机漏水增强判断不可用但不阻断正常启动。
- `standby_leak_action` 第一版固定为 `alarm_block_station`，表示确认待机漏水后生成告警并默认禁止该从站新运行。

## 低液位配置

```text
enabled
input_index             0=不启用，1..4=输入
active_level
confirm_ms
action                  stop_current_source
```

规则：

- 未配置低液位时，不做低液位保护。
- 配置低液位后，运行中触发并确认必须停机。
- 低液位是水源级条件，不是单一路水路特征。

## 自吸泵配置

```text
enabled
output                  dry_contact_1
start_delay_ms
stop_delay_ms
```

规则：

- 未启用时，从站不控制干接点输出。
- 启用时，运行任意分支水路都会联动自吸泵。
- 同时启用低液位时，启动自吸泵前必须先确认低液位未触发。

## 主阀配置

```text
enabled
valve_index             0=无主阀，1..8=主阀
start_delay_ms
stop_delay_ms
```

规则：

- 主阀可选。
- 未配置主阀时，分支阀可直接运行。
- 配置主阀时，启动顺序为自吸泵、主阀、分支阀；停止顺序反向或按安全停机策略关闭。

## 阀门参数

```text
valve_pull_in_ms
valve_pwm_freq_hz
valve_hold_percent
```

规则：

- 默认策略必须能在未验证 PWM 降功率时使用 100% 保持。
- PWM 降功率是增强能力，未验证不阻断运行。
- 安全停机时必须关闭 `DRV_SD` 和所有阀输出。

## 电压监测配置

```text
voltage_monitor_enabled
voltage_low_warn_mv
voltage_high_warn_mv
```

规则：

- 电压监测只用于显示、告警和记录，不作为自动停机条件。
- 主控和从站的电压检测都不承担入口电源控制职责。
- 电压检测未校准或监测未启用时，不阻断运行。
- 未启用时只提示电压监测未启用。
- 已启用、已校准并确认低压或高压时，只生成告警和事件记录。
- 从站不得因为电压阈值主动关闭阀、关闭自吸泵或拒绝启动。
- `voltage_calibrated` 是主控侧校准状态，不作为从站安全配置字段单独下发。
- 主控只有在电压检测已校准后，才允许把 `voltage_monitor_enabled` 下发为启用。

## 配置同步规则

- 主控维护配置版本号和 CRC。
- 从站保存安全配置版本和 CRC。
- 主控轮询时检查从站配置摘要。
- 版本或 CRC 不一致时，相关从站不能运行，必须重新同步配置。
- 从站处于 `STARTING`、`RUNNING`、`STOPPING` 时拒绝写入安全配置。

同步流程：

1. Web 保存主控侧配置。
2. 主控判断是否影响从站安全配置；不影响安全配置的字段只更新主控侧显示和业务配置。
3. 如果影响安全配置，主控生成新的安全配置版本和 CRC，并把对应从站标记为待同步。
4. 用户从水源管理或维护页发起同步；运行中、离线、地址冲突、配置无效或输入输出冲突时不得同步。
5. 主控下发 `SET_SAFE_CONFIG`，再读取 `GET_CONFIG_SUMMARY` 验证版本和 CRC。
6. 同步成功后解除待同步阻断；同步失败保留待同步状态，并记录失败原因。
7. 同步不会自动启动运行，不修改计划启用状态，不删除历史记录。

## 参数默认值与校验范围

参数边界必须同时在 Web 保存、主控业务层和下发从站前校验。下表未列出的业务字段不得自由扩展成通用 IO 或通用寄存器能力。

通信参数：

| 参数 | 默认值 | 范围 | 单位 | 说明 |
|---|---:|---:|---|---|
| `rs485_baud` | 19200 | 9600 / 19200 / 38400 | bps | 量产默认 19200 |
| `request_timeout_ms` | 150 | 100..500 | ms | 普通请求超时 |
| `max_retries_idle` | 2 | 0..3 | 次 | 空闲从站查询 |
| `max_retries_running` | 1 | 0..2 | 次 | 运行中从站查询 |
| `offline_fail_count` | 3 | 2..5 | 次 | 主控判定离线 |
| `conflict_suspect_count` | 3 | 2..5 | 次 | 主控判定疑似地址冲突 |

运行和安全参数：

| 参数 | 默认值 | 范围 | 单位 | 下发方式 |
|---|---:|---:|---|---|
| `comm_timeout_ms` | 5000 | 3000..30000 | ms | 安全配置 |
| `max_run_sec` | 7200 | 60..21600 | sec | 安全配置 |
| `first_flow_timeout_sec` | 30 | 5..300 | sec | 运行命令 |
| `no_flow_timeout_sec` | 20 | 5..300 | sec | 运行命令 |
| `stabilize_sec` | 10 | 0..120 | sec | 运行命令 |
| `abnormal_confirm_sec` | 10 | 3..120 | sec | 运行命令 |
| `low_flow_percent` | 50 | 10..95 | % | 运行命令 |
| `high_flow_percent` | 150 | 105..300 | % | 运行命令 |

输入、低液位和流量计参数：

| 参数/场景 | 默认值 | 范围 | 单位 | 说明 |
|---|---:|---:|---|---|
| `input_debounce_ms`，普通数字输入 | 20 | 0..200 | ms | 用于低液位或普通开关量 |
| `input_debounce_ms`，流量计输入 | 0 | 0..10 | ms | 避免吞脉冲 |
| `low_level_confirm_ms` | 1000 | 100..10000 | ms | 低液位确认 |
| `sample_window_sec` | 5 | 1..60 | sec | 流量显示和判断窗口 |
| `display_refresh_ms` | 1000 | 200..5000 | ms | Web 状态刷新建议值 |
| `standby_leak_confirm_sec` | 30 | 5..600 | sec | 待机漏水确认 |
| `pulses_per_liter` | 0 | 0..100000 | pulse/L | 0 表示计量未校准 |
| `baseline_ml_min` | 0 | 0..1000000 | ml/min | 0 表示标准流速未学习 |
| `baseline_sample_sec` | 60 | 10..600 | sec | 标准流速学习采样 |

天气显示参数：

| 参数 | 默认值 | 范围 | 单位 | 说明 |
|---|---:|---:|---|---|
| `weather_enabled` | 0 | 0..1 | bool | 只控制 Web 天气显示和建议 |
| `weather_location_name` | 空 | 0..32 字符 | text | 页面显示的位置名称 |
| `weather_refresh_min` | 60 | 15..1440 | min | 天气数据刷新建议值 |
| `weather_stale_min` | 180 | 30..2880 | min | 超过后显示天气数据过期 |

天气显示参数只保存在主控侧，不下发从站，不参与启动准入、停机、计划跳过、运行时长调整或安全配置。

通知配置参数：

| 参数 | 默认值 | 范围 | 单位 | 说明 |
|---|---:|---:|---|---|
| `notification_enabled` | 0 | 0..1 | bool | 是否启用手机通知与轻操作 |
| `notify_run_completed` | 1 | 0..1 | bool | 灌溉完成通知 |
| `notify_fault` | 1 | 0..1 | bool | 保护停机、阻断、通信异常等通知 |
| `notify_pause_resume` | 1 | 0..1 | bool | 暂停、恢复和到期恢复通知 |

通知通道的具体类型、凭据、目标地址和安全策略后续单独裁决；不得在文档、日志或提交中泄露 token、账号、密钥或隐私数据。

自动浇水暂停状态：

```text
pause_scope              none / all / source
source_id                scope=source 时有效
paused_until
paused_by
pause_reason
```

规则：

- 自动浇水暂停状态只保存在主控侧，不下发从站。
- 暂停只影响自动计划触发，不影响手动运行。
- 暂停到期自动恢复，可提前恢复。
- 暂停不修改计划配置、不禁用计划、不删除计划。
- 第一版不支持按水路暂停、按单个计划暂停、补跑或天气自动暂停。

阀门、主阀和自吸泵参数：

| 参数 | 默认值 | 范围 | 单位 | 说明 |
|---|---:|---:|---|---|
| `valve_pull_in_ms` | 500 | 100..3000 | ms | 实测前不低于 500 |
| `valve_pwm_freq_hz` | 1000 | 100..5000 | Hz | 仅用于 PWM 保持 |
| `valve_hold_percent` | 100 | 20..100 | % | 未验证降功率前使用 100 |
| `master_start_delay_ms` | 1000 | 0..10000 | ms | 无主阀时忽略 |
| `master_stop_delay_ms` | 1000 | 0..10000 | ms | 无主阀时忽略 |
| `pump_start_delay_ms` | 1000 | 0..10000 | ms | 无自吸泵时忽略 |
| `pump_stop_delay_ms` | 1000 | 0..10000 | ms | 无自吸泵时忽略 |

电压参数：

| 参数 | 默认值 | 范围 | 单位 | 说明 |
|---|---:|---:|---|---|
| `voltage_low_warn_mv` | 10800 | 9000..13000 | mV | 12V 系统低压预警 |
| `voltage_high_warn_mv` | 15000 | 13000..18000 | mV | 12V 系统高压预警 |

参数关系：

- `low_flow_percent` 必须小于 100，`high_flow_percent` 必须大于 100。
- `voltage_high_warn_mv` 必须大于 `voltage_low_warn_mv`。
- `comm_timeout_ms` 必须小于 `max_run_sec * 1000`。
- 主控运行时长不得大于从站 `max_run_sec`。
- 标准流速未学习时，低/高流量保护不启用，但普通灌溉仍允许。
- 计量未校准时，水量估算不可用或仅供参考，但无水保护仍启用。
- 电压检测未校准时，电压监测告警不启用，但普通灌溉仍允许。
- `low_flow_action=disabled` 或 `high_flow_action=disabled` 时，对应保护不生成阻断性故障。
- `standby_leak_enabled=0` 时，不生成待机漏水阻断性告警。

## 增强能力降级原则

允许启动但必须提示保护不完整：

- 流量计计量校准未完成。
- 标准流速未学习。
- 电压检测未校准或电压监测未启用。
- SHT30、LCD、待机漏水等增强诊断或显示能力不可用。
- 主阀未配置。
- 低液位未配置。
- 自吸泵未配置。

不得把“保护未启用”显示为“设备故障”。
