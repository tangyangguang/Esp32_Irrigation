# 06 Web 信息架构与页面职责

## 基本原则

Web 原型先表达完整功能逻辑，不追求炫酷可视化。

页面按对象和任务组织，不按高级、维护、日常等主观层级组织。

页面内不设计跨页面快捷入口。系统已经有完整顶层导航，跨页面移动依赖顶层导航完成。只有当前任务闭环必须的动作，才允许在页面内部出现。

页面内容避免重复。某个信息如果已经由专门页面承载，其它页面只保留摘要，不重复完整模块。

列表页不在列表旁边或下方直接铺新建、编辑、配置表单。列表页只承载筛选、分页、列表、单条详情入口和必要的行级动作。简单表单通过弹层出现；复杂表单或需要完整上下文的配置进入独立页面，保存或取消后返回列表或来源页面。详情也进入独立详情页，不和列表同屏展开。

记录和事件默认考虑分页。

顶层导航控制在 7 个：

- 总览
- 运行
- 计划
- 水源
- 记录
- 事件
- 系统参数

## 总览

总览回答：现在系统是否正常，是否正在浇水，是否有需要处理的问题。

总览显示：

- 主控状态。
- 水源在线和异常摘要。
- 当前运行任务。
- 最近事件摘要。
- 今天或最近的浇水概况。

总览允许：

- 查看系统摘要。
- 查看当前运行摘要。
- 查看异常摘要。

总览不承载完整配置表单。

总览不重复运行页的完整手动启动表单，不重复记录页或事件页的完整列表。

## 运行

运行页回答：现在能不能浇水，正在浇哪一路，如何手动启动或停止。

运行页显示：

- 当前运行中的水路任务。
- 各水源是否空闲、运行中、禁止启动或离线。
- 可手动启动的水路。
- 运行保护状态，例如低液位、漏水、无流量。

运行页允许：

- 手动启动一个水路任务。
- 停止正在运行的任务。

运行页不编辑计划，不修改硬件映射。

## 计划

计划页回答：什么时候自动浇水，浇哪些水路，浇多久。

计划页显示：

- 计划列表。
- 计划启用状态。
- 下一次触发时间。
- 计划步骤摘要。

计划编辑不直接铺在计划列表旁边或下方。新建或编辑计划进入独立编辑页面，编辑内容包含：

- 名称。
- 启用/停用。
- 每天或自定义周期。
- 开始时间列表。
- 水路步骤。

计划页不显示运行记录明细，不处理系统事件。

## 水源

水源页回答：有哪些水源和水路，每个水源当前能力是否可用。

水源页显示：

- 水源列表。
- 从站地址和在线状态。
- 分支水路列表。
- 主阀、水泵、流量计、低液位是否启用。
- 电压读数。

水源详情显示：

- 水路启用状态。
- 参考流量和校准状态。
- 最近一次运行摘要。

水源配置和水路配置属于水源详情下的独立配置视图，不作为顶层导航页面。水源配置视图应正常展示主阀、水泵、流量计输入、低液位输入和阀 PWM 参数。

水源页不直接发起普通手动浇水；手动入口在运行页。

## 记录

记录页回答：哪些水路真实运行过，运行结果是什么。

记录只展示真实启动过的水路任务。

记录显示：

- 来源：自动计划或手动。
- 水源和水路。
- 开始和结束时间。
- 计划时长和实际时长。
- 估算水量。
- 稳定流速。
- 结束原因。
- 是否触发保护。

记录页不展示未运行的跳过项；跳过进入事件页。

记录页必须支持分页和筛选。记录详情从单条记录进入，不在列表页旁边或下方展开完整详情或表单。

## 事件

事件页回答：为什么没有运行，为什么停止，系统发生了什么需要注意的事。

事件是产品层结构化事件/操作日志，不是底层文本日志。事件显示：

- 用户操作，例如手动启动、用户停止、保存配置、编辑计划。
- 低液位有效或恢复。
- 待机漏水。
- 运行无流量。
- 低流量或过流量提示。
- 计划步骤跳过。
- 从站离线或恢复。
- 地址异常、配置不完整或安全停机待同步。
- 系统参数变更。

事件页不替代浇水记录。

事件页必须支持分页和筛选。事件详情从单条事件进入，不在列表页旁边或下方展开详情或表单。

## 系统参数

系统参数页回答：影响全局运行、通信和保护规则的系统级参数是什么。

系统参数页显示和修改：

- 全局运行参数。
- RS485 通信参数。
- 从站离线判定参数。
- 全局流量保护参数。

系统参数页不显示也不修改水源级和水路级参数。

水源名称、主阀、水泵、流量计、低液位、阀 PWM 等水源级参数在水源详情下的配置视图修改。

水路名称、启用状态、参考流量等水路级参数在水路详情下的配置视图修改。

系统参数页不发起普通浇水，不显示完整记录列表，不作为诊断后台。

## 不做顶层页面

当前不单独设置这些顶层页面：

- 诊断
- 维护
- 统计
- 天气
- 通知
- OTA
- LCD

需要表达时：

- 诊断信息不做独立顶层页；和水源相关的状态进入水源页，系统参数页只保留参数配置。
- 统计先由记录页摘要承担。
- LCD 和主控按键最后单独抽取最小现场能力。
- OTA 暂不进入当前 Web 信息架构。
