# 13 方案审查与明确结论

## 审查结论

当前方案的主线是成立的：主控只做 Web、调度、记录和 RS485 主站；从站对应真实水源；分支水路是计划、手动运行、记录和流量统计的最小对象。

本轮补充后的关键原则是：逻辑清晰合理优先；在逻辑清晰合理的前提下，简单可靠优先。涉及安全动作时，本地安全闭环优先。

需要收紧的是表达层和安全边界，而不是重新推翻对象模型。当前文档中有几处命名不一致、入口不清和配置层级不清的问题；这些问题如果不先收掉，高保真原型会继续变复杂。

## 已修正的矛盾

### 文档入口

原问题：

- 当前文档放在 `docs/v5/`，但当前阶段不再需要用版本目录表达方案。

当前结论：

- 当前方案文档直接放在 `docs/` 下。
- `docs/README.md` 是文档入口。
- 第 5 版低保真和高保真原型已废弃并移入待删除区，当前没有有效原型入口。

### 设置和系统参数命名

原问题：

- 多数文档使用“系统参数”作为顶层页面。
- `10-水路校准与低保真原型准备.md` 仍写“设置”，容易让水源配置、水路配置和系统参数重新混到一个设置页里。

当前结论：

- 顶层页面名称统一为“系统参数”。
- 不再使用“设置”作为顶层页面名。
- 系统参数只放全局运行、通信和保护参数。

### 水源页是否允许配置

原问题：

- 文档一边说水源页不编辑硬件映射，一边又说水源级参数在水源配置页修改。
- 如果不说明“水源配置页”的层级，原型会变成隐藏的第八个顶层页面。

当前结论：

- 水源配置和水路配置不是顶层页面。
- 它们是水源详情和水路详情下的独立配置视图。
- 水源列表页只做查看和进入详情，不在列表旁边或下方直接铺配置表单。
- 列表页统一不和新建、编辑、配置表单同屏。简单表单用弹层，复杂表单和详情用独立页面，保存或取消后返回列表或来源页面。

### 无流量事件的阻断范围

原问题：

- “运行无流量”被列入阻断事件，但同时又写“不长期禁止该水路启动”。

当前结论：

- 无流量是本次任务异常停止事件。
- 它会解释为什么本次任务停止，但停止完成后不长期禁止再次启动。
- 长期影响启动的阻断只保留低液位、待机漏水、从站离线等当前仍有效的问题。

补充结论：

- 长期阻断还包括地址异常、配置不完整和安全停机待同步。
- 事件页表达的是结构化事件/操作日志，不是底层文本日志。

### 校准和浇水记录

原问题：

- 校准会实际打开水路运行，但文档需要明确它是否进入普通浇水记录。

当前结论：

- 校准运行不生成普通浇水记录。
- 校准成功、失败或取消只生成校准相关事件。
- 普通浇水记录只记录自动计划或手动浇水产生的真实水路任务。

## 复杂或过度设计风险

### RS485 通信参数展示层级

原问题：

- 之前把 RS485 通信速率、轮询周期、离线失败次数视为“工程调试项”，倾向于弱化或隐藏。
- 这个判断不适合当前项目。项目主要是个人使用，只要参数真实可配置，就应该按正确分组正常展示。

结论：

- RS485 通信速率、轮询周期、离线失败次数属于系统参数。
- 高保真原型应在系统参数页按“通信参数”分组正常展示和配置。
- 展示重点是默认值、单位、可选范围和修改后生效规则，不需要把它们藏起来。

### 阀 PWM 参数展示层级

原问题：

- 之前把 PWM 参数视为不应在原型中暴露的技术项。
- 实际上 PWM 是阀驱动的可配置安装参数，应该能配置，只是不应逐水路重复配置。

结论：

- PWM 参数按水源/从站统一配置，不逐水路配置。
- 高保真原型应在水源详情下的配置视图中正常展示和配置。
- 展示重点是吸合阶段、保持阶段、关闭阶段等参数的分组和默认值；不把 PWM 放到运行页或手动启动流程里。

### 统一周期模型需要用户友好表达

风险：

- “周期长度、周期起始日期、周期内执行日”作为底层模型没问题，但直接放给用户会显得抽象。

结论：

- 计划底层继续使用统一周期模型。
- 原型界面只保留两个主要入口：“每天”和“自定义周期”。
- 每天作为默认入口；自定义周期用于表达浇几天停几天，也可以覆盖更复杂的周期执行日。

### 事件信息过多会淹没异常

风险：

- 如果配置变更、任务正常完成等信息事件和低液位、漏水、离线混在同一视觉层级，事件页会变成流水账。

结论：

- 事件页优先显示当前提示和阻断。
- 信息事件保留历史查询，但不进入当前异常数量，也不抢占首屏。
- 用户操作、配置变更、计划触发、计划跳过、校准、从记录采用参考流量等都进入结构化事件/操作日志。

### SHT30、LCD2004 和本地按键不进入 Web 主流程

风险：

- SHT30 和 LCD2004 都是外接能力，按键本地操作也尚未抽取最小能力。
- 过早进入 Web 原型，会让当前主线从灌溉控制偏到外设展示。

结论：

- 当前 Web 原型不设计 LCD 菜单和本地按键流程。
- SHT30 最多作为后置的主控附近环境读数，不作为灌溉控制、计划或保护条件。

### 电压读数不要扩大成诊断系统

风险：

- 从站电压当前只确认“显示读数”，如果继续扩展告警、维护建议或停机，会超出已确认边界。

结论：

- 电压只显示，不告警、不保护、不停机。
- 高保真原型里只做低权重状态信息。

### 从站本地安全闭环

风险：

- 如果从站只是被动 IO 扩展，运行中 RS485 失联后，主控可能无法继续下发停止命令。
- 灌溉系统的安全输出状态应该明确，不能依赖“通信恢复后再处理”。

结论：

- 从站必须具备本地运行闭环和通信失联安全停机能力。
- 运行期间通信保持信号超时后，从站本地关闭泵和阀。
- 主控发现离线时，运行任务按通信失联异常处理；通信恢复后重新同步输出状态和安全停机结果。
- 从站仍不保存计划、不做脚本、不承担复杂业务逻辑。

### 从站地址和发现

结论：

- 从站地址来自 4 位拨码，Web 不修改地址。
- 有效地址使用 `1..15`，地址 `0` 作为未配置或保留地址。
- 主控扫描有效地址，发现未绑定从站后由用户确认创建或绑定水源。
- 地址冲突只能提示疑似冲突，需要现场修改拨码解决。
- 配置不完整的水源不能启动任务，但可以显示和继续配置。

### 校准统计

结论：

- 校准不生成普通浇水记录，只生成校准事件。
- 校准事件保留运行时长、估算水量、稳定流速、是否保存参考流量和失败原因等字段。
- 校准用水不计入今日浇水次数、今日浇水总时长和今日浇水估算水量。

## 明确后的方案基准

### 顶层页面

高保真原型继续按 7 个顶层页面推进：

1. 总览
2. 运行
3. 计划
4. 水源
5. 记录
6. 事件
7. 系统参数

不新增诊断、维护、统计、天气、通知、OTA、LCD 顶层页面。

### 配置作用域

- 系统参数：全局运行、全局保护、RS485 通信和离线判定。
- 水源配置：从站/水源名称、主阀、水泵、流量计输入、低液位输入、阀 PWM 参数。
- 水路配置：水路名称、启用状态、参考流量和校准状态。

### 运行规则

- 同一水源同一时刻只运行一路。
- 不同水源可以并行运行。
- 手动运行不排队、不抢占。
- 自动计划触发时，水源不可用则跳过该水源本次步骤，生成事件，不补跑。
- 运行中通信失联时，从站本地安全停机，主控恢复通信后同步结果。

### 保护规则

- 低液位、待机漏水、从站离线、地址异常、配置不完整和安全停机待同步会阻断新任务。
- 运行无流量导致本次任务异常停止，但不长期禁止再次启动。
- 低流量和过流量默认提示；如果配置为停机，只影响本次任务。

### 记录和事件

- 浇水记录只记录真实启动过的自动或手动水路任务。
- 跳过只生成事件，不生成浇水记录。
- 异常停止既补全浇水记录，也生成事件。
- 校准不生成普通浇水记录，只生成校准事件。
- 事件是结构化事件/操作日志，用于记录用户操作、自动运行、状态保护、通信设备和校准事实。

### 校准

- 校准从水路详情进入，不做顶层页面。
- 校准默认采集稳定流量 60 秒。
- 正常浇水记录中的稳定流速可以被采用为该水路参考流量。

## 高保真原型前需要确认

- 每页区块顺序。
- 表格、详情和独立页面的使用边界。
- 主要按钮和危险操作确认方式。
- 高保真先按桌面 Web 管理界面设计，移动端适配后置。
