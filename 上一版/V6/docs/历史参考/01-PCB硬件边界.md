# 01 PCB 硬件边界

## 资料来源

当前 `pcb/` 目录存在 4 个硬件文件，对应主控板和从站板：

```text
pcb/BOM_Board1-主控_Schematic1_2026-06-18.xlsx
pcb/Netlist_Schematic1_2026-06-18.tel
pcb/BOM_Board2-从站_Schematic2_2026-06-18.xlsx
pcb/Netlist_Schematic2_2026-06-18.tel
```

当前硬件边界可以重新确认：本项目不是单主控直接控制阀的产品，而是“ESP32 主控 + 至少 1 个从站”的灌溉控制系统。

## 硬件事实分级

- 已确认：当前 BOM 和网表能直接支持，且本轮用户已经确认进入产品方向。
- 存在但当前目标产品不启用：PCB 上存在，但本轮已确认不使用，或只作为预留/实验能力。
- 待后续设计：硬件存在，但产品逻辑、Web 表达和固件行为还没有进入本轮确认。

## 已确认的系统结构

- 主控板只做主控。
- 从站板负责真实水源侧执行。
- 当前产品必须支持从站；没有从站时系统不能完成灌溉控制。
- 至少需要 1 个从站才能工作。
- 一个从站对应一个真实水源。
- 主控通过 RS485 控制从站。
- 当前目标产品不使用主控板本地 2 路阀输出。
- 当前目标产品不使用主控板干接点输出。
- 当前目标产品不使用主控侧脉冲输入、ADC、NTC 等预留输入做灌溉业务。
- Web 先设计完整功能逻辑；LCD2004 和本地按键只在后续从 Web 功能中抽取最小必要现场能力。

## 主控板边界

### 当前目标产品启用

主控板在当前目标产品中启用以下能力。

#### ESP32 主控

BOM：

```text
U1 = ESP-WROOM-32E
```

用途：

- 作为系统主控。
- 承载 Web、配置、调度、记录、状态汇总和 RS485 主站通信等后续待设计能力。

这里先只确认硬件角色，不确认具体 Web/API/固件实现。

#### RS485 主站通信

BOM：

```text
U16 = TP8485E-SR
D10 = SM712
L4 = ACP3225-102-2P-T000
R40 = 120Ω
```

网表：

```text
485_DIR
485_RX
485_TX
A
B
```

用途：

- 主控通过 RS485 与从站通信。
- RS485 是主控和从站之间的核心控制链路。

当前只确认硬件链路，不在本文确认协议帧、命令集或重试策略。

#### RTC 时钟

BOM：

```text
U9 = DS3231SN#
BT1 = BS-10-B2AA010
```

网表：

```text
I2C_SCL
I2C_SDA
VBAT
IO4_INT
```

用途：

- 主控需要可靠时间能力。
- 后续计划调度、记录时间和本地显示可以基于 RTC 设计。

#### 外接 I2C 显示和温湿度

主控板 BOM 中没有 LCD2004 或 SHT30 芯片本体；当前 PCB 提供 I2C 和 5V I2C 外设接口。

相关 BOM：

```text
U6 = PCA9306DCTR
U3 = 4Pin 外设接口
```

网表：

```text
I2C_SCL
I2C_SDA
SCL2
SDA2
```

当前确认：

- 当前目标产品考虑外接 LCD2004 作为本地显示屏。
- 当前目标产品考虑外接 SHT30 或同类 I2C 温湿度传感器，用于观察主控附近温湿度。
- LCD2004 的显示内容和按键操作不在当前 Web 原型阶段展开；后续从 Web 完整功能中抽取最小必要本地能力。

风险：

- SHT30 不是 BOM 中的板载芯片；如果需要作为正式产品能力，后续必须确认具体外接模块、接口和安装方式。
- LCD2004 也不是 BOM 中的板载显示器；后续必须确认具体 I2C 背包地址、供电和安装方式。

#### 主控本地按键

网表：

```text
H2 = BTN_OK / BTN_ESC / BTN_UP / BTN_DOWN / GND
```

当前确认：

- 主控有 4 个外接业务按键。
- 这些按键后续用于本地 LCD2004 的最小现场操作。
- 本轮 Web 原型先不设计 LCD2004 菜单。

#### BOOT / GPIO0 长按恢复出厂

网表：

```text
BOOT
SW2
```

用户确认的产品意图：

- 启动后长按 GPIO0 / BOOT 按键 5 秒或 10 秒，用于恢复出厂设置。

当前边界：

- 这是后续固件和安全交互设计项。
- 本文只记录硬件和意图，不确认触发时长、确认机制、恢复范围或是否需要 Web 二次确认。

#### 主控状态灯

BOM：

```text
LED1..LED6
```

网表中可作为主控状态分配的 MCU 可控灯：

```text
IO27_LED = LED2
IO14_LED = LED3
```

其它 LED 已绑定到固定硬件链路：

```text
LED1 = 电源/固定指示链路
LED4 = 主控干接点输出指示
LED5 / LED6 = 主控本地阀输出指示
```

当前确认：

- 当前目标产品不使用主控干接点和主控本地阀输出，因此 `LED4/LED5/LED6` 不作为系统状态灯设计。
- 主控可分配的产品状态灯按 `LED2/LED3` 两个处理。
- 状态灯设计遵循最少分配原则，不为了有灯就增加复杂状态。

### 8 路阀输出

BOM：

```text
Q1..Q8 = IRLR7843TRPBF
U5/U6/U9/U10 = EG27324
D8/D9/D10/D11/D18/D19/D20/D21 = SS54
D14/D15/D16/D17/D22/D23/D24/D25 = BZT52C15
LED7..LED14 = 输出指示相关 LED
```

网表：

```text
PWM1..PWM8
DRV_PWM1..DRV_PWM8
DRV_SD
VALVE1-..VALVE8-
```

当前确认：

- 从站板具备 8 路电磁阀输出硬件。
- `DRV_SD` 作为阀驱动使能相关网络存在。
- 主阀默认使用阀 8，但可在水源配置中选择具体阀号。
- 被选为主阀的阀不再作为分支水路。
- 未被选为主阀的阀作为分支水路。

### 4 路数字输入

网表：

```text
H4 = IN1 / IN2 / IN3 / IN4
T_PULSE_IN1..T_PULSE_IN4
U7/U11 = SN74LVC2G17DBVR
```

当前确认：

- 从站板具备 4 路数字输入硬件。
- 这些输入可作为水源侧传感输入的硬件基础。
- 流量计默认使用 `IN1`，可在水源配置中改为 `IN1..IN4` 中的一路。
- 低液位默认使用 `IN2`，可在水源配置中改为 `IN1..IN4` 中的一路。
- 低液位支持常开/常闭和去抖设置。
- 同一输入不能同时承担流量计和低液位。
- 当前目标产品不把这些输入设计成通用 IO 或任意传感器平台。

### 从站电压采样和预留 ADC

网表：

```text

ADC_IN1 / ADC_SEN1
ADC_IN2 / ADC_SEN2

```

当前确认：

- 从站板存在系统电压采样相关网络 `ADC_VIN`。
- 从站板还有 2 路外部 ADC 预留接口 `H8/H9`。
- 从站电压采样只用于显示电压读数。
- 电压读数不产生告警、事件、维护建议、启动禁止、自动停机或其它保护动作。
- `H8/H9` 作为外部 ADC 预留接口，不进入当前目标产品功能。

- 阀输出指示灯、干接点输出指示灯只作为对应硬件动作指示，不承担额外系统状态。

后续需要裁决：

- 本地按键是否只用于维护识别、清除故障或其它现场动作。

## 本轮已确认但仍需后续设计的硬件产品规则

以下规则已由用户在当前重建阶段确认：

- 当前产品必须支持从站。
- 至少一个从站才能完成灌溉工作。
- 一个从站对应一个真实水源。
- 主控只做主控，不直接使用主控侧阀输出和主控侧水源输入。
- 主控使用 ESP32、RS485、RTC、外接 LCD2004、外接 SHT30 或同类温湿度传感器、4 个业务按键，以及 BOOT/GPIO0 长按恢复出厂意图。
- 从站负责水源侧执行和传感输入。
- 主阀默认使用阀 8，但允许在水源配置中选择具体阀号。
- 从站电压采样只显示电压读数，不参与告警、维护、保护或停机。
- Web 原型先完整表达主要功能逻辑；LCD2004 和本地按键能力后置，只抽取最小现场能力。

## 后续待办

硬件边界已恢复到主控 + 从站两板。LCD2004 和主控本地按键暂不设计：

- 等产品范围、状态模型、信息架构和 Web 原型方案全部确定后，最后单独抽取最小现场能力。
