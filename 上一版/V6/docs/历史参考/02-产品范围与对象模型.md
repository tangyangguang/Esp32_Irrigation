# 02 产品范围与对象模型

## 产品定位

当前目标产品是智能浇水/灌溉系统，不绑定家庭庭院、农场、菜地、雨棚、露天等具体使用场景。设计时只按灌溉系统本身的对象、硬件能力、状态和操作任务建模。

当前目标产品不按第一版、第二版或 MVP 拆分。本轮需要把期望功能设计完整；后续如果继续变化，应作为优化、缺陷修复或硬件小改处理。

## 用户边界

当前按一个用户完成安装、使用和维护处理：

- 不设计多用户、多角色、多权限。
- 不设计云端运维后台。
- 不按安装者、使用者、维护者拆成不同产品路径。

## 核心对象

### 主控

主控是系统中心：

- 提供 Web 管理入口。
- 负责时间、配置、调度、记录和状态汇总。
- 通过 RS485 控制从站。
- 不直接控制灌溉阀。
- 不使用主控侧输入做水源传感。

### 从站

从站是水源侧执行单元：

- 一个从站对应一个真实水源。
- 至少一个从站才能完成灌溉工作。
- 一个从站最多提供 8 路阀输出。
- 一个从站提供水源侧输入、泵输出和本地状态基础。
- 从站不是简单 IO 扩展；运行中必须具备本地安全闭环，通信失联时能够进入安全停机状态。

### 水源

水源是用户理解灌溉能力的上层对象。当前产品中，水源和从站是一一对应关系：

- 一个从站 = 一个水源。
- 水源下包含分支水路、流量计、低液位输入、自吸泵/水泵输出等能力。
- Web 中可以面向用户显示“水源”，但系统实现和诊断必须能对应到具体从站。

### 分支水路

分支水路是真正执行浇水的对象：

- 每个分支水路对应一个电磁阀输出。
- 没有主阀时，一个从站最多 8 个分支水路。
- 使用主阀时，一个从站最多 7 个分支水路。
- 分支水路是计划、手动浇水、运行记录和流量统计的主要对象。

## 从站阀输出规则

从站有 8 路 PWM 阀输出。主阀是否启用、使用哪一路阀，由水源配置决定。

默认规则：

- 默认不启用主阀时，阀 1 到阀 8 都是分支水路。
- 默认启用主阀时，主阀使用阀 8。
- 可以在水源配置中把主阀改为阀 1 到阀 8 中的任意一路。

运行规则：

- 被选为主阀的阀不再作为分支水路。
- 使用主阀时，一个从站最多 7 个分支水路。
- 分支水路运行时，主阀需要按规则联动打开。
- 主阀本身不作为独立浇水水路。
- 不使用主阀时，一个从站最多 8 个分支水路。

## 阀 PWM 驱动规则

8 路阀输出支持 PWM 控制，用于降低长期工作发热并延长电磁阀寿命。

当前产品采用从站级统一 PWM 参数：

- 不按每一路阀单独配置 PWM。
- 不在运行页、手动启动流程或普通水路操作中调节 PWM。
- PWM 参数属于水源配置，同一水源/从站统一使用，不逐水路配置，并在水源配置中正常展示和配置。
- 后续状态模型需要确认吸合阶段、保持阶段、关闭阶段和异常关闭时的行为。

## 从站输入与输出映射

输入和输出采用“默认约定 + 水源配置可改”的方式。它不是运行页里的日常操作，也不是单独的高级配置体系；应在水源配置中正常展示。

### 流量计输入

默认约定：

- 流量计使用 `IN1`。

水源配置：

- 可在 `IN1` 到 `IN4` 中选择哪一路作为流量计输入。
- 同一输入不能同时承担流量计和低液位。
- 流量计是水源级输入，不属于某一个分支水路。

### 低液位输入

默认约定：

- 低液位使用 `IN2`。

水源配置：

- 可在 `IN1` 到 `IN4` 中选择哪一路作为低液位输入。
- 支持常开/常闭配置。
- 支持去抖时间配置。
- 同一输入不能同时承担低液位和流量计。

### 预留输入

`IN3` 和 `IN4` 默认不承担产品功能。它们可以作为流量计或低液位的可选输入来源，但当前产品不把它们设计成通用 IO、脚本输入或任意传感器平台。

### 自吸泵/水泵输出

从站干接点输出用于自吸泵或水泵控制：

- 当前硬件只有 1 路从站干接点输出。
- 该输出固定作为自吸泵/水泵控制输出。
- 是否启用该输出属于水源配置。
- 后续状态模型需要确认泵与主阀、分支水路、流量保护、低液位保护之间的联动顺序。

## 电压显示

从站电压采样只作为读数显示：

- 不产生告警。
- 不进入事件。
- 不产生维护建议。
- 不禁止启动。
- 不自动停机。
- 不参与保护策略。

## 状态灯边界

状态灯按最少分配原则设计：

- 主控可用于产品状态的 MCU 可控灯按 2 个处理。
- 从站硬件最多有 3 个 MCU 可控灯，但当前产品状态灯按 2 个处理。
- 阀输出灯只表示对应阀输出动作。
- 干接点输出灯只表示泵/干接点输出动作。
- 不为了用满状态灯而增加状态分类。

## 系统参数和对象配置边界

系统参数只用于全局运行、通信和保护规则，例如：

- 最大单次运行时长。
- 主阀和水泵启停时间默认值。
- RS485 通信速率、轮询周期和离线失败次数。
- 流量保护的全局默认参数，例如启动宽限、漏水判定、低流量比例、过流量比例和持续时间。

水源配置用于一个从站/水源的水源级配置，例如：

- 是否启用主阀，以及主阀使用哪一路阀，默认阀 8。
- 流量计输入选择。
- 低液位输入选择、常开/常闭和去抖。
- 自吸泵/水泵输出是否启用。
- 从站级 PWM 参数。

水路配置用于一个分支水路的名称、启用状态和参考流量。

这些配置不是“高级/低级”分层，也不应拆成日常配置、高级参数、维护参数等多套页面。后续 Web 信息架构应按对象和任务组织，而不是按主观复杂度组织。
