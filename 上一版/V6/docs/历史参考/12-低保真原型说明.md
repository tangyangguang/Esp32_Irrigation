# 12 低保真原型说明

## 目标

低保真原型只用于确认产品结构，不确认视觉风格。

本阶段必须回答：

- 顶层到底有几个页面。
- 每个页面负责什么、不负责什么。
- 用户完成主要任务时从哪里进入、到哪里结束。
- 哪些信息必须在页面上出现，哪些信息不能被提前复杂化。

本阶段不做：

- 高保真视觉。
- 配色、品牌、图标细节。
- LCD2004 和主控四个业务按键。
- 固件目录、API、存储实现和通信协议细节。

## 顶层导航

低保真暂定 7 个顶层页面：

1. 总览
2. 运行
3. 计划
4. 水源
5. 记录
6. 事件
7. 系统参数

这些页面按对象和任务组织，不按“高级、维护、日常”拆分。

## 信息架构规则

- 页面内不做跨页面快捷入口，跨页面移动依赖顶层导航。
- 页面内容避免重复。其它页面只保留摘要，不重复专门页面的完整模块。
- 列表页不在列表旁边或下方直接铺新建、编辑、配置表单。
- 列表页只承载筛选、分页、列表、详情入口和必要行级动作。
- 简单表单通过弹层出现；复杂表单或需要完整上下文的配置进入独立页面。
- 保存或取消后返回列表或来源页面；详情进入独立详情页，不和列表同屏展开。
- 记录和事件默认分页。
- 页面中的内容和元素必须来自当前已确认方案。原型可以使用少量示例数据验证列表、分页、状态和空状态，但示例数据不能引入未确认功能。
- 状态图只列核心状态。简单、直观、不影响页面设计的状态，不单独展开。

## 页面边界

### 总览

总览回答：现在系统是否正常，是否正在浇水，是否有需要处理的问题。

页面区块：

- 系统状态：主控时间、从站数量、当前异常数。
- 当前运行：正在运行的水源、水路、剩余时间、估算水量。
- 水源概况：每个水源的在线、运行、禁止启动、离线状态。
- 最近事件：最近几条提示或阻断事件。
- 今日概况：今日运行次数、总时长、估算总水量。

允许操作：

- 查看系统摘要。
- 查看当前运行摘要。
- 查看水源和异常摘要。

不允许：

- 编辑计划。
- 修改硬件映射。
- 展开完整记录列表。
- 放置运行页的完整手动启动表单。

### 运行

运行回答：现在能不能浇水，正在浇哪一路，如何手动启动或停止。

页面区块：

- 当前任务：按水源显示正在运行的任务。
- 水源可用性：空闲、运行中、禁止启动、离线。
- 手动启动：点击按钮后在弹层中选择水源、水路、时长来源和时长。
- 启动阻断：低液位、漏水、离线、忙碌、水路停用、配置不完整、地址异常。
- 保护状态：低液位、漏水、无流量、低流量、过流量。

允许操作：

- 手动启动一个水路任务。
- 停止正在运行的任务。
- 查看阻断原因。

不允许：

- 编辑自动计划。
- 修改系统参数。
- 修改水源硬件映射。

### 计划

计划回答：什么时候自动浇水，浇哪些水路，浇多久。

页面区块：

- 计划列表：名称、启用状态、下一次触发、步骤摘要。
- 筛选或状态切换：全部、启用、停用。
- 分页：计划数量增加时使用。
- 计划编辑视图：每天或自定义周期、开始时间列表、水路步骤。

允许操作：

- 新建计划。
- 编辑计划。
- 启用或停用计划。
- 添加、删除、排序水路步骤。

不允许：

- 直接启动手动浇水。
- 显示浇水记录明细。
- 处理事件。
- 把计划编辑表单直接放进计划列表区域。

关键规则：

- 停用计划只阻止后续触发，不停止已经运行的任务。
- 修改计划只影响下一次触发，不改变已经展开的本次任务。
- 计划触发时水源不可用则跳过该水源本次步骤，并生成事件。
- 每天是默认计划入口；自定义周期用于表达浇几停几和更复杂的周期执行日。

### 水源

水源回答：有哪些水源和水路，每个水源当前能力是否可用。

页面区块：

- 水源列表：名称、从站地址、在线状态、运行状态、异常状态。
- 水源详情：主阀、水泵、流量计、低液位、电压读数。
- 水路列表：名称、启用状态、参考流量、最近运行。
- 校准入口：每条水路可进入校准。
- 水源配置视图：主阀阀号、水泵启用、流量计输入、低液位输入、阀 PWM 参数。
- 从站发现和配置状态：未配置从站、地址异常、配置不完整。

允许操作：

- 查看水源详情。
- 查看水路详情。
- 进入水路校准。

不允许：

- 普通手动浇水。
- 在水源列表中直接编辑硬件映射。
- 展开完整记录或事件历史。

说明：

- 电压只显示，不告警、不保护、不停机。
- 校准不是顶层页面，只从水路详情进入。
- 水源配置和水路配置不是顶层页面；它们是水源详情和水路详情下的独立配置视图。
- 主阀、流量计、低液位和阀 PWM 都是水源级可配置项，不固定死在默认位置。
- 从站地址来自拨码硬件，Web 只显示、发现和绑定，不修改地址。

### 记录

记录回答：哪些水路真实运行过，运行结果是什么。

页面区块：

- 记录列表：时间、水源、水路、来源、时长、水量、稳定流速、结束原因。
- 筛选：时间、水源、水路、来源、结束原因。
- 分页。
- 记录详情视图：开始结束、计划时长、实际时长、脉冲数、估算水量、稳定流速、保护触发。

允许操作：

- 查看记录详情。
- 筛选记录。
- 对正常完成且流速稳定的记录执行“采用为参考流量”。

不允许：

- 展示未运行的跳过项。
- 处理事件。
- 发起浇水。
- 把详情或表单直接放进记录列表区域。

说明：

- 跳过只进入事件，不进入浇水记录。
- 记录只保存真实启动过的水路任务。

### 事件

事件回答：为什么没有运行，为什么停止，系统发生了什么需要注意的事。

页面区块：

- 当前事件：仍在发生或仍影响启动的提示、阻断事件。
- 事件列表：结构化事件/操作日志，统一展示用户操作、自动运行、状态保护、通信设备和校准事件。
- 筛选：来源分类、影响级别、状态、水源、时间。
- 分页。
- 事件详情视图：时间、水源、水路、原因、影响、恢复时间。

事件分类：

- 信息：用户操作、配置变更、任务正常完成、用户停止、校准完成等。
- 提示：低流量、过流量、计划步骤跳过等默认不强制停机的情况。
- 阻断：低液位、漏水、离线、配置不完整、地址异常、安全停机待同步等会影响启动的情况；无流量只表达本次任务异常停止，不长期禁止再次启动。

允许操作：

- 查看事件详情。

不允许：

- 手动清除发生中事件。
- 把事件当作浇水记录。
- 发起普通浇水。
- 把详情或表单直接放进事件列表区域。

### 系统参数

系统参数回答：影响全局运行、通信和保护规则的系统级参数是什么。

页面区块：

- 全局运行参数：最大单次运行时长、主阀和水泵启停时间。
- 通信参数：RS485 通信速率、从站轮询周期、离线失败次数。
- 流量保护参数：启动宽限、无流量确认、漏水窗口、低/过流量比例和动作。

允许操作：

- 修改系统级参数。
- 保存或取消。

水源名称、主阀、水泵、流量计、低液位、阀 PWM 等水源级参数在水源详情下的配置视图修改。

水路名称、启用状态、参考流量等水路级参数在水路详情下的配置视图修改。

不允许：

- 发起普通浇水。
- 展示完整记录列表。
- 作为通用诊断后台。

运行中限制：

- 水源运行中不能修改该水源主阀、流量计、低液位、水泵映射。
- 水源运行中不能停用正在运行的水路。
- 停用水源或水路只阻止新任务，不停止正在运行的任务。

## 主要任务流

### 查看当前状态

1. 用户通过顶层导航打开总览。
2. 查看系统状态、当前运行、水源概况和最近事件。
3. 如果需要处理，用户通过顶层导航进入对应页面。

### 手动启动水路

1. 用户通过顶层导航打开运行页。
2. 选择空闲且可用的水源。
3. 选择启用的分支水路。
4. 选择时长来源：本次输入、上次手动时长、从计划步骤带入。
5. 系统检查低液位、漏水、离线、忙碌、水路停用和最大单次运行时长。
6. 检查通过后启动任务。
7. 任务进入当前任务区，并生成浇水记录。

### 停止任务

1. 用户通过顶层导航打开运行页，选择正在运行的任务。
2. 点击停止。
3. 系统按停止顺序停泵、关分支阀、关主阀。
4. 浇水记录结束原因为用户停止。
5. 事件页记录一次用户停止事件。

### 编辑自动计划

1. 用户通过顶层导航打开计划页。
2. 新建或选择一个计划。
3. 进入计划编辑视图。
4. 选择每天或自定义周期；每天作为默认，自定义周期用于设置周期长度、周期起始日期和周期内执行日。
5. 设置一个或多个开始时间。
6. 添加水路步骤，设置水源、水路、时长、启用状态。
7. 保存后只影响未来触发。

### 计划触发

1. 到达触发时间。
2. 系统按水源拆分任务序列。
3. 同一水源串行运行，不同水源可并行。
4. 水源忙碌或不具备启动条件时，该水源本次步骤跳过。
5. 跳过生成事件，不生成浇水记录，不排队，不补跑。

### 水路校准

1. 用户通过顶层导航打开水源页，再打开水源详情。
2. 选择某条水路进入校准。
3. 系统检查水源在线、空闲、低液位、漏水和水路启用状态。
4. 系统按正常启停顺序运行水路。
5. 采集稳定流量，默认 60 秒。
6. 用户确认后保存为该水路参考流量。
7. 系统生成校准事件，不生成普通浇水记录，不计入今日普通浇水统计。

### 从记录采用参考流量

1. 用户通过顶层导航打开记录页。
2. 打开一条正常完成且流速稳定的记录。
3. 点击“采用为参考流量”。
4. 系统把该次稳定流速写入对应水路。
5. 系统生成参考流量更新事件。

### 修改系统参数

1. 用户通过顶层导航打开系统参数页。
2. 用户修改系统级参数。
3. 用户保存参数。
4. 系统生成系统参数变更事件。

## 低保真评审口径

评审时只看这些问题：

1. 7 个顶层页面是否够用，是否有页面应该合并或拆出。
2. 每个页面的职责是否清楚，是否有跨页重复造成混乱。
3. 手动、计划、校准、记录、事件、系统参数这 6 条任务流是否走得通。
4. 记录和事件分离是否符合使用习惯。
5. 水源页只查看和校准，普通手动浇水只在运行页，是否合理。
6. 系统参数页只放系统级参数，水源级和水路级参数回到对应页面，是否足够清晰。
7. 诊断、维护、统计、天气、通知、OTA、LCD 是否继续不做顶层页面。

## 进入高保真前的确认点

低保真确认后，才能进入高保真原型。

进入高保真前需要得到明确结论：

- 顶层导航最终数量和名称。
- 每页区块顺序。
- 表格、详情和独立页面的使用边界。
- 主要按钮和危险操作确认方式。
- 移动端后置；高保真先按桌面 Web 管理界面确认内容和信息架构。
