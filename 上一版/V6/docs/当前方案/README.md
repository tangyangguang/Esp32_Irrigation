## 当前工作原则

- 先重新判断目标、约束和成功标准，不沿旧方案惯性优化。
- 所有历史结论都要重新审查，不能因为已经写过就继承。
- 逻辑清晰优先；在逻辑清晰的前提下，简单可靠优先。
- 不为未确认场景增加复杂度。
- 面向个人使用，可配置项不因为技术性强而隐藏，但必须有清楚分组和默认值。
- 涉及硬件能力时，只以当前 PCB BOM 和网表为依据。
- 从站是固定水源侧执行单元，不做通用 IO 平台，也不做复杂安全控制器。
- Web 界面方案验证页面结构、任务流、信息层级和视觉表达；软件收口文档用于正式编码前确认实现边界。
- 页面内容必须使用真实逻辑、默认值、空状态或待配置状态，不使用虚构设备和虚构记录。

## 当前判断

- 主线应围绕“主控 + 从站 + 水源 + 水路”建立。
- 主控只做系统中心，不直接做灌溉执行。
- 从站是水源侧执行单元，负责阀、流量计、低液位输入和自吸泵/水泵输出。
- Web 首先表达用户需要判断和操作的对象，不做工程调试后台。
- 主控基础环境使用 `Esp32Base` 完整能力，包括 Web、OTA、RTC、FS、App Config、App Events 和 RS485 基础口；业务实现仍保持克制。
- 分站 8 路 PWM 可使用 `Stc8hBase` 的 PWM `pin_select` 能力；RS485 使用 `STC8H_UART1_PIN_GROUP=1` 选择 UART1 P3.6/P3.7。
- LCD、本地按键、天气、统计独立页等都先后置。
