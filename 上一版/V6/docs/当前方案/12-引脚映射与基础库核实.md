# 12 引脚映射与基础库核实

## 目标

本文用于在正式实现前，把当前 PCB 网表、芯片引脚功能和基础库能力对齐。

结论只服务当前项目：

- 不重新设计硬件。
- 不在业务项目里补基础库能力。
- 能由 `Esp32Base` / `Stc8hBase` 提供的能力，项目只做配置和调用。
- 如果基础库缺失通用能力，先回基础库补齐，再回本项目实现。

## 主控 ESP32

依据：

- `pcb/Netlist_Schematic1_2026-06-18.tel`
- 主控 BOM：`ESP-WROOM-32E`、`DS3231SN#`、`TP8485E-SR`
- ESP-WROOM-32E 模组常用引脚定义

### 主控必须使用的基础能力

主控基础环境继续按完整能力启用：

- Web / WiFi / 认证
- OTA
- FS
- RTC / DS3231
- Watchdog / Health
- App Config
- App Events
- RS485 基础口

项目业务只在这些能力上做灌溉逻辑，不重新实现系统能力。

### 主控关键引脚

| 功能 | 网表网络 | U1 引脚 | ESP32 GPIO | 项目用途 | 基础库能力 |
| --- | --- | ---: | ---: | --- | --- |
| RS485 DIR | `485_DIR` | 16 | GPIO13 | 半双工方向控制 | `Esp32BaseRs485Port` |
| RS485 RX | `485_RX` | 27 | GPIO16 | 主控接收分站响应 | `Esp32BaseRs485Port` |
| RS485 TX | `485_TX` | 28 | GPIO17 | 主控发送请求 | `Esp32BaseRs485Port` |
| I2C SDA | `I2C_SDA` | 33 | GPIO21 | DS3231 / I2C 总线 | `Wire` + `Esp32Base` RTC |
| I2C SCL | `I2C_SCL` | 36 | GPIO22 | DS3231 / I2C 总线 | `Wire` + `Esp32Base` RTC |

说明：

- RS485 引脚可以直接作为 `Esp32BaseRs485Port::Config` 的 `rxPin`、`txPin`、`dePin`。
- RTC 不在项目里写 DS3231 驱动，走 `Esp32Base` 的 RTC 能力。
- 主控网表里还有本地按键、干接点和 2 路 PWM 相关网络；当前软件主线先不把它们做成业务控制能力，只保留硬件事实。

## 分站 STC8H8K64U

依据：

- `pcb/Netlist_Schematic2_2026-06-18.tel`
- 从站 BOM：`STC8H8K64U-45I-LQFP48`
- `Stc8hBase/docs/vendor/stc/STC8H8K64U_Features.pdf`
- `Stc8hBase` 当前 PWM / UART / RS485 driver 代码

### 分站职责

分站是固定水源侧执行单元：

- 接收主控 RS485 命令。
- 控制 8 路阀/驱动输出。
- 读取流量、低液位、输入和电压类信号。
- 维护当前任务的本地执行和异常停机。

分站不做：

- 通用 IO 平台。
- 独立计划。
- Web。
- 复杂历史。
- 主动抢占 RS485 总线。

## 分站 RS485

| 功能 | 网表网络 | U1 引脚 | STC 引脚功能 | 当前基础库状态 |
| --- | --- | ---: | --- | --- |
| RS485 DIR | `485_DIR` | 29 | P4.1 | `drv_rs485_uart` 可通过板级宏控制 |
| RS485 RX | `485_RX` | 27 | P3.6 / `RxD_2` | `STC8H_UART1_PIN_GROUP=1` |
| RS485 TX | `485_TX` | 28 | P3.7 / `TxD_2` | `STC8H_UART1_PIN_GROUP=1` |

当前判断：

- `drv_rs485_uart` 的方向控制模型可用，项目侧只需在 `board_pins.h` 定义 `BOARD_RS485_TX_ENABLE()` 和 `BOARD_RS485_RX_ENABLE()`。
- 这块分站板的 UART 不在默认 P3.0/P3.1，而在 P3.6/P3.7。
- `Stc8hBase` 已在 `bf072c8 Add UART1 pin group selection` 中支持 UART1 引脚组选择。
- 分站工程必须配置 `STC8H_UART1_PIN_GROUP=1`。
- 项目内不手写 `P_SW1` 寄存器补丁。

## 分站 8 路 PWM

分站网表的 8 路输出可以继续保持业务编号 `1..8`，但底层要按实际复用脚配置。

建议映射：

| 业务输出 | 网表网络 | U1 引脚 | STC 引脚功能 | PWM 配置 |
| --- | --- | ---: | --- | --- |
| 输出 1 | `PWM1` | 33 | P2.0 / `PWM1P_2` | `STC8H_PWM_GROUP_A`, `STC8H_PWM_CHANNEL_1`, `STC8H_PWM_PIN_PWM1_P20` |
| 输出 2 | `PWM2` | 35 | P2.2 / `PWM2P_2` | `STC8H_PWM_GROUP_A`, `STC8H_PWM_CHANNEL_2`, `STC8H_PWM_PIN_PWM2_P22` |
| 输出 3 | `PWM3` | 37 | P2.4 / `PWM3P_2` | `STC8H_PWM_GROUP_A`, `STC8H_PWM_CHANNEL_3`, `STC8H_PWM_PIN_PWM3_P24` |
| 输出 4 | `PWM4` | 39 | P2.6 / `PWM4P_2` | `STC8H_PWM_GROUP_A`, `STC8H_PWM_CHANNEL_4`, `STC8H_PWM_PIN_PWM4_P26` |
| 输出 5 | `PWM5` | 43 | P0.0 / `PWM5_3` | `STC8H_PWM_GROUP_B`, `STC8H_PWM_CHANNEL_5`, `STC8H_PWM_PIN_PWM5_P00` |
| 输出 6 | `PWM6` | 44 | P0.1 / `PWM6_3` | `STC8H_PWM_GROUP_B`, `STC8H_PWM_CHANNEL_6`, `STC8H_PWM_PIN_PWM6_P01` |
| 输出 7 | `PWM7` | 45 | P0.2 / `PWM7_3` | `STC8H_PWM_GROUP_B`, `STC8H_PWM_CHANNEL_7`, `STC8H_PWM_PIN_PWM7_P02` |
| 输出 8 | `PWM8` | 46 | P0.3 / `PWM8_3` | `STC8H_PWM_GROUP_B`, `STC8H_PWM_CHANNEL_8`, `STC8H_PWM_PIN_PWM8_P03` |

当前基础库状态：

- `stc8h_pwm_init_channel(group, channel, pin_select)` 已提供 `pin_select`。
- 以上 `pin_select` 宏在 `Stc8hBase` 中已存在。
- 项目不需要自写 PWM 寄存器。

注意：

- 同一 PWM 组共享周期和预分频。
- 本项目阀类输出不需要每路独立 PWM 频率。
- 若只是开关阀，可以先固定 duty，使用 enable / disable 表达开关。

- 不要把这些输入做成通用 IO 配置页面；只按当前灌溉语义使用。

## 实现前状态

已经满足：

- `Stc8hBase` 已支持 UART1 P3.6/P3.7 引脚组选择。

可以直接进入项目实现：

- ESP32 RS485 基础口配置。
- ESP32 RTC / OTA / Web / FS / App Config / App Events 接入。
- STC 8 路 PWM 映射配置。
- STC RS485 方向控制宏与 `STC8H_UART1_PIN_GROUP=1`。
- STC ADC / GPIO / WDT 基础使用。

仍需硬件实测确认：

- RS485 DE/RE 方向切换时序。
- 分站 PWM 输出有效电平与驱动芯片输入逻辑。
- 拨码默认电平和地址位取反关系。
- 流量/低液位输入的有效电平、滤波和脉冲宽度。
- ADC 分压比例和阈值。
