# 06 Web 交互与 API 边界

## 交互原则

- 本项目第一版采用三入口模型：局域网 Web 完整控制台、手机通知与轻操作、主控 LCD2004 + 本地按键基础控制。
- Web 是完整配置、操作、维护、故障处理和记录查看入口。
- 手机通知与轻操作只承担通知、少量状态查看、暂停/恢复自动浇水；不承担完整远程控制台。
- 主控 LCD2004 + 本地按键承担现场基础查看和基础控制；复杂配置、计划编辑、校准、OTA 和完整诊断仍进入 Web。
- Web 不做通用硬件调试台。
- Web 不提供通用寄存器读写。
- Web 不提供任意 RS485 帧透传。
- Web 不提供通用 IO 映射 UI。

## 页面范围

| 页面 | 职责 |
|---|---|
| 首页 | 总览设备状态、当前运行、阻断事件、从站在线状态 |
| 运行 | 选择一个水源，在该水源内按顺序编辑水路和本次时长，启动或停止 |
| 设置 | 进入水源列表，并在当前水源上下文内进入水源本体配置、水路配置和校准 |
| 水源列表 | 选择或新增真实水源，查看在线、启用、拨码地址绑定、配置同步和装配完整性摘要 |
| 水源本体配置 | 管理水源名称、拨码地址绑定、启用状态、主阀、流量计、低液位、自吸泵、保护策略、分支水路摘要和配置同步状态 |
| 水路配置 | 管理当前水源下所有阀路的角色、名称、启用状态和标准流速状态 |
| 校准 | 在当前水源上下文内执行流量计校准和具体分支水路的标准流速学习 |
| 计划 | 自动计划、同站水路顺序、启用状态、跳过结果查看 |
| 记录 | 查看水路级浇水记录；手动、计划和维护作为触发来源字段 |
| 统计 | 查看用水、完成率、保护停机、水路用水和计量质量趋势 |
| 事件 | 查看告警、配置、维护、系统和 OTA 事件，处理阻断事件 |
| 天气 | 查看天气预报、天气数据状态、显示设置和人工判断建议 |
| 维护 | 从站识别、清除故障、重新同步配置、清空从站本地安全配置、从站 OTA，以及进入诊断详情 |
| 诊断详情 | 查看 RTC、RS485、从站诊断、供电质量、轮询细节和配置同步证据 |
| 手机轻入口 | 接收通知、查看少量关键状态、暂停/恢复自动浇水 |
| 主控 LCD2004 + 本地按键 | 查看现场基础状态、停止当前运行、手动启动一次浇水、暂停/恢复自动浇水、清除可恢复故障 |

## 首页

首页显示：

- 主控时间状态。
- RS485 总体状态。
- 当前运行中的从站和水路。
- 阻断事件数量。
- 最近一次故障摘要。
- 从站在线、离线、配置未同步、故障、锁定状态。
- 自动浇水暂停状态、暂停范围和恢复时间。
首页不直接显示 RTC 有效、RS485 正常、输入输出位图、固件版本等工程诊断字段；这些信息进入诊断详情。
首页不显示天气摘要、天气建议或天气设置；这些信息进入天气页面。
首页可以提供运行主入口；水源卡、水路卡和配置页面不提供启动灌溉入口。
首页可以提供暂停/恢复自动浇水入口；暂停只影响自动计划，不影响手动运行。

## 设置

设置显示：

- 水源列表：水源名称、启用状态、在线状态、拨码地址绑定状态、配置同步状态、可运行分支水路数量、装配完整性、最近必须处理的问题和编辑入口。
- 水源本体配置：基础信息、RS485 拨码地址绑定、主阀有无及阀路选择、流量计输入和校准状态、低液位输入、自吸泵输出和延时、保护策略、分支水路摘要、保存结果和同步状态。
- 水路配置：当前水源下所有阀路的分支水路状态；被主阀占用的阀路显示为主阀占用，不可编辑为分支水路，不能手动或计划运行，标准流速不适用。
- 校准：流量计校准和标准流速学习；流量计校准结果属于当前水源的流量计，标准流速学习结果属于具体分支水路。

规则：

- 设置围绕选中的真实水源组织，不把水源、水路、流量保护做成三个互不相干的后台模块。
- 设置只负责进入水源本体配置、水路配置、流量计校准和标准流速学习，不直接启动灌溉。
- 从站地址来自现场 4 位拨码，范围 `1..15`，地址 `0` 保留；Web 只表达“发现到的拨码地址绑定到哪个真实水源”，不能表现为直接修改从站地址。
- 主阀、流量计、低液位和自吸泵归属水源本体配置的硬件装配；被主阀占用的阀不能再作为分支水路。
- 第一版不暴露独立手动/计划运行权限；启用的分支水路默认可被手动运行和计划选择，实际能否启动由运行准入、锁定、阻断、配置同步和在线状态决定。
- 第一版不在水路配置中保存运行默认值；手动运行时长在运行入口临时填写，计划运行时长在计划表单中填写。
- 保护策略保留在水源本体配置；校准页不承载保护策略配置，不与 OTA、清空配置或清故障并列。
- 保存影响从站安全配置的内容后，必须显示待同步状态。
- 从站运行中可以保存主控侧配置，但不得立即下发安全配置。
- 自动计划在计划页管理，天气显示设置在天气页管理。
- 水源列表只显示摘要和操作入口，不直接显示完整编辑表单；新增、编辑或设置动作进入独立表单页或受控步骤页。
- 第一版不单独制作系统设置页；LCD2004、SHT30、RS485、记录和系统时间只作为状态或诊断信息展示。未确认的显示、记录、刷新选项不得在 Web 原型或 API 中凭空生成。

## 天气页面

天气页面显示：

- 位置名称。
- 当前天气、温度、湿度、风力。
- 今日降雨概率和预计降雨量。
- 未来 3 天天气趋势。
- 天气数据来源状态和更新时间。
- 天气显示设置摘要。
- 基于天气的人工判断建议。

规则：

- 天气不参与启动准入、停机、安全状态、计划跳过或运行时长调整。
- 天气建议不得作为禁止启动、跳过计划或停机原因。
- 天气建议可以提示用户考虑暂停自动浇水，但不得自动执行暂停。
- 天气数据不可用时只显示“天气暂不可用”或“建议暂停显示”，不生成阻断性告警。
- 天气页不直接显示设置表单；点击修改设置后进入独立天气设置表单页。
- 天气设置表单只保存位置名称、启用状态、刷新间隔、数据过期提示和建议显示开关。

## 诊断详情

诊断详情显示：

- RTC 状态。
- RS485 总体状态、波特率、轮询周期和最近通信错误。
- 从站在线状态、配置版本、固件版本。
- 输入位图、输出位图、脉冲计数、电压值。
- 电压状态只作为显示、告警和记录，不作为启动或停机条件。
- 主控最近启动时间、运行时长、最近一次断电恢复事件、估算离线持续时间、错过计划摘要、恢复后自动计划状态和上电通知投递状态。

规则：

- 诊断详情面向工程师和现场维护人员，不作为普通顶层入口。
- 诊断详情不提供通用寄存器读写、通用 IO 映射或任意 RS485 透传。
- 诊断详情只显示固件版本、最近 OTA 状态和诊断证据，不发起 OTA。

时间来源、时区、网络校时和基础 Web/API 访问能力属于 `Esp32Base` 基础能力。灌溉 Web 只显示系统时间有效状态，不提供时间来源或时区配置。

## 手动运行

Web 一次手动运行只绑定一个水源。该水源内可以选择多个分支水路，并按确认顺序逐路运行；不同水源之间可以分别发起手动运行，并在硬件允许时并行执行。

手动运行不保存为计划，不修改水路配置，不改变计划配置。每一路实际启动、跳过、失败或停止，都按水路级生成运行记录。

Web 手动顺序运行通过 `start_manual_sequence` 发起；主控 LCD2004 + 本地按键的单水路手动运行可以复用 `start_run`。

启动流程：

1. 选择水源；单水源模式默认使用当前水源。
2. 选择该水源下已启用且未锁定的分支水路。
3. 为每一路设置本次运行时长，并确认执行顺序。
4. 主控检查目标水源在线、配置同步、故障、锁定和已有活动运行状态。
5. 主控按顺序下发 `START_ZONE`。
6. 页面显示启动中、运行中、停止中和每一路结果。

规则：

- 其他从站运行中不阻止当前从站启动。
- 目标水源或从站已有活动运行时禁止启动。
- 已有活动运行的水源不能再发起第二条手动运行，但不阻止其它可运行水源发起自己的手动运行。
- Web 手动顺序运行执行中，当前水路故障停机、启动失败或状态无法确认时，后续水路不继续执行。
- 流量计计量未校准、标准流速未学习、电压检测未校准不阻断启动，但必须提示对应能力不可用。
- 停止后记录停止原因。
- 如果从站已因本地安全策略停机，页面显示从站上报的停止原因，不覆盖成用户停止。

## 水源本体配置中的从站信息

显示：

- 从站 ID。
- 拨码地址。
- 在线状态。
- 配置同步状态。
- 水源名称。
- 最近故障。
- 固件版本和硬件能力。

动作：

- 启用/禁用从站。
- 同步安全配置。
- 查看从站诊断。
- 识别从站、清除可恢复故障、清空从站本地安全配置和 OTA 进入维护页受控执行。

新水源/从站配置规则：

- 从站地址来自现场拨码，地址 `0` 不可用，疑似地址冲突必须先处理。
- 水源名称、从站显示名、备注和启用状态保存在主控侧。
- 主阀、流量计、低液位、自吸泵、分支水路、流量保护和阀门参数会影响从站安全配置，保存后必须进入待同步。
- 同步成功前相关水源禁止运行。

## 硬件装配边界

当前只在水源本体配置的硬件装配中覆盖：

- `IN1..IN4` 中 1 路流量计输入。
- `IN1..IN4` 中可选 1 路低液位输入。
- `IN1..IN4` 中未使用输入的 `disabled` 或 `reserved` 占位。
- 可选自吸泵干接点输出 `dry_contact_1`。
- `VALVE1..VALVE8` 中主阀占用阀号和剩余分支水路。
- 电压监测启用状态，以及启用时的告警阈值。

不覆盖：

- 通用输入输出配置页面。
- 主控板预留 ADC、脉冲输入、干接点、阀输出。
- 从站 H6 预留 I2C。
- 从站 H8/H9 预留 ADC。
- `SPARE_*` 引脚。

## 计划页面

计划配置包含：

- 计划名称。
- 启用状态。
- 周期规则：`cycle_length_days`、`cycle_anchor_date`、`active_day_mask`。
- 最多 4 个开始时间；每个开始时间包含启用状态和 `HH:MM`，未启用或为空的时间不触发。
- 目标从站和水路顺序。
- 每路运行时长。

运行规则：

- 计划创建和编辑只保存主控侧计划，不下发从站。
- 计划只能加入已启用且未被锁定或阻断的分支水路。
- 周期 UI 可以显示为“几天一周期 + 周期内第几天执行”，也可以把“每天”“每周固定日期”作为该模型的快捷表达，但保存模型仍按周期长度、起始日期和周期日掩码表达。
- 同一个计划在一个有效执行日内可以有最多 4 次触发，例如上午一次、下午一次；每次触发都执行同一组目标水源、水路顺序和每路固定时长。
- 同一从站同一时间只运行一路分支水路。
- 同一从站下按顺序运行多个水路。
- 不同从站代表不同真实水源，可以同时运行。
- 全局或按水源自动浇水暂停期间，受影响的自动计划项未启动即跳过，并记录暂停原因。
- 离线、未同步配置、故障锁定或同站冲突导致未启动的计划项直接跳过并记录原因。
- 不提供排队或隐藏延后策略。
- 删除计划必须二次确认，且不删除历史运行记录和事件。

计划结果：

- `completed`：全部应执行水路完成。
- `partial`：部分完成，部分跳过或失败。
- `skipped`：全部应执行水路都未启动即跳过。
- `failed`：至少尝试启动或运行，但没有水路成功完成。
- `cancelled`：用户取消。

## 记录页面

记录页面只显示水路级浇水记录。一条记录对应一个水源下某一路水路的一次运行或运行尝试。

记录字段：

- 开始时间、结束时间。
- 水源、水路。
- 触发方式：手动、计划或维护。
- 触发来源：Web 手动、本地 LCD 手动、计划名称或维护任务名称。
- 请求时长、实际时长。
- 结果。
- 停止原因。
- 脉冲数、估算水量、流量摘要。
- 本次启用和未启用的保护。
- 详情入口。

规则：

- 一个计划触发多个水路时，每个水路各生成一条记录。
- 未启动即跳过的计划项如果需要向用户解释，也按对应水路生成结果为“未启动/跳过”的记录，并写明原因。
- 记录详情必须保留从站上报的停止原因，不把保护停机覆盖成用户停止。
- 天气建议不得成为自动跳过原因。
- 自动浇水暂停可以成为自动计划未启动即跳过原因。
- 记录页不显示统计卡片。
- 记录筛选控件放在记录表格上方，筛选维度包括时间范围、触发方式和结果类型。

## 统计页面

统计页面显示：

- 近 30 天总水量、总浇水时长、日均水量和浇水最多水路；近 7 天只作为次级筛选。
- 按水源和水路平铺的每日趋势。
- 每个水路每天的估算水量和实际浇水时长。

统计页面只基于浇水记录展示正常浇水趋势，不参与自动控制。保护停机、阻断告警、未启动等低频异常进入记录和事件，不作为统计页核心信息。关键指标使用紧凑指标条，时间范围和水源筛选靠近统计内容。

## 事件页面

事件页面显示告警、配置、维护、系统和 OTA 事件。告警是事件类型，不是独立页面。

事件字段：

- 时间。
- 类型。
- 级别。
- 影响对象。
- 当前状态。
- 是否阻断。
- 用户可理解的摘要。
- 详情入口。
- 对阻断事件显示清除状态或进入处理页面的动作。

事件页面主区域显示事件列表。未清除阻断、配置待同步等需要处理的信息可以放在事件表格上方的紧凑提醒条中；最近 OTA 成功等不需要处理的信息不放入摘要。筛选控件必须贴近事件表格，不放到页面远端或独立弱关联区域。

事件详情必须显示触发条件、生成来源、阻断影响范围、处理建议、关联记录入口和清除状态动作。清除状态必须二次确认，且不删除事件记录、不自动启动计划。

阻断性告警事件必须在首页显示；清除状态前，受影响水源禁止手动运行，后续计划项跳过并记录原因。清除状态不删除事件记录。

告警不作为独立资源暴露；告警是 `Event` 的一种类型，阻断性告警通过事件状态表达和清除。

## 关键 API

API 使用 `Esp32Base` 当前支持的 query 参数风格。所有业务操作使用固定入口：

```text
/api/irrigation?op=<operation>&key=value
```

核心 `op`：

| `op` | 关键参数 | 用途 |
|---|---|---|
| `get_status` | 无 | 系统总览 |
| `get_weather` | 无 | 读取天气摘要、未来预报和建议 |
| `get_weather_settings` | 无 | 读取天气显示设置 |
| `save_weather_settings` | 天气显示字段 query 参数 | 保存天气显示设置 |
| `list_stations` | 无 | 从站列表 |
| `get_station` | `station_id` | 从站详情 |
| `save_station` | `station_id` 和从站字段 query 参数 | 保存从站业务配置 |
| `sync_station_config` | `station_id` | 同步从站安全配置 |
| `identify_station` | `station_id` | 识别从站 |
| `clear_station_fault` | `station_id,fault_mask,flags` | 清除可恢复故障 |
| `factory_reset_station_config` | `station_id,confirm` | 清空从站本地安全配置 |
| `get_zones` | `station_id` | 读取水路配置 |
| `save_zone` | `station_id,zone_index` 和水路字段 query 参数 | 保存单路水路配置 |
| `get_inputs` | `station_id` | 读取输入配置 |
| `save_input` | `station_id,input_index` 和输入字段 query 参数 | 保存单路输入配置 |

API 规则：

- 所有参数通过 URL query 传递，不使用 REST 路径参数、请求体 JSON、`PUT`、`DELETE` 或任意透传接口。
- 配置类保存操作必须执行与 `02-配置模型与参数边界.md` 一致的校验。
- `save_station` 和相关配置读取必须把拨码地址表达为绑定关系；Web/API 不提供修改从站现场拨码地址的能力。
- 第一版 `save_zone` 只保存水路角色、名称、启用状态和标准流速相关状态；不接收 `manual_allowed`、`schedule_allowed`、`default_duration_sec`。如后续需要区分手动/计划权限，必须回到总控重新裁决。
- 列表或复杂配置按单项保存，不要求一次请求提交完整大型 JSON。
- 涉及从站安全配置的修改必须把该从站标记为待同步；从站处于 `STARTING`、`RUNNING`、`STOPPING` 时，Web 可以保存主控配置，但不得立即下发到从站。
- 校准和标准流速学习通过受控业务 API 触发，不提供任意 RS485 帧透传。
- `factory_reset_station_config` 只清空从站本地安全配置，必须二次确认，不删除主控侧业务配置和历史记录。
- `stop_all` 对每个可能运行的从站发送保守停止请求，并按从站分别记录结果；不能把未确认停止的从站直接标记为已停止。
- `start_manual_sequence` 只接受同一 `station_id` 下的分支水路，不接受跨水源编排；`duration_secs=0` 的项不纳入执行。
- `pause_auto_watering` 只影响自动计划触发，不停止当前运行、不修改计划、不禁用计划。
- `save_schedule` 只保存主控侧计划，不下发从站；删除计划不删除历史运行记录和事件。
- `sync_station_config` 必须在下发后读取配置摘要确认版本和 CRC，一致后才解除待同步状态。
- `calibrate_flow` 和 `learn_baseline` 都是维护运行，必须走统一启动准入；失败、取消、保护停机或通信丢失时不覆盖原校准值或原标准流速。
- 从站 OTA 通过受控业务 API 触发，不提供 Web 任意上传后直透 RS485 帧的能力。
- 从站 OTA 只面向当前 `STC8H8K64U-45I-LQFP48` 从站固件；Web 不提供选择任意 STC8H 芯片或任意 bootloader 类型的入口。
- 主控必须在发起 OTA 前完成固件下载、签名/hash 校验和完整固件暂存。
- 发起从站 OTA 前必须确认从站空闲、进入维护状态、全部输出关闭；OTA 过程中 Web/API 必须拒绝新的运行命令。
- 从站 OTA 成功、失败、中断和重试必须生成事件记录。
- 通用 OTA 传输、写入、校验和恢复能力归属 `Stc8hBase`，业务 Web/API 只暴露受控产品入口和状态。
- 天气 API 只返回显示数据和建议，不返回自动跳过、自动停机、自动改时长或安全策略字段。

## 高影响动作

以下动作必须二次确认：

- 停止全部。
- 删除计划。
- 清除锁定故障。
- 清空从站本地安全配置。
- 覆盖从站安全配置。
- 发起从站 OTA。

## 手机通知与轻操作

手机通知与轻操作用于覆盖日常低频进入 Web 前的轻量场景。

第一版支持：

- 每次灌溉完成通知。
- 异常即时通知。
- 主控断电后重新上电或恢复在线通知；通知包含断电区间、估算持续时间、错过计划摘要和恢复后自动计划状态。
- 查看少量关键状态：当前是否运行、自动浇水是否暂停、最近一次灌溉结果、是否有阻断事件。
- 暂停自动浇水。
- 提前恢复自动浇水。

第一版不支持：

- 手机端手动启动灌溉。
- 手机端停止当前运行或停止全部。
- 手机端修改计划、配置、安全参数。
- 手机端清除故障、发起 OTA、清空从站配置。
- 完整 App、复杂云端后台、多用户权限体系。

通知通道的具体实现后续单独裁决；当前只定义产品边界。

## LCD 和本地按键

主控 LCD2004 显示：

- 当前时间。
- 主控和 RS485 状态。
- 当前运行摘要。
- 自动浇水暂停状态和恢复时间。
- 未处理故障数量。
- 最短故障提示。

主控按键支持：

- 查看有限状态信息。
- 必要时停止当前运行。
- 选择已启用水源和水路，发起一次手动灌溉。
- 启动前微调本次运行时长。
- 暂停/恢复自动浇水。
- 清除可恢复故障。

LCD 和主控按键不承担完整配置、计划编辑、流量校准、OTA、清空从站配置、完整诊断和记录查看。

主控 LCD2004 + 本地按键不作为从站识别的核心入口。从站识别主要从 Web 维护页发起，必要时由从站本地按键配合现场确认。

从站本地按键只用于维护识别和受控清除可恢复故障，不允许直接启动灌溉。
