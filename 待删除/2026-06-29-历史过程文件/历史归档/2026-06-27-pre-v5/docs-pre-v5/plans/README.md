## 使用原则

- 涉及硬件能力时，以 `pcb/` 下当前硬件文件为准；硬件文件缺失或状态异常时，先报告风险，不自行恢复、覆盖或推断。
- 涉及产品边界、配置模型、运行调度、安全状态、流量保护、RS485、Web/API、告警记录和验收规则时，以 `01..08` 为准。
- 涉及 Web、手机通知、本地 LCD 和本地按键交互时，以 `09-Web界面与交互原型.md` 为准。
- 涉及当前可点击 Web 原型时，只使用 `docs/prototypes/web/final-prototype.html`。
- 可点击原型不能引入本目录未确认的功能边界；原型变更后应同步检查 `09-Web界面与交互原型.md` 是否需要更新。

## 命名原则

- `从站` 是 STC8H 执行板的正式产品概念；`区域板`、`执行板` 只作为历史沟通别名，不作为字段、API 或页面正式命名。
- 一个从站对应一个真实水源；`水源名称` 是从站对应水源的用户显示名称，不是独立调度对象。
- `水路` 是用户可配置和可运行的阀路概念；协议和字段中使用 `zone` 表示同一概念。
- `分支水路` 是可直接运行的水路角色；`主阀` 是水源级硬件装配，占用一路阀输出但不作为可运行分支水路。

## 基础库边界

- ESP32 主控使用 `Esp32Base`。
- STC8H 从站使用 `/Users/tyg/dir/codex_dir/Stc8hBase`。
- 实现必须优先复用基础库已有能力。
- Web/API 风格必须匹配 `Esp32Base` 当前能力：固定路径 + `?key=value` query 参数；不采用 REST 路径参数、请求体 JSON、`PUT`、`DELETE` 或通用透传接口。
- 从站固件优先稳定、简洁、健壮，业务变化尽量由主控配置和运行命令吸收。

- 新增方案内容优先补充现有编号文档；只有跨多个文档且确实需要长期入口时，才更新本文档。
- 不再新增过程记录、提示词、复评日志或中间推进总结。
- 如果需要记录待验证问题，只在对应结果文档中写清楚不确定项、原因、验证步骤、通过标准和验证前保守策略。
- 原型确认前不编写代码实施计划；实施计划确认前不新增正式业务代码。
