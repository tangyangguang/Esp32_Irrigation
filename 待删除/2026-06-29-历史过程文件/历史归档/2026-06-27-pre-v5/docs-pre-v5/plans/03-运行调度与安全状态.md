# 03 运行调度与安全状态

## 核心运行模型

- 主控不是全局单任务系统。
- 多个从站代表多个独立水源，可以同时运行。
- 一个从站同一时间只能有一个活动运行。
- 一个从站内部不支持多水路并行。
- 从站运行只能由主控命令启动。
- 从站本地安全策略可以独立停机。
- 主控通信丢失时，不能假定从站已经停机。
- 任一从站故障默认只影响该从站，不自动停止其他从站。
- 全局急停是用户明确动作，不是普通故障的默认联动。

## 从站状态

| 状态 | 含义 | 输出要求 |
|---|---|---|
| `BOOTING` | 上电或复位初始化 | 全部输出关闭 |
| `UNCONFIGURED` | 无有效安全配置 | 全部输出关闭 |
| `IDLE` | 空闲可运行 | 全部输出关闭 |
| `STARTING` | 正在启动泵、主阀或分支阀 | 只允许按启动步骤打开输出 |
| `RUNNING` | 正在运行水路 | 仅保持本次运行相关输出 |
| `STOPPING` | 正在关闭输出 | 按停止步骤关闭输出 |
| `FAULT` | 可恢复或待处理故障 | 全部输出关闭 |
| `LOCKED` | 安全锁定 | 全部输出关闭 |
| `OFFLINE_LOCAL_SAFE` | 通信超时后本地安全停机 | 全部输出关闭 |

`STOPPING` 必须显示为正在停止，不能显示为已停止。

## 停止原因

停止原因只回答“本次从站运行为什么停止”。计划整体跳过、计划取消、用户确认告警等不作为从站停止原因。

| 原因 | 含义 | 停止后默认状态 |
|---|---|---|
| `DURATION_DONE` | 时长正常结束 | `IDLE` |
| `USER_STOP` | 用户或主控主动停止 | `IDLE` |
| `LOW_LEVEL` | 低液位触发 | `FAULT` 或 `LOCKED` |
| `NO_FLOW_START` | 启动后首次流量超时 | `LOCKED` |
| `NO_FLOW_RUNNING` | 稳定运行中流量中断 | `LOCKED` |
| `HIGH_FLOW` | 高流量保护 | `LOCKED` |
| `LOW_FLOW` | 低流量策略要求停止 | `FAULT` 或 `IDLE` |
| `COMM_TIMEOUT` | 通信超时，本地安全停机 | `OFFLINE_LOCAL_SAFE` |
| `RUN_TIMEOUT` | 超过最大运行时间 | `FAULT` |
| `CONFIG_INVALID` | 运行中配置异常或丢失 | `UNCONFIGURED` 或 `FAULT` |
| `WATCHDOG_RESET` | 看门狗复位导致停止 | 重新判定 |
| `LOCAL_FAULT` | 本地输出、存储或其他故障 | `FAULT` 或 `LOCKED` |
| `UNKNOWN` | 未知原因 | `FAULT` |

## 启动准入

禁止启动条件：

- 从站禁用。
- 从站离线。
- 疑似地址冲突或总线异常。
- 配置无效或未同步。
- 从站已有活动运行。
- 从站处于 `FAULT`、`LOCKED`、`UNCONFIGURED`、`OFFLINE_LOCAL_SAFE`。
- 目标水路禁用或锁定。
- 请求运行时长非法。
- 已配置低液位且当前确认触发。
- 存在未清除的启动无水、运行中无水故障或锁定。
- 已启用高流量保护并确认高流量。
- 已启用低流量保护，并确认达到停机策略。

允许启动但必须提示保护不完整：

- 标准流速未学习。
- 流量计计量校准未完成。
- 电压检测未校准。
- 主阀未配置。
- 低液位未配置。
- 自吸泵未配置。

## 运行对象

`RunSession` 至少包含：

```text
run_id
sequence_id
run_type
station_id
rs485_address
zone_index
requested_duration_sec
state
started_at
deadline_at
source
schedule_id
config_version
stop_reason
last_seen_at
had_lost_contact
```

`sequence_id` 可为空。Web 手动顺序运行会生成一个主控侧临时 `ManualRunSequence`，其中每一路实际执行仍分别生成独立 `RunSession` 和 `RunRecord`。

`RunSession` 状态：

| 状态 | 含义 |
|---|---|
| `pending` | 主控已创建，尚未发出启动 |
| `starting` | 已发启动命令，等待从站确认 |
| `running` | 从站确认运行 |
| `stopping` | 已发停止命令或等待从站关闭输出 |
| `stopped` | 已确认停止，正常结束 |
| `faulted` | 因故障或保护结束 |
| `lost_contact` | 主控无法确认从站状态 |
| `failed_to_start` | 启动请求被拒绝或启动失败 |
| `cancelled` | 启动前取消 |

## 从站本地安全保护

从站运行中必须独立处理：

- 低液位。
- 无水。
- 高流量。
- 通信超时。
- 运行超时。
- 配置失效。
- 本地输出异常。

规则：

- 安全保护触发后，从站先关闭输出，再上报停止原因。
- 主控必须接受从站上报的停止原因，不能覆盖成用户停止。
- 主控超时未收到停止确认时，进入 `lost_contact`，不能直接写成 `stopped`。
- 电压低/高告警不属于从站本地安全停机条件，只提示、记录和显示。

## 手动运行

手动运行分为两个入口：

1. Web 手动运行：一次只绑定一个水源/从站，可以在该水源内选择多个分支水路，主控按顺序逐路执行。
2. 主控 LCD2004 + 本地按键手动运行：一次只选择一个水源/从站和一个分支水路，启动前只微调本次时长。

Web 手动顺序运行使用主控侧临时 `ManualRunSequence` 表达，不保存为计划，不修改水路配置，不改变自动计划配置。

`ManualRunSequence` 至少包含：

```text
sequence_id
source
station_id
items(zone_index, requested_duration_sec, order_index)
state
created_at
started_at
ended_at
cancel_reason
```

规则：

- 手动运行可以由 Web 或主控 LCD2004 + 本地按键发起。
- 手动运行不能绕过流量计输入配置、故障、锁定、配置同步或地址冲突。
- 手动运行可以在其他从站运行时启动。
- 手动运行不能抢占同一从站正在执行的计划水路。
- Web 手动顺序运行中，同一从站仍然一次只打开一路分支水路。
- Web 手动顺序运行启动时，如果目标水源不可运行、已有活动运行，或任一待执行水路存在禁止启动条件，启动按钮保持禁用并显示原因。
- Web 手动顺序运行执行中，若当前水路故障停机、启动失败或状态无法确认，主控停止该序列，后续未启动水路记为取消或跳过，不继续尝试。
- 用户停止 Web 手动顺序运行时，主控停止当前水路，后续未启动水路取消；不能把从站上报的保护停机覆盖成用户停止。
- 主控 LCD2004 + 本地按键发起手动运行时，默认带入固定初始建议时长或上次本地手动使用时长，启动前允许本次时长微调；该微调不修改水路配置或计划配置。

## 自动计划

自动计划由主控执行，从站不保存计划。

计划定义模型：

- 一个计划由主控侧保存，至少包含计划名称、启用状态、周期规则、最多 4 个开始时间、目标水源与水路顺序、每路固定运行时长。
- 周期规则只使用一种底层模型：`cycle_length_days` 表示几天一周期，`cycle_anchor_date` 表示周期第 1 天对应的日期，`active_day_mask` 表示周期内第几天执行。
- 常见表达都由该周期模型换算：每天执行等价于 1 天周期且第 1 天执行；浇 2 天停 1 天等价于 3 天周期且第 1、2 天执行；每周固定日期等价于 7 天周期并选择对应周期日。
- 每个计划最多 4 个开始时间；每个开始时间由启用状态和 `HH:MM` 组成，未启用或为空的时间不触发。
- 同一个计划在一个有效执行日内，可以按已启用的多个开始时间触发多次，例如上午一次、下午一次；每次触发都按该计划内固定的水源、水路顺序和每路时长执行。
- 计划中的每一路水路使用计划内固定时长；不浇的水路不加入计划，或在界面上明确显示为本计划跳过，不作为运行队列的一部分。

规则：

- 一个计划可以包含多个从站。
- 同一从站下多个水路按配置顺序执行。
- 不同从站计划项可以并行执行。
- 自动计划不抢占手动运行。
- 自动计划不自动清除故障。
- 自动计划不自动修改配置。
- 自动计划不自动重启因故障停止的水路。

## 自动浇水暂停

自动浇水暂停是主控调度层能力，表示在指定范围和时间窗口内暂停自动计划触发；它不是运行中暂停再恢复。

第一版支持：

- 全局暂停：所有水源的自动计划都不启动。
- 按水源暂停：指定水源/从站的自动计划不启动，其他水源照常运行。
- 暂停到指定时间，到期自动恢复。
- 提前恢复。

规则：

- 暂停只影响自动计划，不影响 Web 或主控 LCD2004 + 本地按键发起的手动运行。
- 暂停不修改计划、不删除计划、不禁用计划。
- 暂停期间已经正在运行的水路不因暂停自动停止；停止当前运行仍走停止入口。
- 暂停期间本应执行的计划项未启动即跳过，并记录 `auto_watering_paused` 或对应水源暂停原因。
- 如果一个计划包含多个水源，未暂停的水源照常执行，被暂停的水源跳过；计划结果可为 `partial` 或 `skipped`。

第一版不支持：

- 按水路暂停。
- 按单个计划暂停。
- 暂停结束后补跑。
- 天气自动暂停。

暂停流程：

1. 用户从 Web、手机轻入口或主控 LCD2004 + 本地按键选择暂停范围：全局或指定水源。
2. 用户选择恢复时间；第一版不提供复杂重复暂停规则。
3. 主控保存暂停状态，生成 `auto_watering_paused` 事件，并触发通知。
4. 暂停期间，调度器在计划触发前先检查暂停范围；受影响计划项未启动即跳过，写入水路级 `RunRecord` 和计划跳过事件。
5. 暂停不会停止已经运行中的水路；停止运行必须走停止入口。

恢复流程：

1. 到达恢复时间时，主控自动恢复对应范围，生成 `auto_watering_resumed` 事件，并触发通知。
2. 用户也可以从 Web、手机轻入口或主控 LCD2004 + 本地按键提前恢复。
3. 恢复只影响后续计划触发，不补跑暂停期间跳过的计划项。

## 计划冲突

同一从站冲突时：

- 如果从站已有活动运行，后触发计划项跳过并记录 `station_busy`。
- 不做跨计划排队。
- 不做隐藏延后执行。
- 用户需要顺序运行时，应把水路合并到同一个计划。
- 同一个计划的多个开始时间彼此独立触发；如果上一开始时间触发的运行尚未结束，后一个开始时间仍按正常冲突规则评估，不能隐藏排队或自动延后。

多个计划同时触发且包含同一从站时：

1. 触发时间更早者优先。
2. 触发时间相同，`schedule_id` 更小者优先。
3. 其他涉及该从站的计划项跳过并记录冲突。

## 计划结果

| 结果 | 含义 |
|---|---|
| `completed` | 所有应执行水路都正常完成 |
| `partial` | 至少一路完成，且至少一路跳过或失败 |
| `skipped` | 全部应执行水路都未启动即跳过 |
| `failed` | 至少尝试启动或运行，但没有任何水路成功完成 |
| `cancelled` | 用户取消 |

规则：

- `skipped` 是准入或冲突结果，不是运行失败。
- `failed` 必须表示已经尝试启动或运行。
- 被跳过的水路不能生成从站停止原因，因为它没有启动。
- 跳过原因必须记录在对应水路级浇水记录中；计划整体触发、全部跳过或错过执行等解释性信息进入事件。

## 错过执行

- 主控断电、时间无效或系统刚启动导致计划错过时，不自动补跑。
- 主控恢复后记录错过事件。
- 下一次符合重复规则的时间再执行。

当前产品不提供补跑配置。

## 主控上电和断电恢复

主控重新上电或复位后，先进入安全状态，不自动启动任何水路，不补跑断电期间错过的计划。恢复后的自动计划只从下一次符合规则的触发时间继续。

第一版支持“断电恢复摘要”，但不声称能精确记录断电瞬间。没有备用电源或独立断电检测电路时，主控只能在断电前周期性保存最近确认在线时间，并在下次上电后推断断电发生区间。

恢复摘要需要回答：

- 大概什么时候断电：用 `last_alive_at` 到 `restored_at` 表示断电可能发生的区间。
- 大概断电多久：用 `restored_at - last_alive_at` 表示估算离线持续时间，并标注这是估算值。
- 是否耽误计划：检查断电区间内本应触发但没有运行记录的计划项。
- 来电后是否能继续自动运行：检查系统时间、自动浇水暂停、配置同步、从站在线和阻断状态。

记录规则：

- 主控周期性持久化 `last_alive_at`，表示主控最后一次确认自己在线且系统时间有效。
- 主控启动后记录本次启动时间、系统时间恢复时间、通知通道恢复时间、复位原因和是否属于预期维护重启。
- `restored_at` 表示主控本次重新具备有效时间和可记录能力的时间；如果通知通道稍后恢复，通知发送时间可以晚于 `restored_at`。
- 如果上次不是 OTA、用户维护重启或其他预期重启，生成 `power_restored` 系统事件。
- `power_restored` 事件必须包含最近确认在线时间、本次恢复在线时间、估算离线持续时间、复位原因、系统时间恢复时间和断电区间说明。
- 主控恢复后扫描 `last_alive_at..restored_at` 内的启用计划触发点；没有对应运行记录的触发点生成错过事件，原因指向断电恢复。
- 如果断电前存在未闭合的 `RunSession`，恢复后按状态无法确认结束并关联 `power_restored` 事件；不能写成正常完成，也不能自动继续该次运行。
- 如果上电时系统时间无效，先记录本地启动事实；等待时间恢复有效后再补齐可读时间、计算计划影响并发送通知。
- OTA、用户明确发起的维护重启或正常软件重启只记录启动事实，默认不作为断电恢复通知推送。

通知规则：

- 上电恢复通知在系统时间和通知通道可用后发送。
- 通知必须包含：断电可能发生区间、估算离线持续时间、错过计划摘要、恢复后自动计划状态。
- 通知文案不能表达为精确停电时刻，应表达为“主控已重新上电；上次确认在线为 ...，本次恢复在线为 ...，估算离线 ...”。
- 错过计划摘要至少显示错过数量；有错过时显示最早错过计划名称、时间、水源或水路摘要。
- 恢复后自动计划状态使用三类结论：`已恢复`、`仍暂停`、`暂不可用`。`暂不可用` 必须显示原因，例如系统时间无效、配置未同步、从站离线或存在阻断。
- 从站单独掉电或重启按从站离线、重新在线、通信恢复和电压记录处理；第一版不承诺逐从站精确断电通知。

## 主控本地 LCD2004 和按键

主控 LCD2004 + 本地按键是现场基础控制入口，由主控发起动作并统一下发业务命令。

允许能力：

- 查看当前时间、主控状态、RS485 摘要、当前运行、剩余时间、未处理故障数量和最短故障提示。
- 停止当前运行。
- 选择已启用水源和水路，发起一次手动运行。
- 在启动前微调本次运行时长。
- 暂停或恢复自动浇水。
- 清除可恢复故障。

限制：

- 不创建或修改计划。
- 不修改水源、水路、流量计、低液位、主阀、自吸泵或安全配置。
- 不执行流量校准、标准流速学习、OTA 或清空从站本地安全配置。
- 不绕过 Web/API 和主控统一启动准入。
- 不直接控制从站输出；所有运行仍由主控下发业务命令。

## 从站本地按键

从站默认使用 1 个维护/识别按键：

- 短按：进入从站识别模式，用状态灯辅助现场确认。
- 长按：进入维护确认流程，可用于清除可恢复故障或退出 `OFFLINE_LOCAL_SAFE`。
- 长按必须有确认动作，避免误触。
- 从站本地按键不能启动灌溉。
