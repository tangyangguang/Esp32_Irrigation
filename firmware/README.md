# ESP32 灌溉控制器固件

本目录是固件实现位置，代码只依据当前确认方案。

后续实现只依据：

- `../docs/当前方案/` 中确认的软件设计。
- `../pcb_irrigation/` 下的定稿 BOM 和网表。
- `../../Esp32Base` 已核实的公共能力。

当前已建立 PlatformIO 工程、业务配置 JSON 编解码与掉电安全保存、完整规则校验、定稿 PCB 引脚定义和硬件安全层。浇水控制已实现阀吸合与维持、泵启停时序、多区域顺序、用户停止、启动和运行中无流量保护、已学习区域的低/高流量处理，以及空闲时非计划流量报警。自动调度已实现三种总控状态、时间安全、到点执行和 NVS 重启防重复。流量计校准采用定点加权计算，区域流量学习使用五个稳定窗口并设十分钟上限。业务记录与事件已经接入 `Esp32Base`。尚未完成六个业务 Web 页面。启动时只有加载到有效配置或成功创建默认配置后才继续；存在配置文件但所有副本均无效时保持输出关闭，不用默认值覆盖。

常用验证命令：

```sh
pio test -e native
pio run -e esp32_irrigation
pio test -e esp32_record_test --upload-port <serial-port> --test-port <serial-port>
```

以上命令持续在 2026-07-14 验证。`esp32_record_test` 当前共 7 项，除记录 Store 和业务事件外，还验证调度状态在真实 NVS 中的往返保存和 CRC 损坏识别；测试结束删除专用 Store 和测试调度状态，不格式化文件系统。硬件输出、实阀、流量计、突然断电写入和 LittleFS 真实写失败仍需在完整 PCB 上验证。
