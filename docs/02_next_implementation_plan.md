# 下一阶段实施计划

当前工程已经完成基础环境：PlatformIO、`Esp32Base` 引用、最小 Web 页面、最小状态 API。下一阶段目标是建立“安全硬件闭环”，先让 1-2 路阀门控制和紧急停止可靠，再继续流量、浇花计划和交互。

## 阶段 A：安全硬件闭环

目标：任何时候都能确定阀门不会误开，红色停止键始终有效。

计划新增最少模块：

- `src/domain/ValveController.*`
- `src/domain/SafetyManager.*`
- `src/io/ButtonInput.*`

关键行为：

- `ValveController::begin()` 配置 2 路阀门 GPIO 为输出，并立即写关闭电平。
- `ValveController::allOff(reason)` 作为唯一全关入口。
- `SafetyManager::begin()` 调用阀门全关，记录启动安全状态。
- `STOP ALL`、`R1/UP`、`R2/DOWN` 采用轮询消抖；浇水中检测到短按后立即执行对应停止动作。
- `LOCK` 短按切换锁定状态；锁定后停止类按键仍有效。
- GPIO0/BOOT 长按 3 秒只进入恢复出厂确认流程，不直接执行恢复。
- 暂不接入 LCD、流量、定时任务，避免多个变量同时进入。

验收：

- 编译通过。
- 串口日志能看到安全初始化。
- 代码中没有任何绕过 `ValveController` 直接写阀门 GPIO 的业务逻辑。

## 阶段 B：最小手动浇水闭环

目标：通过 Web/API 启动一路或多路浇水，到时间自动关闭。

计划新增：

- `src/domain/WateringSession.*`
- 扩展 `IrrigationWeb` 的状态和手动控制 API。

关键行为：

- 手动浇水参数：第 1 路时长秒、第 2 路时长秒、执行模式；0 表示该路不参与。
- 单路时长范围：0 或 60-14400 秒。
- 执行模式只支持 `simultaneous` 和 `sequential`，默认 `simultaneous`。
- 至少一条已启用路的时长必须大于 0。
- 只允许已启用路；启用路配置未实现前默认只启用第 1 路。
- 会话运行期间 `handle()` 非阻塞推进执行模式：
  - `simultaneous`：参与路同时启动，按路独立到期关阀。
  - `sequential`：按路号从小到大启动，当前路完成后再启动下一路。
- 停止原因包含：完成、手动停止、紧急停止、参数错误。

验收：

- API 能启动第 1 路 60 秒浇水。
- 启用 2 路后，API 能启动第 1 路 60 秒、第 2 路 120 秒的同时浇水会话。
- 启用 2 路后，API 能启动第 1 路 60 秒、第 2 路 120 秒的顺序浇水会话。
- 同时模式每路到期自动独立关阀；顺序模式只在前一路完成后启动后一路。
- 紧急停止能打断会话。
- 运行中短按 `R1/UP` 只停止第 1 路，短按 `R2/DOWN` 只停止第 2 路，短按 `STOP ALL` 停止整个会话。

## 阶段 C：流量计数与水流异常

目标：接入 YF-S201 脉冲计数，并实现首个安全异常。

计划新增：

- `src/domain/FlowMeter.*`

关键行为：

- 2 路流量输入使用中断计数。
- ISR 只增加计数，不做日志、不做浮点、不做业务判断。
- loop 中计算脉冲增量和估算水量。
- 浇水启动后每路独立检测；默认 10 秒无脉冲，判定水流异常并关闭对应阀门。
- 水流异常检测窗口配置范围为 1-60 秒；1、10、60 为有效值，0、61 必须拒绝保存或回退默认并记录错误。

验收：

- 串口/API 可查看每路脉冲计数。
- 模拟无水流时，默认 10 秒后关闭对应阀门并记录状态。

## 阶段 D：配置、记录和浇花计划

目标：让设备从“可手动控制”进入“可日常使用”。

计划新增：

- `src/storage/SettingsStore.*`
- `src/storage/RecordStore.*`
- `src/storage/EventStore.*`
- `src/domain/WateringPlanScheduler.*`

关键行为：

- 配置使用 `Esp32BaseConfig` 的 NVS 能力；记录使用 `Esp32BaseFs` 文件能力，不在业务层直接访问 `LittleFS` 或 Arduino `File`。
- 所有重要参数都通过 SettingsStore 统一加载、校验、保存。
- 记录保存原始字段：时间戳、执行模式、每路目标/实际时长、原始脉冲数、校准系数快照、脉冲当量快照、估算水量、结束原因。
- 系统事件记录保存原始事件事实：启动、配置变更、计划变更、浇水启动/停止/异常、漏水告警、恢复出厂请求和执行。
- 记录和事件均采用定长二进制环形文件，分页读取通过 `Esp32BaseFs::readBytesAt()`，写入通过 `Esp32BaseFs::writeBytesAt()`。
- 浇花计划最多 8 条。
- 首版实现按星期、按间隔 N 天。
- 每条计划保存第 1 路时长、第 2 路时长和执行模式；0 表示该路不参与。
- 支持跳过今天和取消跳过今天。
- 提供记录和事件的 JSON 分页查询与 CSV 导出。

验收：

- 重启后配置保留。
- 手动浇水和浇花计划均写记录。
- 关键业务动作写系统事件。
- 浇花计划按分钟触发。
- `/` 进入灌溉业务首页，`/esp32base` 保留系统工具入口。

## 阶段 E：本地交互

目标：现场无需手机也能完成高频操作。

计划新增：

- `src/ui/Display1602.*`
- `src/ui/LedIndicator.*`
- `src/ui/LocalMenu.*`

关键行为：

- LCD 文案严格限制 2 行、每行 16 字符。
- 红色停止键仍由安全层优先处理。
- 使用 6 个面板按键：STOP ALL、R1/UP、R2/DOWN、START/OK、MENU/BACK、LOCK。
- 常规操作全部短按；只有 GPIO0/BOOT 恢复出厂使用长按 3 秒并进入确认流程。
- 本地菜单先支持待机、手动浇水、系统状态，再扩展配置和浇花计划。

验收：

- 待机状态显示时间和 WiFi 状态。
- 本地可启动一次手动浇水。
- 锁定状态下 STOP ALL、R1/UP、R2/DOWN 的停止功能仍有效。
