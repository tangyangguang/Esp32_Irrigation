# 当前实现与剩余验收计划

本文替代旧实现计划。当前代码已经采用固定 4 路、路级浇水任务、每路 6 条计划的模型。

## 已完成的软件闭环

- 固定 4 路水路，默认启用第 1、2 路。
- 阀门控制扩展到 4 路，并使用启动 100%、5 秒后 70% 的 PWM 策略。
- 流量计脉冲输入扩展到 4 路。
- 手动浇水和计划浇水统一为路级任务。
- 同一路互斥，不同路可并行。
- 每路最多 6 条计划，总计 24 个计划槽。
- 计划配置页面已完成；单次跳过 API 已完成。近期计划 Web 视图尚未实现，后续需确认页面边界后再落地。
- 历史记录按一路一次浇水任务保存。
- 计划触发结果按 `planId + 日期 + 分钟` 持久化保存，用于解释未形成浇水记录的原因并避免重启后重复处理。
- 浇水记录启动时从定长记录文件扫描重建元数据，降低掉电或 NVS meta 保存失败造成的记录不可见风险。
- Web 页面使用 Esp32Base 默认页脚系统导航，不覆盖基础库底部导航。

## 必须通过的软件验证

```bash
node scripts/check-web-structure.mjs
pio run
```

## 必须通过的实机验收

1. 上电后所有阀门关闭。
2. 四路 PWM 输出均能打开和关闭假负载。
3. 四路 PWM 在启动 5 秒后降到 70% 占空比。
4. 四路流量计输入能稳定计数，外部上拉可靠。
5. 首页手动启动每一路后，到时自动关闭并写入历史记录。
6. 同时启动多路时，各路独立倒计时和停止。
7. 单路停止只停止目标路，停止全部停止所有运行路。
8. 计划按分钟触发，目标水路忙碌或停用时保存计划触发结果。
9. 单次跳过后计划不启动，取消跳过后恢复等待。
10. 水流异常只关闭异常水路并写入异常结果。
11. 待机漏水检测触发后关闭全部阀门并显示当前告警。
12. GPIO0/BOOT 长按恢复出厂前先关闭运行水路。
13. 72 小时连续运行无异常重启、无阀门失控、无记录损坏。

## 不能用编译替代的项目

- 电磁阀实际吸合电压、电流和 70% PWM 保持是否可靠。
- MOSFET、继电器或驱动模块的温升和长期稳定性。
- 流量计脉冲抖动、线缆干扰和外部上拉阻值。
- 水路真实启动延迟与 10 秒无脉冲阈值是否匹配。
- 防水结构、按键误触和 GPIO0 启动模式风险。
