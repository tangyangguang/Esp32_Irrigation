# 09 通信同步与设备状态

## 基本原则

主控是 Web、计划、记录和调度中心。从站是固定水源侧执行单元。

从站不保存计划，不主动承担复杂业务逻辑。主控通过 RS485 轮询和命令控制从站。

## 设备状态

主控状态：

- 正常。
- 时间无效。
- 配置未完成。
- 存在异常。

从站状态：

- 在线。
- 离线。
- 运行中。
- 禁止启动。
- 配置不完整。

水源状态由从站状态和水源保护状态组合得到。

## 通信规则

主控定期轮询从站：

- 在线状态。
- 当前输出状态。
- 输入状态。
- 流量脉冲计数。
- 电压读数。
- 错误状态。

从站离线判定：

- 连续多次轮询失败后判定离线。
- 离线水源禁止启动新任务。
- 已运行任务遇到从站离线，主控标记异常停止，并在通信恢复后重新同步输出状态。

RS485 通信速率、轮询周期和离线失败次数属于系统参数，应在系统参数页的通信参数分组中正常展示和配置。设备状态展示区只需要表达在线、离线和禁止启动状态，不需要展示底层轮询细节。

## 命令边界

主控向从站下发业务命令：

- 打开或关闭指定阀。
- 打开或关闭水泵/干接点。
- 设置阀 PWM 参数。
- 查询输入、输出、流量、电压和状态。
- 固件升级相关命令后续单独设计。

当前不设计 Web 任意 RS485 透传。

## 同步规则

主控启动后：

- 扫描已配置从站地址。
- 读取从站状态。
- 同步输出状态。
- 如果发现输出状态和主控记录不一致，优先进入安全停止：关闭泵和阀，再重新进入空闲状态。

配置变更后：

- 主控保存配置。
- 相关水源空闲时再应用硬件映射变化。
- 运行中的水源禁止修改会影响输出和输入映射的配置。

## 安全边界

通信异常时不尝试继续执行计划。

- 离线水源跳过自动计划步骤。
- 离线水源不能手动启动。
- 离线恢复后不补跑错过的计划。

## 当前不设计

- 从站主动上报。
- 多主控。
- Web 任意 RS485 透传。
- 从站保存计划。
- 从站脚本规则。
- 通用寄存器调试页。
