# 点灯科技界面原型

## 文件

- `blinker-layouter-import.txt`：复制全文，粘贴到点灯科技 APP 的界面配置导入框。
- `blinker-layouter-prototype.json`：同一界面的可读 JSON，仅用于核对组件和后续固件开发。
- `blinker-layouter-seed.txt`：用户提供的原始示例导出，用于确认 Layouter 2.0 格式。

以上文件都不包含设备识别码、密钥或其他账号信息。

## 导入前

导入通常会替换当前设备的界面配置。虽然当前使用的是空闲设备，仍建议先保留现有导出文本，再导入原型。

## 原型结构

1. 顶部：在线状态、下次计划、今日执行、今日用水、当前任务、剩余时间、实时流量和紧急停止。
2. 手动浇水：可先套用 8 个计划之一；六个区域始终列出。点选区域后用快捷分钟数或加减 1 分钟调整，最后统一开始。
3. 浇水记录：按天前后切换，显示当日次数、用水量和最近三条执行摘要。
4. 自动计划：选择 8 个计划之一，修改启用状态、最多四个执行时间和六个区域时长，再统一保存。

## 当前边界

- 这是交互和排版原型，导入后只有静态示例数据；按钮动作和实时数据要等固件接入点灯科技后才会生效。
- 原型只使用用户示例中已经确认的 `btn`、`tex`、`num` 组件字段，没有猜测未公开的滑块或下拉框格式。
- 压缩文本已完成 Zipson 回读校验；点灯科技 APP 对组件数量、页面高度和实际渲染的限制仍需通过本次导入验证。
- 区域名称暂用“前院、后院、菜地、花坛、果树、备用”占位，确认交互后再与设备中的实际水路名称动态同步。

## 固件私有配置

`firmware/platformio.local.ini` 已预留以下本机字段，并继续由 Git 忽略：

```ini
[env:esp32_irrigation]
custom_blinker_device_id =
custom_blinker_auth_key =
```

只在本机填写真实值，不要把识别码或密钥写进仓库文件、界面导出或聊天记录。
