# 04 配置与参数边界

## 分层原则

配置只按作用域分层：

- 系统参数：影响全局。
- 水源配置：影响一个从站/水源。
- 水路配置：影响一条分支水路。

这是当前整理配置的起点，不是最终结论。如果后续发现有更合理的页面组织，可以调整。

不按“日常 / 高级 / 维护”分层。这个项目面向个人使用，只要参数真实影响行为，就应该分组清楚地展示。

## 系统参数

系统参数保留：

- 最大单次运行时长。
- 主阀和水泵启停延时默认值。
- 流量保护默认参数。
- RS485 通信速率。
- 从站离线判定参数。
- 通信异常判定参数。

默认值：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 最大单次运行时长 | 120 分钟 | 手动和计划都受限 |
| 主阀提前开启时间 | 1 秒 | 非负秒数 |
| 主阀延后关闭时间 | 1 秒 | 非负秒数 |
| 水泵/自吸泵启动延后时间 | 1 秒 | 非负秒数 |
| 水泵/自吸泵停止提前时间 | 1 秒 | 非负秒数 |
| RS485 通信速率 | 待实现默认 | 方案只要求可配置 |
| 从站离线判定 | 待实现默认 | 方案只要求可配置 |

建议压缩表达：

- 页面只说明参数含义、单位、默认值和影响范围。
- 不展示底层调试细节。
- 修改系统参数只影响新任务或后续判断，不改写已经展开的任务。

## 水源配置

水源配置保留：

- 水源名称。
- 启用/停用。
- 从站地址选择。
- 主阀是否启用。
- 主阀使用阀号。
- 水泵/自吸泵是否启用。
- 流量计是否启用。
- 流量计输入选择。
- 流量计每升脉冲数。
- 低液位是否启用。
- 低液位输入选择。
- 低液位常开/常闭、去抖时间。
- 主阀全功率启动时间。
- 主阀维持占空比。
- 分支水阀全功率启动时间。
- 分支水阀维持占空比。
- 电压读数显示。

默认值：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 水源启用 | 启用 | 停用只阻止新任务 |
| 主阀启用 | 不启用 | 启用后占用一个阀号 |
| 主阀阀号 | 阀 8 | 可改为阀 1..8 |
| 水泵/自吸泵启用 | 不启用 | 启用后使用从站干接点输出 |
| 流量计启用 | 启用 | 当前产品默认有流量计 |
| 流量计输入 | IN1 | 在当前从站 `IN1..IN4` 中选择 |
| 每升脉冲数 | 未设置 | 水源流量计参数，接水量校准后生成 |
| 低液位启用 | 不启用 | 低液位传感器不是必配 |
| 低液位输入 | IN2 | 启用低液位后选择，不能与流量计相同 |
| 低液位触点 | 常开 | 可改常闭 |
| 低液位去抖 | 1 秒 | 实现阶段可调整默认 |
| 主阀全功率启动时间 | 待实现默认 | 主阀打开时先 100% 全功率输出 |
| 主阀维持占空比 | 待实现默认 | 全功率时间结束后用该占空比维持主阀 |
| 分支水阀全功率启动时间 | 待实现默认 | 分支水阀打开时先 100% 全功率输出 |
| 分支水阀维持占空比 | 待实现默认 | 全功率时间结束后用该占空比维持分支水阀 |
| 电压读数 | 只显示 | 不告警、不保护、不停机 |

边界：

- 从站地址来自硬件拨码；Web 不修改硬件地址，只选择该水源绑定的从站地址。
- 电压只显示，不产生告警或保护。
- 输入只用于流量计和低液位，不做通用 IO。
- 流量计和低液位使用哪一路输入由对应水源/从站配置选择；同一水源内同一输入不能同时作为流量计和低液位。
- 干接点只作为水泵或自吸泵，不做通用继电器。

## 水路配置

水路配置保留：

- 水路名称。
- 启用/停用。
- 标准流量。
- 校准状态。

默认值：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 水路启用 | 启用 | 停用只阻止新任务 |
| 标准流量 | 未设置 | 稳定流量校准或历史采用后生成 |
| 最近手动时长 | 无 | 有历史后可复用 |

边界：

- 水路不直接选择阀号。
- 手动浇水时长不是水路配置。
- 计划浇水时长属于计划步骤。

## 运行中修改限制

建议规则：

- 水源运行中，不能修改该水源的主阀、水泵、流量计、低液位和 PWM 配置。
- 水源运行中，不能停用正在运行的水路。
- 停用水源或水路只阻止新任务，不停止当前任务。
- 停止任务必须使用明确的停止操作。

## 我的建议

- 系统参数页不应该变成“所有设置”。水源配置和水路配置必须回到对象详情里。
- 阀驱动参数必须收敛在水源配置里，不逐水路配置，也不放到运行页；主阀和分支水阀可以使用两组参数，所有分支水路共用同一组分支水阀参数。
- 通信参数可以展示，但不要做成调试台。用户只需要知道默认值、可选范围和修改影响。
- 输入选择应该放在水源配置里，因为一个水源对应一个独立从站板。它不是通用 IO 配置，只是在该从站的 4 路输入中选择流量计和低液位各用哪一路。
- 配置页面要按用户理解组织，不强行证明只有三层。系统、水源、水路只是当前最自然的三个归属。

## 后续细化

后续只需要补默认值和可选范围：

- 水源级主阀和分支水阀 PWM 参数。
- RS485 速率、轮询和离线判定参数。
- 流量保护参数的页面分组和说明文案。
