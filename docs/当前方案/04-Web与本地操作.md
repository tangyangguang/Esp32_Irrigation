# 04 Web 与本地操作

## Web 原则

Web 页面按用户任务组织，不按硬件接口组织。

页面必须回答：

- 当前是否正常。
- 现在能不能浇水。
- 自动计划如何安排。
- 6 个分区分别是什么状态。
- 为什么没启动、为什么停止。
- 实际浇水和异常记录是什么。

## 顶层页面

建议顶层页面：

1. 总览
2. 运行
3. 计划
4. 分区
5. 历史
6. 系统参数

不新增：

- RS485 页面。
- 分站页面。
- 通用诊断后台。
- 通用 IO 页面。
- 天气页面。
- 多用户权限页面。

## 总览

回答：现在设备是否正常。

显示：

- 当前时间。
- 设备状态。
- 当前运行任务。
- 今日浇水摘要。
- 当前异常和提示。
- 流量计状态。
- 低液位状态，仅在启用时显示。
- 自吸泵状态，仅在启用时显示。

不做：

- 完整配置表单。
- 计划编辑。
- 历史详情。

## 运行

回答：现在能不能浇水，如何手动启动或停止。

显示：

- 当前任务。
- 6 个分区可启动状态。
- 手动启动表单。
- 运行时长输入。
- 停止按钮。
- 不能启动原因。
- 实时流量、脉冲数和估算水量。

不做：

- 修改计划。
- 修改流量计参数。
- 修改低液位触点类型。

## 计划

回答：什么时候自动浇水，浇哪些分区，浇多久。

显示：

- 计划列表。
- 启用状态。
- 下一次触发时间。
- 分区步骤摘要。
- 新建和编辑计划。

不做：

- 任务排队。
- 补跑错过计划。
- 天气条件。
- 脚本触发。

## 分区

回答：6 个分区分别如何配置和校准。

显示：

- 分区列表。
- 分区名称。
- 启用状态。
- 最近浇水。
- 标准流量。
- 校准状态。
- 分区详情。
- 校准入口。

分区页不做普通手动浇水主入口，手动浇水放在运行页。

## 历史

回答：发生过什么。

显示：

- 浇水历史。
- 异常历史。
- 配置变化。
- 校准历史。
- 用户操作。
- 类型筛选。
- 详情查看。

浇水历史建议字段：

- 来源：手动 / 计划 / 校准。
- 分区。
- 开始时间。
- 结束时间。
- 计划时长。
- 实际时长。
- 结束原因。
- 脉冲数。
- 估算水量。
- 平均流量。
- 是否触发保护。

## 系统参数

回答：影响全局运行和保护的参数是什么。

显示：

- 阀驱动参数。
- 自吸泵启用和延时。
- 低液位启用、常开/常闭和去抖。
- 流量计每升脉冲数。
- 流量保护参数。
- 历史保留策略。
- OTA 和基础维护入口。

不做：

- 通用设置集合。
- 输入输出映射。
- 调试控制台。

## 本地按键

本地按键只提供最小现场能力。

建议能力：

- 查看当前状态。
- 停止当前任务。
- 简单选择分区并启动预设时长。

完整配置仍以 Web 为主。本地按键不承载计划编辑、校准参数录入或复杂系统设置。
