# 04 Web 与本地操作

## Web 原则

Web 页面按用户任务组织，不按硬件接口组织。

页面必须回答：

- 当前是否正常。
- 现在能不能浇水。
- 每天自动计划如何安排。
- 6 个 Zone 分别是什么状态。
- 为什么没启动、为什么停止。
- 实际浇水和异常记录是什么。

## 顶层页面

建议顶层页面：

1. 总览
2. 运行
3. 计划
4. Zone
5. 历史
6. 系统参数

不新增：

- RS485 页面。
- 分站页面。
- 通用诊断后台。
- 通用 IO 页面。
- 天气页面。
- 多用户权限页面。

## 总览

回答：现在设备是否正常。

显示：

- 当前时间。
- 设备状态。
- 当前运行任务。
- 今日浇水摘要。
- 当前异常和提示。
- 流量计状态。
- 自吸泵状态，仅在启用时显示。

不做：

- 完整配置表单。
- 计划编辑。
- 历史详情。

## 运行

回答：现在能不能浇水，如何手动启动一组顺序浇水批次或停止。

显示：

- 当前任务。
- 已启用 Zone 的可启动状态。
- 每个已启用 Zone 的手动时长输入。
- 一键启动顺序浇水。
- 选择已有自动计划并立即执行。
- 停止按钮。
- 不能启动原因。
- 实时流量、脉冲数和估算水量。

不做：

- 修改计划。
- 修改流量计参数。

## 计划

回答：每天什么时候自动浇水，浇哪些 Zone，浇多久。

显示：

- 计划列表。
- 启用状态。
- 每日启动时间，最多 4 个。
- Zone 时长摘要。
- 新建和编辑计划。
- 立即执行计划入口。

计划编辑只显示当前已启用 Zone。停用 Zone 不出现在计划时长表、计划摘要和立即执行步骤中。

不做：

- 任务排队。
- 补跑错过计划。
- 天气条件。
- 脚本触发。

## Zone

回答：6 个 Zone 分别如何配置和校准。

显示：

- Zone 列表。
- Zone 名称。
- 启用状态。
- 最近浇水。
- 标准流量。
- 校准状态。
- Zone 详情。
- 校准入口。

Zone 页不做普通手动浇水主入口，手动浇水放在运行页。

## 历史

回答：发生过什么。

显示：

- 浇水历史。
- 异常历史。
- 配置变化。
- 校准历史。
- 用户操作。
- 类型筛选。
- 详情查看。

浇水历史建议字段：

- 来源：手动 / 计划 / 校准。
- Zone。
- 开始时间。
- 结束时间。
- 计划时长。
- 实际时长。
- 结束原因。
- 脉冲数。
- 估算水量。
- 平均流量。
- 是否触发保护。

历史事件使用 `Esp32Base` App Events，项目保留最近 1000 条事件并环形覆盖。校准默认只写校准事件，不写普通浇水历史。

## 系统参数

回答：影响全局运行和保护的参数是什么。

显示：

- 阀驱动参数。
- 自吸泵启用和延时。
- 流量计每升脉冲数。
- 流量保护参数。
- 低流量动作和过流量动作。
- 历史事件保留策略。
- OTA 和基础维护入口。

不做：

- 通用设置集合。
- 输入输出映射。
- 调试控制台。

## 本地按键

本地按键只提供最小现场能力。

建议能力：

- 查看当前状态。
- 停止当前任务。
- 启动预设的顺序浇水批次。

完整配置仍以 Web 为主。本地按键不承载计划编辑、校准参数录入或复杂系统设置。
