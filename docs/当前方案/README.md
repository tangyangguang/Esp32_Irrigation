# 当前方案

本目录定义当前唯一有效的 12V DC 智能浇水控制器产品与软件方案，是后续实现和验收的依据。

## 当前产品边界

- 产品是一块本地运行的单板控制器，直接管理 6 个 12V DC 电磁阀区域；同一时刻最多打开一个区域。
- 默认启用区域 1、区域 2，区域 3 至区域 6 默认停用；用户可修改区域名称和启用状态。
- 流量计固定接入，所有出水过程都保留无流量保护；各区域独立管理用于流量异常判断的基准流量。
- 自吸泵功能可选，控制器只通过干接点信号控制外部继电器模块，不能确认泵是否真实启动。
- 最多保存 8 个每日浇水计划，每个计划最多设置 4 个启动时间；计划的自动执行开关不影响手动使用。
- 全部自动计划可统一暂停并定时或手动恢复；手动浇水不受该暂停状态影响。
- 阀默认全功率吸合 3 秒，随后以 20 kHz、75% 占空比维持，参数可在方案规定的范围内配置。
- WiFi、Web、认证、OTA、文件系统、系统日志、时间和 RTC 等公共能力使用同级目录 `../Esp32Base`；本项目只实现浇水业务。
- 浇水记录基于 `Esp32BaseRecordStore`，业务事件基于 `Esp32Base` App Events，本项目负责业务字段、规则和页面展示。

## 当前实现状态

截至 2026-07-24，固件已实现配置安全保存、硬件安全控制、浇水执行与保护、自动调度、流量计校准、区域基准流量管理、浇水记录、业务事件、在线检查点和业务 Web 页面。水路页最下方已提供“单次出水”，支持选择单条已启用水路后按秒级时长或 0.1 L 步进的目标估算水量执行；目标方式使用紧凑分段控件，时长使用单个复合输入，页面始终只显示当前模式的编辑器。可用现有水路基准做双向估算，结果写入正常浇水记录。单水路最长运行时间默认 120 分钟、可配置为 1～720 分钟，单次出水量上限默认 100 L、可配置为 1～1000 L。当前 Native 测试为 86/86 通过，设备事件测试固件和 ESP32 正式固件构建通过；仍待用户在实机验收单次出水停止精度、关阀尾水及桌面与窄屏页面。

仍需在完整 PCB 和真实水路上验证电磁阀、自吸泵、流量计、RTC、突然断电和 LittleFS 真实写失败等场景。LCD2004 和 4 个按钮已有硬件接口，但本地显示与按键交互尚未设计和实现，待 Web 功能稳定后另行确认。
