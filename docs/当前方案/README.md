# 当前方案

本目录定义当前 12V 智能浇水控制器的产品方案，是后续软件设计、代码实现和验收的依据。

硬件事实以以下文件为准：

- `pcb_irrigation/Netlist_Schematic1_2026-07-11.tel`
- `pcb_irrigation/BOM_Board1_Schematic1_2026-07-11.xlsx`

## 阅读顺序

1. `01-产品与硬件边界.md`：产品定位、硬件接口和固定参数。
2. `02-业务规则与安全.md`：浇水计划、手动操作、流量保护、校准、记录和事件。
3. `03-软件设计原则与边界.md`：下一阶段软件设计必须遵守的原则和边界。

## 核心结论

- 控制器直接管理 6 个浇水区域，同一时刻运行一个区域。
- 默认启用区域 1、区域 2；区域 3 至区域 6 默认停用。
- 流量计固定接入，所有出水过程都具备无流量保护。
- 可选自吸泵通过干接点信号控制外部继电器模块。
- 最多 8 个每日浇水计划，每个计划最多 4 个启动时间。
- 阀默认全功率吸合 3 秒，随后以 20 kHz、75% 占空比维持。
- 浇水记录由本项目保存；事件使用 `Esp32Base` 提供的事件能力。
- WiFi、Web、认证、OTA、文件系统、日志、时间和 RTC 使用 `Esp32Base`。

PCB 已引出 4 个按钮输入，并具备连接 LCD2004 所需的 I2C 扩展条件。本地显示和按键操作留到 Web 功能稳定后单独设计，当前方案只保留硬件事实和后续接入原则。
