# 05 任务、计划、校准和历史

## 任务

任务是一次水路运行。

任务状态：

- 等待。
- 运行中。
- 完成。
- 用户停止。
- 异常停止。
- 跳过。

边界：

- 真实启动过的普通浇水任务生成浇水类历史。
- 跳过、异常、配置变化和校准也进入历史，但用类型区分。
- 当前先不单独拆独立记录和独立事件，用统一历史降低页面复杂度。

## 手动浇水

手动浇水是立即启动一条水路任务。

启动时需要：

- 选择水源。
- 选择水路。
- 设置时长。

规则：

- 遵守所有保护规则。
- 不排队。
- 不抢占自动计划。
- 水源正在运行时不能启动同一水源另一条水路。
- 时长受最大单次运行时长限制。

## 自动计划

计划表达用户的自动浇水意图。

计划包含：

- 名称。
- 启用/停用。
- 周期规则。
- 开始时间。
- 水路步骤。

周期：

- 默认入口是每天。
- 自定义周期用于表达浇几天停几天、每周固定日等。
- 不引入 cron、脚本、天气条件或复杂触发器。

计划编辑字段：

| 字段 | 说明 |
| --- | --- |
| 名称 | 用户可编辑 |
| 启用状态 | 停用只阻止后续触发 |
| 周期类型 | 每天 / 自定义周期 |
| 开始时间 | 一个或多个时间 |
| 水路步骤 | 水源、水路、时长、启用状态 |

自定义周期底层模型：

- 周期长度：几天一个周期。
- 周期起始日期：哪一天算第 1 天。
- 执行日：周期内哪些天执行。

触发规则：

- 到时间后按水源拆分任务。
- 同一水源串行。
- 不同水源可并行。
- 水源不可用时跳过该水源本次步骤，并留下可追溯原因。
- 不补跑错过的计划。
- 修改计划只影响下一次触发。

计划运行展开：

- 系统按水源拆分步骤。
- 同一水源内按步骤顺序串行。
- 不同水源之间可并行。
- 某个水源不可用时，只跳过该水源相关步骤，不影响其它水源。

## 历史

历史统一保存可追溯事实。

历史类型：

- 浇水：真实运行过的自动或手动水路任务。
- 异常：启动失败、运行停止、低液位、无流量、低/过流量提示、通信异常等。
- 配置：系统参数、水源配置、水路配置变化。
- 校准：校准开始、完成、失败、取消和保存结果。
- 操作：用户停止、手动启动等需要追溯的操作。

浇水类历史建议字段：

- 来源。
- 水源和水路。
- 开始和结束时间。
- 计划时长和实际时长。
- 结束原因。
- 流量脉冲数。
- 估算水量。
- 稳定流速。
- 是否触发保护。

记录用途：

- 追溯实际浇水事实。
- 查看水量、时长和结束原因。
- 从正常记录采用稳定流速为参考流量。

异常类历史回答：

- 为什么没启动。
- 为什么停止。
- 哪个水源或水路异常。
- 是否需要用户处理。

影响级别如果需要，可以简单分为：

- 信息：只记录事实。
- 提示：需要注意。
- 异常：本次运行失败、停止或状态不正常。

不建议先做复杂告警确认、认领、清除流程。

## 校准

校准用于得到两个参数：

- 水路参考流量：用于低/过流量判断。
- 每升脉冲数：用于估算水量。

规则：

- 从水路详情进入，不做顶层页面。
- 遵守低液位、漏水、离线、忙碌等保护规则。
- 稳定流量校准：打开指定水路，读取流量计脉冲和流速，滑动窗口稳定后保存稳定流量为参考流量。
- 接水量校准：用户接一段固定水量，输入实际水量，系统用脉冲数换算每升脉冲数。
- 两种校准都可以保存结果，并进入历史。
- 校准不是复杂向导，只是清楚的工具流程。

稳定流量校准流程：

1. 用户从水路详情进入校准。
2. 系统检查水源在线、空闲、水路启用。
3. 系统启动该水路。
4. 过启动流量宽限时间后开始观察流速。
5. 滑动窗口稳定后，显示建议参考流量。
6. 用户保存或取消。
7. 系统停止水路并写入历史。

接水量校准流程：

1. 用户准备接水容器。
2. 系统启动该水路并统计脉冲。
3. 用户停止或到达指定时长后停止。
4. 用户输入实际接水量。
5. 系统计算每升脉冲数。
6. 用户保存或取消。
7. 系统写入历史。

## 我的建议

- 先用统一历史更简单。用户可以通过类型筛选浇水、异常、配置和校准。
- 计划不要做队列系统。跳过比排队更容易解释，也更适合本地灌溉控制器。
- 校准并不复杂，不应因为文档表达复杂就取消。它本质上只是打开水路、读取稳定流量或按接水量换算脉冲系数、保存参考参数。
- 校准页面不要做复杂向导，做成一个清楚的工具流程即可。
