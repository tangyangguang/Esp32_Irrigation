# 08 页面任务流与界面准备

## 目标

本文定义 Web 界面方案的内容边界、页面职责和任务流。

界面方案要验证：

- 6 个顶层页面是否够用。
- 用户能否完成手动浇水、计划、校准、配置和追溯。
- 每个页面是否只做自己该做的事。
- 有没有因为页面拆分或合并造成理解困难。

## 顶层页面

当前 6 个顶层页面：

1. 总览
2. 运行
3. 计划
4. 水源
5. 历史
6. 系统参数

不做顶层页面：

- 事件
- 记录
- 诊断
- 维护
- 统计
- 天气
- OTA
- LCD

这些内容如果要出现，必须挂在当前 6 个页面的合理位置。

## 总览

回答：系统现在是否正常。

显示：

- 主控时间和状态。
- 水源数量、在线数量、异常数量。
- 当前运行任务摘要。
- 当前需要注意的异常摘要。
- 今日浇水次数、总时长、估算水量。

允许：

- 查看摘要。
- 看到问题来源。

不允许：

- 直接编辑配置。
- 直接铺完整手动启动表单。
- 展示完整历史列表。

## 运行

回答：现在能不能浇水，怎么手动启动或停止。

显示：

- 当前运行任务。
- 各水源状态：空闲、运行中、离线、配置不完整、异常。
- 手动启动入口。
- 启动失败或不能启动原因。

手动启动流程：

1. 选择水源。
2. 选择该水源下启用的水路。
3. 设置运行时长。
4. 系统检查水源、水路、从站通信、低液位和必要配置。
5. 主控下发有时长的当前任务。
6. 从站本地计时并到点关闭输出。
7. 历史写入浇水或异常结果。

停止流程：

1. 用户选择当前运行任务。
2. 点击停止。
3. 从站按停止顺序关闭泵、分支阀和主阀。
4. 历史写入用户停止。

不允许：

- 编辑计划。
- 修改水源硬件配置。
- 修改系统参数。

## 计划

回答：什么时候自动浇水，浇哪些水路，浇多久。

显示：

- 计划列表。
- 启用状态。
- 下一次触发时间。
- 步骤摘要。

计划编辑流程：

1. 新建或编辑计划。
2. 填名称和启用状态。
3. 选择周期类型：每天或自定义周期。
4. 设置一个或多个开始时间。
5. 添加水路步骤：水源、水路、时长、启用状态。
6. 保存后只影响未来触发。

触发规则：

- 到时间后按水源拆分任务。
- 同一水源内步骤串行。
- 不同水源之间可以并行。
- 某个水源不可用时，只跳过该水源相关步骤。
- 错过计划不补跑。

不允许：

- 直接手动浇水。
- 展示完整历史。
- 修改系统参数。

## 水源

回答：有哪些水源和水路，它们是否可用。

显示：

- 水源列表。
- 从站地址、在线状态、运行状态。
- 主阀、水泵/自吸泵、流量计、低液位摘要。
- 水路列表。
- 电压读数。

水源详情：

- 基础信息。
- 水源当前状态。
- 水源配置入口。
- 水路列表。

水源配置：

- 水源名称。
- 启用状态。
- 主阀启用和阀号。
- 水泵/自吸泵启用。
- 流量计启用。
- 流量计输入选择。
- 流量计每升脉冲数。
- 低液位启用、常开/常闭、去抖。
- 低液位输入选择。
- 阀全功率启动时间。
- 阀维持占空比。

水路详情：

- 水路名称。
- 启用状态。
- 标准流量。
- 最近运行摘要。
- 校准入口。

不允许：

- 普通手动浇水。
- 展示完整历史。
- 把复杂配置直接铺在水源列表里。

## 历史

回答：实际发生过什么。

历史类型：

- 浇水
- 异常
- 配置
- 校准
- 操作

列表字段：

- 时间。
- 类型。
- 水源。
- 水路。
- 摘要。
- 结果。

筛选：

- 类型。
- 时间。
- 水源。
- 水路。
- 结果。

详情显示：

- 浇水详情：来源、开始结束、计划时长、实际时长、水量、稳定流速、结束原因。
- 异常详情：原因、影响对象、是否已恢复或是否仍需处理。
- 配置详情：修改对象、修改项。
- 校准详情：校准方式、脉冲数、稳定流量、每升脉冲数、是否保存。

允许：

- 从正常浇水历史采用稳定流速为水路标准流量。

不允许：

- 发起浇水。
- 复杂告警确认、认领、清除。

## 系统参数

回答：全局运行、通信和保护参数是什么。

分组：

- 运行：最大单次运行时长、主阀/水泵启停延时。
- 通信：RS485 速率、轮询、离线判定。
- 流量保护：无流量、待机漏水、低/过流量参数。

不允许：

- 水源级配置。
- 水路级配置。
- 通用 IO 配置。
- 任意 RS485 调试。

## 界面检查清单

检查页面时逐项确认：

- 对象列表页是否只展示列表、筛选、状态摘要和操作入口，没有把完整编辑表单直接铺在列表旁边。
- 总览能不能一眼看出是否正常。
- 运行页能不能清楚完成手动启动和停止。
- 计划页能不能表达每天和自定义周期。
- 水源页能不能解释从站、水源、水路关系。
- 校准入口是否自然，不像复杂向导。
- 历史页是否能同时承载浇水、异常、配置、校准和操作。
- 系统参数页是否只放全局参数，没有混入水源、水路配置和从站输入选择。
