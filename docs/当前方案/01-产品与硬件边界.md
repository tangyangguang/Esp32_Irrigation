# 01 产品与硬件边界

## 产品定位

当前产品是一块本地运行的单板 12V DC 智能浇水控制器：

- 直接控制 6 路 12V DC 电磁阀。
- 使用固定流量计检测出水、估算流量和用水量。
- 可通过干接点信号控制外部自吸泵继电器模块。
- 使用板载 RTC 完成每日计划调度。
- 通过 WiFi / Web 完成配置、操作、记录查看和设备维护。

## 电源与控制器

- 外部输入：12V DC。
- 阀和阀驱动供电：`VIN`。
- 板上电源：5V 和 3.3V。
- 控制模块：`ESP32-WROOM-32E-N4`。
- RTC：`DS3231SN#`，带 `BT1` 备用电池。
- 适用阀：12V DC 电磁阀。

## 6 个浇水区域

一个浇水区域对应一路物理电磁阀。界面统一显示“区域 1”至“区域 6”，需要与英文资料对照时使用 `Zone 1` 至 `Zone 6`。

| 区域 | 接口 | 阀网络 | ESP32 网络 | ESP32 模块引脚 | GPIO |
| --- | --- | --- | --- | --- | ---: |
| 区域 1 | `CN6` | `VALVE1-` | `PWM1` | `U1.9` | 33 |
| 区域 2 | `CN7` | `VALVE2-` | `PWM2` | `U1.8` | 32 |
| 区域 3 | `CN8` | `VALVE3-` | `PWM3` | `U1.11` | 26 |
| 区域 4 | `CN9` | `VALVE4-` | `PWM4` | `U1.10` | 25 |
| 区域 5 | `CN10` | `VALVE5-` | `PWM5` | `U1.13` | 14 |
| 区域 6 | `CN11` | `VALVE6-` | `PWM6` | `U1.12` | 27 |

固定规则：

- 区域与阀输出一一对应。
- 同一时刻最多打开一个区域。
- 默认启用区域 1、区域 2。
- 区域 3 至区域 6 默认停用，用户接线后再启用。
- 每条水路可以改名，并可手工设置或通过流量学习保存独立基准流量。

## 阀驱动

- 全局驱动关断：`DRV_SD`，`U1.31 / GPIO19`。
- 高电平关断全部阀驱动，低电平允许驱动。
- 板上 4.7k 上拉保证 ESP32 引脚未驱动或复位时默认关断。
- 上电、复位和没有活动阀时必须保持驱动关断。
- 开阀先使用 100% 输出吸合，再切换为 PWM 维持。
- 默认吸合时间：3 秒。
- 默认维持占空比：75%。
- 默认 PWM 频率：20 kHz。
- 吸合时间、维持占空比和 PWM 频率是全局阀驱动配置，不逐区域配置。
- PWM 频率允许在 1 kHz 至 25 kHz 范围内配置；修改后从下一次出水开始生效，不在阀已打开时切换频率。
- PWM 分辨率由软件根据 ESP32 定时器能力统一选择，不作为用户配置。
- 20 kHz 和 75% 是当前保守起始值。实阀测试后可以调整，但必须同时验证可靠吸合、可靠保持、线圈与驱动温升、工作电流、噪声和低水压下的实际出水。

## 流量计与预留输入

### 必配流量计

- 接口：`CN2`。
- 外部信号：`PUL_SIG`。
- ESP32 信号：`PULSE_IN1`。
- ESP32 模块引脚：`U1.28 / GPIO17`。
- 用途：无流量保护、流速估算、用水量估算、基准流量判断和校准。

普通浇水使用当前稳态流量系数估算流量和水量。流量计校准直接使用原始脉冲并重新计算系数；无论系数取值如何，所有出水过程都保留无流量保护。

### 预留信号输入

- 接口：`CN4`。
- 外部信号：`PUL_SIG2`。
- ESP32 信号：`PULSE_IN2`。
- ESP32 模块引脚：`U1.27 / GPIO16`。

该输入是 PCB 预留接口，当前没有对应的软件需求。

## 自吸泵控制信号

- 接口：`CN5`。
- 干接点网络：`RELAY1A / RELAY1B`。
- 控制网络：`DRY_OUT1`。
- ESP32 模块引脚：`U1.30 / GPIO18`。
- 器件：`AQY210EHAX`。

边界：

- 该输出只控制外部自吸泵继电器模块的信号输入。
- 定稿网表中 `DRY_OUT1` 低电平使 PhotoMOS 输入导通，高电平关闭；板上 47k 上拉保证 ESP32 引脚未驱动时默认关闭。
- 不直接驱动泵，也不直接驱动继电器线圈。
- 系统只能显示“泵控制信号已输出/未输出”，不能声称泵已真实启动，因为当前没有泵反馈输入。
- 自吸泵功能未启用时，页面不显示泵控制状态。

## RTC、I2C、按钮和维护接口

| 接口 | 定稿硬件用途 |
| --- | --- |
| `CN1` | 3.3V I2C：`I2C_SDA / I2C_SCL / 3.3V / GND` |
| `CN3` | 5V I2C：`SDA2 / SCL2 / 5V / GND`，经 `PCA9306` 电平转换 |
| `H1` | 4 个外接按钮信号和 GND |
| `H2` | 3.3V 串口维护口：`RX / TX / 3.3V / GND` |

H1 按钮网络：

| 按钮 | H1 | ESP32 模块引脚 | GPIO |
| --- | --- | --- | ---: |
| OK | `H1.2 / BTN_OK` | `U1.4` | 36 |
| ESC | `H1.3 / BTN_ESC` | `U1.5` | 39 |
| UP | `H1.4 / BTN_UP` | `U1.6` | 34 |
| DOWN | `H1.5 / BTN_DOWN` | `U1.7` | 35 |

当前阶段只确认 4 个按钮和 I2C 扩展接口存在。本地菜单、按键细节、LCD2004 接口、显示内容和完整交互流程留到 Web 功能稳定后单独设计，不进入当前软件实现。

以下内容只作为后续设计输入，避免需求遗失，不代表本阶段已经完成交互设计：

- LCD2004 显示当前运行状态。
- 空闲时可以选择一个已经在 Web 中配置好的浇水计划并手动启动；本地界面不编辑计划。
- `ESC` 同时作为停止入口。浇水过程中按一次进入停止确认，按 `OK` 后停止当前整次浇水；未在确认时限内确认则自动返回运行界面。
- 停止确认时限预期可配置，当前设想默认 30 秒，其最终取值和配置位置留到本地交互设计时确认。

未来本地操作必须与 Web 共用同一套状态和业务命令，不能形成第二套浇水逻辑。

## 安全硬件不变量

任何软件状态都必须满足：

- 固件启动后的第一项硬件动作是进入安全关闭状态。
- 配置加载前和系统初始化期间，所有输出保持关闭。
- 没有活动区域时，6 路 PWM 为关闭状态且 `DRV_SD` 为关断。
- 自吸泵信号默认关闭。
- 切换区域必须先完成当前区域的安全关闭，再启动下一区域。
- 用户停止、保护停止、配置错误、重启和 OTA 都不能留下阀或泵信号。
- 重启后不恢复上次未完成任务。
