# 13 工程开工执行清单

## 目标

本文是正式写代码前的执行清单。

它不重新设计架构，只把已经收口的结论变成开工时必须遵守的边界：

- 用基础库已有能力。
- 工程结构简单。
- 先打通主控和分站的真实通信闭环。
- 不做临时寄存器补丁。
- 不做大型测试矩阵。

## 基础库前置版本

开工前按以下基础库能力为准：

| 基础库 | 前置提交 | 必须使用的能力 |
| --- | --- | --- |
| `Esp32Base` | `83bebb8 Add lightweight RS485 port helper` | FULL profile、OTA、RTC、FS、App Config、App Events、RS485 port |
| `Stc8hBase` | `bf072c8 Add UART1 pin group selection` | UART1 pin group、RS485 UART driver、PWM pin_select、GPIO、ADC、Timer、WDT、CRC |

如果实际工程环境里的基础库早于这些提交，先更新基础库，不在本项目里绕过。

## 工程目录

项目内只建两个固件工程：

```text
master-esp32/
slave-stc8h/
```

不建：

- `common/` 大共享库。
- 多固件 monorepo 框架。
- 协议代码生成器。
- 通用设备平台。

主控和分站可以各自有一份同名协议常量文件。第一阶段允许少量重复，优先保证代码直观可读。

## 主控开工配置

主控 `platformio.ini` 必须启用完整基础环境：

```ini
ESP32BASE_PROFILE=ESP32BASE_PROFILE_FULL
ESP32BASE_ENABLE_RTC=1
ESP32BASE_RTC_DRIVER=ESP32BASE_RTC_DRIVER_DS3231
ESP32BASE_ENABLE_RS485_PORT=1
ESP32BASE_ENABLE_APP_CONFIG=1
ESP32BASE_ENABLE_APP_EVENTS=1
```

主控板级引脚：

```text
RS485_RX  = GPIO16
RS485_TX  = GPIO17
RS485_DIR = GPIO13
I2C_SDA   = GPIO21
I2C_SCL   = GPIO22
```

主控第一轮只需要初始化这些能力：

- `Esp32Base::begin()` / `Esp32Base::handle()`。
- RTC 路径存在。
- OTA 由 `Esp32Base` 提供。
- `Esp32BaseRs485Port` 初始化成功。
- 业务 Web 可以显示通信状态。

不要在第一轮实现：

- 自动计划。
- 校准流程。
- 完整历史文件。
- 复杂页面。
- 本地按键和 LCD。

## 分站开工配置

分站 `platformio.ini` 必须启用当前板子需要的基础库能力：

```ini
STC8H_UART1_PIN_GROUP=1
STC8H_UART_ENABLE_RX=1
STC8H_PWM_GROUP_MASK=0x03
STC8H_PWM_A_CHANNEL_MASK=0x0F
STC8H_PWM_B_CHANNEL_MASK=0x0F
STC8H_PWM_ENABLE_DISABLE=1
```

是否启用 duty clamp / channel check 先保持基础库默认，等通信和输出闭环稳定后再考虑裁剪。

分站 RS485：

```text
UART1 RX = P3.6 / RxD_2
UART1 TX = P3.7 / TxD_2
DIR      = P4.1
```

分站 PWM 映射：

```text
业务输出 1 -> PWM1 P2.0 -> STC8H_PWM_PIN_PWM1_P20
业务输出 2 -> PWM2 P2.2 -> STC8H_PWM_PIN_PWM2_P22
业务输出 3 -> PWM3 P2.4 -> STC8H_PWM_PIN_PWM3_P24
业务输出 4 -> PWM4 P2.6 -> STC8H_PWM_PIN_PWM4_P26
业务输出 5 -> PWM5 P0.0 -> STC8H_PWM_PIN_PWM5_P00
业务输出 6 -> PWM6 P0.1 -> STC8H_PWM_PIN_PWM6_P01
业务输出 7 -> PWM7 P0.2 -> STC8H_PWM_PIN_PWM7_P02
业务输出 8 -> PWM8 P0.3 -> STC8H_PWM_PIN_PWM8_P03
```

分站第一轮必须保证：

- 上电先全关输出。
- RS485 默认接收态。
- 看门狗启用。
- 1ms tick 或等价 soft timer 可用。
- 8 路 PWM 初始化后保持关闭。

## 第一轮通信闭环

第一轮只实现三个命令：

```text
PING
GET_INFO
GET_STATUS
```

验收标准：

- 主控能轮询地址 `1..15`。
- 无响应地址显示离线或未发现。
- 有响应分站显示地址、能力、在线状态。
- 分站坏帧不复位、不误动作。
- 分站收到非本机地址不响应。
- 分站收到合法请求后能返回同 `seq` 响应。
- 任意通信异常都不能打开输出。

不做：

- `START_TASK`
- `STOP_TASK`
- 自动调度。
- 保护停机。
- 流量统计。
- 分站 OTA。

原因不是拆产品版本，而是先验证硬件链路、基础库配置和帧解析正确。

## 第二步任务闭环

第一轮通信通过后，再补当前任务闭环：

```text
START_TASK
STOP_TASK
GET_LAST_RESULT
GET_INPUTS
```

此时才接入：

- 当前任务状态机。
- 单路阀打开/关闭。
- 主阀/泵延时。
- 低液位输入。
- 无流量宽限和确认。
- 最近任务结果。

仍然不接入：

- 自动计划。
- 校准。
- 长期历史。
- 完整 Web 页面。

这些功能等“当前任务从启动到停止”稳定后再接入。

## 文件边界

主控：

```text
BoardPins.h
  只放主控固定引脚和硬件常量

Rs485Protocol
  只做帧 encode / decode / crc

Rs485Master
  只做请求队列、超时、重试、轮询

RuntimeController
  只做业务状态和启动/停止决策

WebPages / WebApi
  只读运行状态，通过 RuntimeController 发起动作
```

分站：

```text
board_pins.h
  只放当前板子的端口、bit、PWM pin_select、RS485 DIR 宏

irrigation_frame
  只做帧解析和组包

command_handler
  只把命令转成 task_runner / status_reporter 调用

output_driver
  只控制输出安全态、PWM enable/disable/duty

task_runner
  只维护当前任务

input_monitor
  只采样输入，不决定业务动作
```

不要把协议解析、输出控制和任务状态机写在同一个文件里。

## 最小验证

每一轮只做小验证：

- 主控工程能编译。
- 分站工程能编译。
- 协议 encode / decode 的固定样例能通过。
- 分站上电输出保持关闭。
- 主控能发现真实分站或明确显示未发现。
- 拔掉 RS485 或分站断电时，主控状态能变为离线。

不做大型测试矩阵。

## 开工顺序

建议按这个顺序开工：

1. 建 `master-esp32/` 和 `slave-stc8h/` 空工程。
2. 配好基础库路径和编译宏，先编译空主循环。
3. 分站先实现板级安全态和 WDT。
4. 双端实现帧结构、CRC、PING。
5. 加 `GET_INFO` 和 `GET_STATUS`。
6. 主控页面只显示发现结果和在线状态。
7. 确认真实硬件链路后，再进入当前任务闭环。

如果某一步发现基础库能力缺口，暂停本项目实现，回基础库补齐。
