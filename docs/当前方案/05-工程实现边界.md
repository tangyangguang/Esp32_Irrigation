# 05 工程实现边界

## 实现原则

当前工程只实现这块 12V 单板控制器需要的功能。

不做通用平台，不做历史 RS485 架构兼容层。

ESP32 固件必须先基于 `../Esp32Base` 搭建基础环境，再在本项目内实现智能浇水业务。基础库能力优先级高于本项目自写框架。

## 基础库使用边界

固件基础环境使用 `Esp32Base` 的完整配置：

- 使用 `ESP32BASE_PROFILE_FULL`。
- 启用 Web、WiFi、认证、OTA、Web OTA、LittleFS、系统日志、健康信息、看门狗、DNS、mDNS、NTP 和时间能力。
- 显式启用 RTC，当前按板载 DS3231 和 I2C `GPIO21/GPIO22` 接入。
- 显式启用 App Config，业务配置通过基础库配置能力管理。
- 显式启用 App Events，浇水、停止、异常、校准等业务事件写入基础库事件日志。
- 固件分区使用 `Esp32Base` 提供的 OTA 分区表。

项目业务代码只实现：

- 板级引脚语义。
- 阀、流量计、低液位、自吸泵的硬件抽象。
- 浇水批次状态机。
- 每日计划调度。
- 手动顺序浇水和选择计划立即执行。
- 校准、历史和面向用户的业务 Web 页面。

如果基础库已有能力不满足项目需要，应优先修改和验证 `../Esp32Base`，再在本项目中使用；不要在本项目里复制基础库能力或写局部旁路补丁。

注意：虽然 `Esp32Base` 可提供更多通用能力，但当前产品已明确取消 RS485、主从站和分站概念，本项目不启用 RS485 相关能力。

## 固件范围

固件需要实现：

- ESP32 基础启动。
- WiFi / Web。
- RTC 时间。
- 配置存储。
- 历史存储。
- 6 路阀输出控制。
- 阀全功率吸合和 PWM 维持。
- 流量计脉冲计数。
- 可选低液位输入处理。
- 可选自吸泵干接点输出。
- 任务状态机。
- 每天自动计划调度，单个计划最多 4 个每日启动时间。
- 手动顺序浇水批次。
- 选择计划立即执行。
- 校准流程。
- OTA。

固件不实现：

- RS485。
- 分站协议。
- 从站发现。
- 多设备同步。
- 多水源并行调度。
- 输入输出任意映射。
- 主阀。
- 自定义周期计划。
- 每周固定日计划。

## 引脚语义

| 业务语义 | 硬件网络 | ESP32 模块引脚 | 说明 |
| --- | --- | --- | --- |
| Zone 1 | `PWM1` | `U1.13 / GPIO14` | 阀 1 控制 |
| Zone 2 | `PWM2` | `U1.12 / GPIO27` | 阀 2 控制 |
| Zone 3 | `PWM3` | `U1.11 / GPIO26` | 阀 3 控制 |
| Zone 4 | `PWM4` | `U1.10 / GPIO25` | 阀 4 控制 |
| Zone 5 | `PWM5` | `U1.9 / GPIO33` | 阀 5 控制 |
| Zone 6 | `PWM6` | `U1.8 / GPIO32` | 阀 6 控制 |
| 驱动关断 | `DRV_SD` | `U1.31 / GPIO19` | 高电平关断，低电平允许输出 |
| 流量计 | `PULSE_IN1` | `U1.28 / GPIO17` | 必配脉冲输入 |
| 低液位 | `LOW_IN2` | `U1.27 / GPIO16` | 可选干接点输入 |
| 自吸泵控制 | `DRY_OUT1` | `U1.30 / GPIO18` | 可选干接点输出 |
| RTC 中断 | `IO4_INT` | `U1.26 / GPIO4` | RTC INT/SQW，当前软件不使用 |
| I2C SDA | `I2C_SDA` | `U1.33 / GPIO21` | RTC 和 I2C 扩展 |
| I2C SCL | `I2C_SCL` | `U1.36 / GPIO22` | RTC 和 I2C 扩展 |
| 串口 RX | `RX` | `U1.34 / U0RXD` | 维护口 |
| 串口 TX | `TX` | `U1.35 / U0TXD` | 维护口 |

## 安全态

上电和复位阶段必须保证：

- 阀驱动默认关闭。
- 6 路阀输出关闭。
- 自吸泵干接点输出关闭。
- 配置加载失败时不启动任何任务。
- OTA、重启、异常恢复后不自动恢复运行中的任务。

系统重启后，如果发现上次有未完成任务，应记录为异常结束，而不是继续浇水。

## 浇水批次状态机

浇水批次状态至少包含：

- 空闲。
- 启动检查。
- 开阀。
- 等待自吸泵。
- 运行中。
- 当前 Zone 结束并进入下一 Zone。
- 正在停止。
- 完成。
- 用户停止。
- 异常停止。
- 跳过。

状态机必须非阻塞，不能用长时间 delay 阻塞 Web、计时、流量计数或安全停机。

RTC 时间由 `Esp32BaseTime` 和 `Esp32BaseRtc` 提供。本项目不新增 RTC 驱动或 RTC 状态机，也不使用 `RTC INT/SQW` 做计划触发。

## 存储边界

建议存储分组：

- 系统参数。
- 供水配置。
- Zone 配置。
- 计划配置。
- 历史索引和历史记录。
- 校准过程临时数据。

不要把所有配置打成一个不可维护的大 blob。

## 验收闭环

硬件级：

- 上电所有阀关闭。
- 驱动使能逻辑正确。
- 6 路阀可单独打开和关闭。
- 流量计脉冲可计数。
- 低液位常开/常闭配置有效。
- 自吸泵干接点输出只在启用且任务运行时动作。

业务级：

- 手动设置已启用 Zone 的时长，按顺序启动并逐路到时关闭。
- 自动计划每天到点启动，并按步骤逐个 Zone 到时关闭。
- 选择自动计划立即执行，不改变该计划下一次自动触发。
- 无流量触发停机。
- 自吸泵和低液位都启用时，低液位启动前阻止、运行中停机。
- 自吸泵禁用时，低液位不影响浇水流程。
- 自吸泵禁用时流程不包含泵动作。
- 自吸泵启用时按延时输出控制信号。
- 历史能记录浇水、停止、异常和校准。
