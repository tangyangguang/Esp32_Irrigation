一、当前核心结论

已确定

主控站是系统大脑

ESP32 主控负责 Web/API、计划、记录、天气、调度、权限、事件和所有区域从站的统一编排。

已确定

区域从站是一块区域执行单元

一块区域从站对应一个供水区域，默认包含 1 个流量计、8 路电磁阀、若干输入和干接点输出。

已确定

全系统第一阶段只运行一路

即使未来有多个区域，第一阶段仍保持同一时刻只开启 1 个水路，避免调度、故障和记录复杂化。

三、核心对象关系

推荐用户心智

  • 一块区域从站就是一个供水区域。
  • 这个区域有一根被独立计量的供水管。
  • 这个区域的所有阀门默认都归这个流量计管理。
  • 用户只需要理解“区域”和“水路”，不需要逐个绑定流量计。

内部软件关系

  • System 包含一个主控本地区域和多个 RS485 区域从站。
  • Station 有且只有一个 Flow Meter。
  • Station 有多个 Zone，每个 Zone 对应一路电磁阀输出。
  • Zone 的流量、流速、异常判断都来自所属 Station 的 Flow Meter。
关键收敛：不把“一块从站两个流量计、阀门再手工分配归属”作为标准模型。这样能避免用户在安装和配置时面对复杂的归属关系。

四、主控站和区域从站的边界

模块 应该负责 不应该负责
主控站 ESP32 Web/API、自动计划、手动浇水、记录和事件、天气判断、所有 Station 调度、统一配置、权限和同源检查。 不直接承担远端阀门的实时电气保护；不把每个远端流量脉冲都通过 RS485 实时接收。
主控本地区域 保留最小独立运行能力：本地流量计、少量阀门、少量传感器输入、少量干接点输出。 不作为所有复杂场景的唯一扩展方式；复杂现场交给区域从站。
区域从站 STC8H8K64U 本区域 8 路阀门、1 个流量计本地脉冲计数、本区域传感器、本区域干接点输出、状态上报、通信超时关阀。 不做 Web、不做计划、不自行决定浇水策略、不做跨区域调度。
传感器输入从站 未来可作为专门扩展：集中采集雨量、土壤湿度开关、外部暂停、门磁、压力开关等。 不是第一阶段核心；不能把普通输入直接当作高频流量脉冲输入。

五、运行和安全规则

运行并发

第一阶段全系统同一时刻只运行 1 个 Zone。同一区域内永远不并发。跨区域并发属于未来能力，不进入第一阶段。

流量计计量

一个流量计代表一条被独立计量的供水管段。它不是一个电磁阀，也不是一个单独 Zone。

低液位和无水

低液位用于启动前保护；无水由流量计运行中兜底。低液位触发时不启动泵，运行中触发时停泵关阀。

远端阀门保护

区域从站上电默认全部关阀。主控失联、RS485 超时、命令过期或从站复位时，从站必须本地关闭输出。

流量计远端采集

流量计接在区域从站时，从站本地计数脉冲，主控读取累计脉冲和窗口脉冲，不通过 RS485 逐个转发脉冲。

停止逻辑

页面仍然只设计“停止当前运行”，不设计 Stop All、暂停/恢复、多个 Zone 同时运行或按水量停止。

六、硬件接口结论

主控板已确定方向

  • ESP32E + PCF8574T。
  • 三灯：电源、工作状态、故障。
  • 四键：上、下、确认、返回。
  • 恢复出厂键：GPIO0，上电后长按 10 秒触发。
  • I2C 屏幕预留：OLED 或 1602，注意 5V 屏幕电平转换和地址冲突。
  • UART/RS485 扩展：用于区域从站通信。

输入输出已确定方向

  • 干湿兼容输入：三针端子 5V / GND / IN。
  • 干接点输入：只接 GND + IN，并启用上拉。
  • 5V 有源输入：接 5V + GND + IN，并关闭上拉。
  • 流量计是脉冲计数输入，不能只当普通干湿输入看待。
  • PhotoMOS 干接点输出对外只标 COM / NO，不提供电源。
  • PCF8574 控制 PhotoMOS 时采用低电平有效，下拉吸电流驱动。
PCF8574 适合状态灯、低速输入、辅助干接点输出；不适合 UART、流量脉冲计数、阀门 PWM 或严格实时的保护输入。

灌溉从站状态灯

  • PWR：电源灯，接从站逻辑电源，不占 MCU IO。
  • COM：通信灯，显示 RS485 收发和在线状态。
  • RUN：运行灯，本从站有阀门、主阀或泵输出动作时点亮。
  • ALM：故障灯，低液位、通信超时关阀、输出异常等需要处理时点亮或闪烁。

灌溉从站识别按钮

  • 从站保留 1 个按钮，只做识别功能。
  • 按下后，从站状态灯按识别灯序闪烁，主站页面高亮对应从站。
  • 用于现场确认地址、命名和接线，不参与日常浇水操作。
  • 不做本地停止、急停、阀门测试、恢复默认、地址设置或本地计划功能。

每路阀门通道灯

  • 每个电磁阀输出建议有独立通道灯，不额外占 MCU IO。
  • 通道灯表示“控制器正在命令这一路打开”。
  • PWM 保持阶段灯保持可见点亮或略暗，不需要单独处理。
  • 如果某一路被设为主阀，它的通道灯仍按该输出动作状态显示。
通道灯位置 结论 原因
逻辑控制侧 推荐 不接入 12V 阀门负载回路，不受线圈反冲、续流和钳位影响；LED 故障也不影响电磁阀供电回路。它显示控制命令状态。
MOS / 负载侧 不作为当前首选 更接近输出导通状态，但会接入 12V 侧，可能引入漏电路径和瞬态应力；如果需要真实线圈诊断，应另做电流检测。

七、Web 原型后续调整方向

总览页

从“固定 6 路”改为显示当前系统、区域列表、当前运行区域、当前水路、流量趋势、天气影响和需要处理的问题。

区域管理

新增区域从站/本地区域概念。用户先管理区域，再在区域下管理水路、流量计和传感器。

水路管理

水路不再固定 1..6，而是属于某个区域；页面默认只显示已启用水路。

流量校准

按区域的流量计校准。支持复制同型号流量计的 K-factor，但每个流量计保存独立参数。

计划管理

计划仍按时间运行，水量只用于观察、记录和异常判断。第一阶段计划执行仍全局串行。

设置页

系统设置保留全局策略；区域/水路相关配置不要堆到系统设置里，避免页面过重。

八、暂不做和待确认

暂不做
  • 不做按水量浇水。
  • 不做多个 Zone 同时运行。
  • 不做 Stop All、软件急停、暂停/恢复。
  • 不把天气、雨量、土壤湿度显示成故障红灯。
  • 不把 ADC 模拟量作为第一阶段核心能力。
  • 不让区域从站自己决定计划和调度。
待确认
  • 区域从站阀门数是否最终固定为 8 路。
  • 主控本地最小阀门数是否固定为 2 路。
  • 区域从站传感器输入和干接点输出的具体数量。
  • RS485 协议：地址、发现、配对、CRC、续期、超时和状态回读。
  • Web 原型从固定 Zone 模型迁移到区域模型的具体页面结构。
  • 未来是否开放跨区域并发，第一阶段先不开放。

九、命名建议

内部概念 用户页面建议名称 说明
Master 主控站 负责系统级管理和所有区域调度。
Station 区域 一块区域从站或主控本地区域，对应一个供水区域。
Flow Meter 流量计 计量某个区域的供水管段。
Zone 水路 用户实际理解的一路浇水分支，对应一个电磁阀。
Dry Output 干接点输出 只提供触点闭合/断开，不提供外部电源。
