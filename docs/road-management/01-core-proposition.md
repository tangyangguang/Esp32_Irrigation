# Zone 自治模型：单路自治，N 倍扩展

> Current status: historical background only. Current implementation must follow the 2 Flow / 6 Zone redesign in `docs/superpowers/specs/2026-06-05-two-flow-six-zone-irrigation-design.md`.

> 核心命题：每个 Zone 是一个完整的自治单元，多路系统 = 单 Zone 逻辑 × N 的实例化。

## 命题

每个 Zone（水路）是一个完整的自治单元（Zone Unit），拥有闭环的管理、计划与执行能力。

多 Zone 系统（2 路、4 路、未来更多路）是在单个 Zone 逻辑之上乘以 N 份实例，管理模式完全一致。扩充只需增加实例数量，不改业务逻辑。

> Zone = 水路。灌溉行业标准术语（Rain Bird、Hunter、Orbit 等品牌统一使用）。

## 结构示意

```
┌─────────────────────────────────────┐
│           ZoneManager               │
│  职责：初始化 N 份 Zone 实例，       │
│       分发全局事件（停机/告警/恢复）  │
└──────────┬──────────────────────────┘
           │ 实例化 N 份
    ┌──────┴──────┐
    ▼             ▼
┌────────┐   ┌────────┐   ┌────────┐
│ Zone 1 │   │ Zone 2 │   │ Zone N │
│        │   │        │   │        │
│  ├─计划 │   │  ├─计划 │   │  ├─计划 │
│  ├─执行 │   │  ├─执行 │   │  ├─执行 │
│  ├─状态 │   │  ├─状态 │   │  ├─状态 │
│  ├─阀门 │   │  ├─阀门 │   │  ├─阀门 │
│  ├─流量 │   │  ├─流量 │   │  ├─流量 │
│  ├─异常 │   │  ├─异常 │   │  ├─异常 │
│  └─记录 │   │  └─记录 │   │  └─记录 │
└────────┘   └────────┘   └────────┘
```

## 每个 Zone 单元的完整闭环

```
Zone Unit
├── 配置    启用/停用、名称、引脚绑定、流量参数、超时阈值、错误抑制
├── 计划    独立的计划槽集合，独立调度
├── 执行    独立状态机（停用→空闲→启动中→运行中→完成/停止/异常→空闲）
├── 阀门    独立 PWM 开/关/占空比切换
├── 流量    独立脉冲计数、水量估算、流速计算
├── 异常    独立水流异常检测
└── 记录    独立浇水记录和事件日志
```

## 设计准则

1. **单 Zone 自洽**：任何 Zone 可以脱离其他 Zone 独立存在并完整工作
2. **无跨 Zone 耦合**：Zone 的状态变化、计划触发、异常处理不依赖其他 Zone 的状态
3. **N 倍可扩展**：增加第 5 路、第 8 路时，只需扩展配置和数组容量，不改业务逻辑
4. **操作一致性**：手动启动、停止、计划触发等所有操作对每个 Zone 的判断流程完全相同

## 系统层 vs Zone 级层

系统层负责跨 Zone 的全局操作，不属于任何单一 Zone：

| 系统层职责 | Zone 级职责 |
| --- | --- |
| 初始化 N 份 Zone 实例 | 每 Zone 的完整生命周期管理 |
| 全局停机（stopAll） | 独立开/关阀、计时、流量统计 |
| 待机漏水监控（全局前提判断） | 运行中水流异常检测（Zone 级） |
| 恢复出厂流程 | 配置持久化、计划持久化 |
| 应用事件日志 | Zone 级业务事件记录 |
