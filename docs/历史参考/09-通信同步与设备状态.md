# 09 通信同步与设备状态

## 基本原则

主控是 Web、计划、记录和调度中心。从站是固定水源侧执行单元。

从站不保存计划，不主动承担复杂业务逻辑。主控通过 RS485 轮询和命令控制从站。

从站也不是简单 IO 扩展。运行中的从站必须具备本地安全闭环：当通信失联或主控保持信号超时，从站应进入安全停机状态，关闭泵和阀。

设计取舍：

- 主控负责计划、调度、记录、配置和 Web 表达。
- 从站负责水源侧实时输入输出和本地安全停机。
- 从站不做独立计划、脚本、复杂条件触发或历史记录保存。
- 从站固件应简单可靠，类似遥控设备失联后的安全状态逻辑：信号丢失时进入明确的安全输出状态。

## 设备状态

主控状态：

- 正常。
- 时间无效。
- 配置未完成。
- 存在异常。

从站状态：

- 通信状态：在线、离线。
- 运行状态：空闲、运行中、安全停机待同步。
- 可启动状态：可启动、禁止启动。
- 配置状态：已配置、配置不完整。

水源状态由从站状态和水源保护状态组合得到。

这些状态不是一个互斥枚举。Web 表达时应按维度展示，避免把“在线但配置不完整”“在线但禁止启动”等组合压成一个含混状态。

## 地址和发现模型

从站地址来自 4 位拨码硬件，Web 不修改从站地址。

当前地址方案：

- 有效从站地址使用 `1..15`。
- 地址 `0` 作为未配置或保留地址，不作为正常水源使用。
- 一个有效地址对应一个从站，一个从站对应一个水源。
- 主控按有效地址范围扫描 RS485 总线，发现在线从站。
- 已绑定地址恢复在线时，回到对应水源。
- 未绑定地址被发现时，在 Web 中显示为“未配置从站”，用户确认后创建或绑定为一个水源。

地址冲突处理：

- 如果多个从站使用同一拨码地址，RS485 通信结果可能不稳定，不能通过 Web 远程消除冲突。
- 主控只能把该地址标记为疑似地址冲突或通信异常。
- 用户必须到现场修改从站拨码，使地址唯一。

配置不完整的判定：

- 地址为 `0` 或无效地址。
- 已发现从站尚未绑定为水源。
- 水源配置存在输入冲突，例如流量计和低液位选择了同一输入。
- 主阀配置导致没有可用分支水路。
- 水源被启用但没有任何启用的分支水路。

配置不完整的水源不允许启动自动或手动任务，但可以在水源页继续显示和配置。

## 通信规则

主控定期轮询从站：

- 在线状态。
- 当前输出状态。
- 输入状态。
- 流量脉冲计数。
- 电压读数。
- 错误状态。

从站离线判定：

- 连续多次轮询失败后判定离线。
- 离线水源禁止启动新任务。
- 已运行任务遇到从站离线，主控标记为通信失联异常，不假定输出已经按主控记录停止。
- 从站本地超过运行通信安全超时时间后，应自行执行安全停机。
- 通信恢复后，主控重新同步输出状态和安全停机结果，再补全任务记录和事件。

RS485 通信速率、轮询周期、离线失败次数和运行通信安全超时规则属于系统通信参数，应在系统参数页的通信参数分组中正常展示或说明。设备状态展示区只需要表达在线、离线、禁止启动和安全停机待同步状态，不需要展示底层轮询细节。

## 命令边界

主控向从站下发业务命令：

- 打开或关闭指定阀。
- 打开或关闭水泵/干接点。
- 设置阀 PWM 参数。
- 查询输入、输出、流量、电压和状态。
- 固件升级相关命令后续单独设计。

当前不设计 Web 任意 RS485 透传。

## 同步规则

主控启动后：

- 扫描有效地址范围内的在线从站，并匹配已有水源配置。
- 标识未绑定从站和疑似地址冲突。
- 读取从站状态。
- 同步输出状态。
- 如果发现输出状态和主控记录不一致，优先进入安全停止：关闭泵和阀，再重新进入空闲状态。

配置变更后：

- 主控保存配置。
- 相关水源空闲时再应用硬件映射变化。
- 运行中的水源禁止修改会影响输出和输入映射的配置。

## 安全边界

通信异常时不尝试继续执行计划。

- 离线水源跳过自动计划步骤。
- 离线水源不能手动启动。
- 离线恢复后不补跑错过的计划。
- 运行中通信失联时，从站本地安全停机是必须能力；主控恢复通信前只显示异常和待同步状态。

## 当前不设计

- 从站主动上报。
- 多主控。
- Web 任意 RS485 透传。
- 从站保存计划。
- 从站脚本规则。
- 通用寄存器调试页。
