# 03 运行状态与保护规则

## 并行边界

- 一个从站对应一个水源。
- 同一个从站同一时刻只允许 1 个分支水路运行。
- 同一个从站的流量计归属于当前运行的分支水路。
- 多个从站代表多个水源，不同从站之间可以并行运行。

这个边界的目的很明确：同一水源串行运行，流量计才能把流量和水量归因到具体分支水路。

## 启停顺序

不建议水泵早于分支阀启动。推荐默认顺序是先打开水路，再启动水泵，停止时先停水泵，再关闭水路。

### 启动

1. 检查低液位、漏水、从站在线和水路可用状态。
2. 打开主阀；未启用主阀则跳过。
3. 等待主阀提前开启时间。
4. 打开分支阀。
5. 等待水泵启动延后时间；未启用水泵则跳过。
6. 启动水泵。
7. 进入流量检测宽限时间。

### 停止

1. 停止水泵；未启用水泵则跳过。
2. 等待水泵停止提前时间。
3. 关闭分支阀。
4. 等待主阀延后关闭时间。
5. 关闭主阀；未启用主阀则跳过。

## 启停时间配置

启停时间使用非负秒数，不使用正负数表达提前或延后，避免误配。

- 主阀提前开启时间：默认 1 秒。
- 主阀延后关闭时间：默认 1 秒。
- 水泵启动延后时间：默认 1 秒。
- 水泵停止提前时间：默认 1 秒。

这些参数属于系统参数。它们只改变动作之间的时间差，不改变默认启停顺序。

## 低液位规则

- 启动前低液位有效：禁止自动和手动启动。
- 运行中低液位有效：立即停止水泵，随后关闭分支阀和主阀。
- 因低液位停止的任务不自动恢复。
- 低液位恢复稳定后，允许再次启动新的手动或自动任务。
- 手动浇水同样遵守低液位规则。

## 流量保护

流量保护参数属于系统参数，不放到水源或水路配置里。低流量和过流量的比例是系统级参数；参考流量按水路保存，可由水路校准生成，也可从正常浇水记录中的稳定流速采用。没有参考流量的水路，不检查低流量和过流量。

默认参数：

| 配置项 | 默认值 | 用途 |
| --- | ---: | --- |
| 启动流量宽限时间 | 10 秒 | 分支阀和水泵启动后，等待管路建立流量 |
| 无流量确认时间 | 10 秒 | 宽限结束后仍无脉冲，判定运行无流量 |
| 漏水检测窗口 | 30 秒 | 待机状态下统计流量脉冲 |
| 漏水判定脉冲数 | 3 个脉冲 | 窗口内达到该脉冲数，判定疑似漏水 |
| 低/过流量持续时间 | 20 秒 | 异常比例持续达到该时间才触发 |
| 低流量比例 | 30% | 低于参考流量的 30% 判定低流量 |
| 过流量比例 | 200% | 高于参考流量的 200% 判定过流量 |
| 低/过流量默认动作 | 仅提示 | 可改为停机 |

### 待机漏水

当水泵、主阀和所有分支阀都处于关闭状态，但流量计持续检测到流量时，判定为疑似漏水。

处理规则：

- 保持所有输出关闭。
- 当前水源标记为漏水异常。
- 漏水异常存在时禁止启动该水源的新任务。
- 流量消失并稳定后解除禁止启动。

### 运行无流量

分支水路启动后，先进入启动流量宽限时间。宽限结束后，如果继续在无流量确认时间内没有有效脉冲，判定为无流量异常。

处理规则：

- 停止当前分支水路。
- 停止水泵并关闭相关阀。
- 记录本次运行异常结束。
- 手动浇水同样遵守无流量保护。

### 低流量和过流量

低流量和过流量保留为可配置保护，但不作为默认强制停机逻辑。

- 参考流量来自水路校准或正常浇水记录中的稳定流速，不是用户日常填写的保护参数。
- 水路没有参考流量时，不检查低流量和过流量。
- 水路有参考流量时，按系统参数中的比例和持续时间检查。
- 触发后可以选择仅提示或停机。
- 默认动作为仅提示。
- 不设计累计水量异常保护。

## 手动浇水

手动浇水只改变启动来源，不绕过系统保护。

- 低液位有效时不能手动启动。
- 漏水异常存在时不能手动启动该水源。
- 手动运行中出现低液位或无流量，按同样规则停止。
- 手动运行也需要记录开始、结束、时长、分支水路和流量数据。

## 状态灯方案

主控和从站都按 2 个产品状态灯设计。额外硬件 LED 只作为固定硬件动作指示，不进入产品状态模型。

两个产品状态灯只表达“系统是否在运行”和“是否有需要处理的异常”。具体异常类型由 Web 和后续 LCD 显示，不通过 LED 组合继续细分。

### 主控 2 灯

- `RUN`：主控程序运行指示。正常运行时慢速心跳闪烁；启动、升级或恢复出厂过程中快速闪烁。
- `ALM`：系统异常指示。无异常时熄灭；任一水源存在需要处理的异常时闪烁。

主控不单独用 LED 表达页面、天气、统计、记录等 Web 信息。

### 从站 2 灯

- `RUN`：从站程序运行指示。正常运行时心跳闪烁；启动或通信识别时快速闪烁。
- `ALM`：从站异常指示。无异常时熄灭；本从站存在低液位、漏水、无流量等需要处理的异常时闪烁。

从站运行中的具体输出状态由阀输出指示灯和泵/干接点输出指示灯表达，不再占用产品状态灯。

## 待办：本地显示和按键

LCD2004 和主控四个业务按键暂不设计。等产品范围、状态模型、信息架构和 Web 原型方案全部确定后，最后单独抽取最小现场能力。

该能力应从已经确认的 Web 功能中抽取，不提前另开一套本地菜单体系。
