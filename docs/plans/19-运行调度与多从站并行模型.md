# 19 运行调度与多从站并行模型

状态：成熟产品方案层文档。本文定义主控如何理解、创建、跟踪和结束灌溉运行，以及多个从站并行、同一从站单任务、手动/计划/维护运行之间的边界。不定义线程、任务、队列、类名、文件名或具体代码实现。

## 核心原则

- 主控不是全局单任务系统。
- 多个从站代表多个独立水源，可以同时运行。
- 一个从站同一时间只能有一个活动运行。
- 一个从站内部不支持多水路并行。
- 从站运行只能由主控命令启动。
- 从站本地安全策略可以独立停机。
- 主控通信丢失时，不能假定从站已经停机。
- 任一从站故障默认只影响该从站，不自动停止其他从站。
- 全局急停是用户明确动作，不是普通故障的默认联动。

## 运行对象

主控用 `RunSession` 表示一次运行。

`RunSession` 的业务字段至少包括：

```text
run_id
run_type
station_id
rs485_address
zone_index
source
requested_duration_sec
started_at
expected_stop_at
state
stop_reason
config_version
flow_baseline_ml_min
flow_policy
created_by
```

`run_type`：

| 类型 | 含义 | 是否普通灌溉 |
|---|---|---|
| `manual` | 用户手动启动水路 | 是 |
| `schedule` | 自动计划启动水路 | 是 |
| `flow_calibration` | 流量计校准放水 | 否 |
| `baseline_learning` | 标准流速学习 | 否 |
| `maintenance_test` | 维护测试输出或识别流程 | 否 |

规则：

- 普通灌溉运行不强制要求流量计计量校准完成；未校准时水量估算不可用或仅供参考。
- `flow_calibration` 可以在流量计未校准时运行，但必须由 Web 明确进入校准流程。
- `baseline_learning` 要求流量计已校准。
- 维护类运行必须有明确时长上限，不能参与自动计划。

## 活动运行定义

以下状态都算活动运行，会占用该从站：

- `created`
- `start_requested`
- `starting`
- `running`
- `stop_requested`
- `stopping`
- `lost_contact`

以下状态不再占用该从站：

- `stopped`
- `failed_to_start`
- `faulted`
- `cancelled`

`lost_contact` 特殊说明：

- 主控与从站通信异常，且最后一次已知该从站处于启动或运行相关状态。
- 主控不能确认从站是否已经关闭输出。
- 该从站继续视为被占用，直到重新获取从站状态。
- 通信恢复后，如果从站上报 `IDLE`、`FAULT`、`LOCKED` 或 `OFFLINE_LOCAL_SAFE`，主控结束该 `RunSession` 并记录停止原因。

## 主控运行状态

主控侧 `RunSession` 状态：

| 状态 | 含义 |
|---|---|
| `created` | 主控已创建运行意图，尚未向从站发命令 |
| `start_requested` | 主控正在请求从站启动 |
| `starting` | 从站已接受，正在启动泵、主阀和分支阀 |
| `running` | 从站上报正在运行 |
| `stop_requested` | 主控已决定停止，正在请求从站停止 |
| `stopping` | 从站正在执行关闭顺序 |
| `stopped` | 已确认停止，正常结束 |
| `failed_to_start` | 启动请求被拒绝或启动过程中失败 |
| `faulted` | 运行因故障或保护结束 |
| `lost_contact` | 主控无法确认从站当前状态 |
| `cancelled` | 尚未启动前被取消 |

状态转换原则：

- 主控状态必须以从站上报状态为准。
- 主控可以根据用户动作或计划时长发出停止请求。
- 从站本地保护停机时，主控必须接受从站上报的停止原因，不能覆盖成用户停止。
- 主控超时没有收到停止确认时，进入 `lost_contact`，不能直接写成 `stopped`。

## 启动准入规则

主控启动任意 `RunSession` 前必须检查：

- 从站已启用。
- 从站在线。
- 未标记疑似地址冲突或总线异常。
- 从站安全配置已同步。
- 从站当前没有活动运行。
- 从站未处于 `FAULT`、`LOCKED`、`UNCONFIGURED`、`OFFLINE_LOCAL_SAFE`。
- 目标水路是启用的分支水路。
- 目标水路不是主阀。
- 目标水路未锁定。
- 请求时长在允许范围内。
- `START_ZONE` 使用的 `config_version` 与从站本地版本一致。

普通灌溉额外要求：

- 从站已配置流量计输入。
- 如果流量计计量校准未完成，该水路仍可运行，但水量估算不可用或仅供参考。
- 如果标准流速未学习，该水路仍可运行，但不做低/高流量判断，并在 Web 上明确提示。

低液位和自吸泵规则：

- 未配置低液位时，不阻止启动自吸泵。
- 配置了低液位且配置了自吸泵时，启动自吸泵前必须确认低液位未触发。
- 配置了低液位但未配置自吸泵时，运行前和运行中仍使用低液位保护。

## 多从站并行规则

主控同时允许多个从站运行，前提是：

- 每个从站最多一个活动 `RunSession`。
- 不同从站之间不共享同一个物理水源。
- 不同从站之间没有被配置为全局互斥。

当前产品默认：

- 从站之间互不全局互斥。
- 一个从站故障不停止其他从站。
- 一个从站离线不停止其他从站。
- 一个从站配置未同步不影响其他已同步从站运行。

全局急停：

- 用户在 Web 执行“停止全部”时，主控对所有活动从站分别请求停止。
- 某个从站停止命令失败，不阻塞其他从站停止。
- 停止全部的结果必须按从站分别展示：已停止、停止中、通信失败、未知状态。

## 同一从站冲突规则

同一从站已有活动 `RunSession` 时：

| 新请求来源 | 处理 |
|---|---|
| 手动运行 | 拒绝，提示该从站正在运行 |
| 自动计划同一从站下一水路 | 排队到当前水路结束后顺序执行 |
| 自动计划其他计划项 | 按计划冲突规则处理，不能抢占当前运行 |
| 流量计校准 | 拒绝，要求先停止当前运行 |
| 标准流速学习 | 拒绝，要求先停止当前运行 |
| 维护测试 | 拒绝，要求先停止当前运行 |
| 用户停止当前水路 | 允许 |
| 停止全部 | 允许 |

说明：

- “排队”只限同一计划内部对同一从站的连续水路。
- 不同计划之间不默认抢占。
- 手动运行不自动插队到计划队列中。
- 用户可以先手动停止当前运行，再启动新的手动运行。

## 停止优先级

停止请求和保护事件按优先级解释：

| 优先级 | 来源 | 结果 |
|---:|---|---|
| 1 | 从站本地安全保护 | 立即停机，停止原因以从站为准 |
| 2 | 用户停止全部 | 所有活动从站分别停止 |
| 3 | 用户停止单个运行 | 目标从站停止 |
| 4 | 主控计划时长到达 | 目标运行停止 |
| 5 | 计划取消或禁用 | 尚未启动的运行取消，已运行的按用户策略停止或继续 |
| 6 | Web 页面离开或刷新 | 不影响运行 |

从站本地安全保护包括：

- 低液位。
- 无水。
- 高流量。
- 通信超时。
- 运行超时。
- 电压严重异常。
- 配置失效。
- 本地输出异常。

规则：

- 如果用户点击停止的同时从站已因低液位停机，最终停止原因是 `LOW_LEVEL`。
- 如果主控计划时长到达但停止命令失败，主控进入 `lost_contact`，等待从站通信超时或状态恢复。
- 如果从站上报 `OFFLINE_LOCAL_SAFE`，主控记录通信丢失安全停机，不自动重新启动。

## 计划运行解释规则

自动计划触发时，主控为每个目标从站创建或尝试创建 `RunSession`。

同一计划中：

- 同一从站的多个水路按配置顺序运行。
- 不同从站的水路可以并行运行。
- 一个从站跳过，不影响同一计划中的其他从站。

计划项无法启动时：

| 原因 | 处理 |
|---|---|
| 从站离线 | 跳过该从站计划项并记录 |
| 配置未同步 | 跳过该从站计划项并记录 |
| 地址疑似冲突 | 跳过该从站计划项并记录 |
| 从站故障或锁定 | 跳过该从站计划项并记录 |
| 水路锁定 | 跳过该水路并记录 |
| 流量计计量未校准 | 允许运行并记录“水量估算不可用或仅供参考” |
| 同从站已有非本计划运行 | 跳过并记录 `station_busy`，不隐藏排队 |

当前默认策略：

- 自动计划不抢占手动运行。
- 自动计划不自动清除故障。
- 自动计划不自动重新同步配置后立刻运行，除非用户明确执行同步并确认。
- 自动计划遇到不可运行条件时记录跳过原因，等待下一次计划触发。

## 手动运行解释规则

手动运行是用户即时动作。

规则：

- 手动运行只启动一个从站的一路分支水路。
- 手动运行不能同时选择多个从站或多个水路。
- 手动运行不能绕过流量计校准、故障、锁定、配置同步或地址冲突。
- 手动运行可以在其他从站正在运行时启动。
- 手动运行不能抢占同一从站已有运行。

如果用户需要切换同一从站水路：

1. 先停止当前水路。
2. 等待从站确认停止。
3. 再启动新的水路。

## 校准和学习运行

`flow_calibration`：

- 用于生成从站流量计 `pulses_per_liter`。
- 可以在该从站流量计未校准时进行。
- 必须选择一个实际能出水的分支水路。
- 必须有明确时长上限和用户确认。
- 运行结果不计入普通灌溉计划完成情况。

`baseline_learning`：

- 用于生成某一路分支水路标准流速。
- 要求流量计已校准。
- 只作用于选中的水路。
- 学习失败不锁定水路，但该水路不启用低/高流量判断。

校准和学习运行都必须占用该从站，不能与该从站普通灌溉并行。

## 通信丢失后的主控解释

主控对运行中从站连续通信失败时：

1. 标记该 `RunSession` 为 `lost_contact`。
2. Web 显示“通信丢失，等待从站本地安全停机或恢复上报”。
3. 主控继续按离线重试策略轮询该地址。
4. 不再向该从站发新的启动命令。
5. 不把该运行记为正常完成。

通信恢复后：

| 从站上报状态 | 主控处理 |
|---|---|
| `IDLE` | 结束运行，记录停止原因未知或从站最近停止原因 |
| `FAULT` | 结束运行，记录故障停止 |
| `LOCKED` | 结束运行，记录锁定原因 |
| `OFFLINE_LOCAL_SAFE` | 结束运行，记录通信丢失本地安全停机 |
| `RUNNING` | 保持 `lost_contact` 或恢复为 `running`，但必须记录通信中断事件 |
| `UNCONFIGURED` | 结束运行，标记配置异常 |

规则：

- 通信恢复后不自动补跑遗漏时长。
- 通信恢复后不自动继续下一个同站计划水路。
- 用户或后续计划必须重新触发新的运行。

## 运行结束结果

每个 `RunSession` 结束时必须归类：

| 结果 | 含义 |
|---|---|
| `completed` | 按计划时长或用户正常流程结束 |
| `stopped_by_user` | 用户主动停止 |
| `skipped` | 未启动，因条件不满足跳过 |
| `failed_to_start` | 启动命令失败或从站拒绝 |
| `stopped_by_fault` | 从站本地保护或故障停机 |
| `lost_contact` | 主控无法确认最终状态 |
| `cancelled` | 启动前被取消 |

记录要求：

- 运行结果不能只记录“成功/失败”。
- 必须保留停止原因、触发来源、从站、水路、运行时长、累计流量或脉冲、是否本地保护停机。
- 如果最终状态曾经过 `lost_contact`，记录中必须保留通信中断标志。

## Web 展示规则

Web 首页同时展示多个运行。

排序建议：

1. 正在运行或启动/停止中的从站。
2. `lost_contact` 的从站。
3. 故障或锁定从站。
4. 配置未同步从站。
5. 在线空闲从站。
6. 离线从站。

每个运行至少显示：

- 从站/水源名称。
- 水路名称。
- 运行类型。
- 已运行时长。
- 计划时长。
- 当前流速。
- 累计水量或累计脉冲。
- 停止按钮。
- 当前状态和最近通信时间。

Web 不应把“主控已发停止命令”显示成“已经停止”。只有从站确认停止或状态恢复后，才能显示为已停止。

## 不支持的运行能力

当前产品不支持：

- 一个从站同时运行多个分支水路。
- 一个从站配置多个流量计。
- 从站主动抢占总线上报。
- 多主控管理同一组从站。
- 计划自动抢占手动运行。
- 通信丢失后自动继续未完成水路。
- 未经用户确认的自动故障清除后立即重启。
- Web 任意透传运行命令绕过主控业务校验。
