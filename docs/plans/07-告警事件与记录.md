# 07 告警事件与记录

## 概念边界

| 概念 | 回答的问题 | 是否当前态 | 是否历史态 |
|---|---|---|---|
| `Alarm` 告警 | 现在用户需要处理什么 | 是 | 可归档 |
| `Fault` 故障 | 设备为什么不能运行或为什么停机 | 是 | 可记录 |
| `Event` 事件 | 过程中发生了什么可解释动作 | 否 | 是 |
| `RunRecord` 运行记录 | 一次水路运行的结果 | 否 | 是 |
| `ScheduleRunRecord` 计划记录 | 一次计划触发的整体结果 | 否 | 是 |

规则：

- 清除告警不删除事件和运行记录。
- 清除故障不表示真实问题已经修好。
- 历史记录不能被当前状态覆盖。
- 一个故障通常会产生一个告警和一个事件。
- 一次故障停机会结束一个 `RunSession` 并生成运行记录。

## 影响范围

| 范围 | 含义 |
|---|---|
| `system` | 影响主控或整体系统 |
| `station` | 影响一个从站/水源 |
| `zone` | 影响一个分支水路 |
| `run` | 只影响一次运行 |
| `config` | 影响配置可用性 |

从站级故障默认不影响其他从站。水路级锁定默认只影响该水路。

## 告警生命周期

```text
raised -> active -> acknowledged -> condition_cleared -> cleared -> archived
```

规则：

- 用户确认 `acknowledged` 不等于清除。
- 条件恢复 `condition_cleared` 不等于用户已经处理。
- 只有满足清除条件后，才能进入 `cleared`。
- 进入 `cleared` 后仍保留历史事件。

## 故障生命周期

```text
triggered -> protected -> waiting_clear -> cleared
```

规则：

- 故障触发后必须先进入安全状态，再允许清除。
- 不可清除故障只能通过修复条件或重新同步配置解决。
- 清除故障后不能自动启动运行。

## 告警生成规则

| 条件 | 范围 | 是否禁止运行 |
|---|---|---|
| 低液位 | `station` | 是，禁止该从站 |
| 启动无水 | `station` 或 `zone` | 是，按锁定范围 |
| 运行中无水 | `station` 或 `zone` | 是，按锁定范围 |
| 高流量 | `zone` | 是 |
| 低流量 | `zone` | 按策略 |
| 待机漏水 | `station` | 默认禁止该从站新运行 |
| 通信丢失 | `station` | 是，直到确认状态 |
| 地址疑似冲突 | `system` 或 `station` | 是，禁止相关地址 |
| 配置未同步 | `config` | 是，禁止相关从站 |
| 配置无效 | `config` | 是 |
| 电压检测未校准 | `station` | 否，仅提示电压监测未启用 |
| 已启用电压监测并确认电压低/高 | `station` | 否，仅告警和记录 |
| 主控时间无效 | `system` | 禁止自动计划 |

低/高流量告警只在流量计输入有效、目标水路标准流速已学习、对应保护已启用后生成。标准流速未学习时只记录保护未启用，不生成阻断性低/高流量告警。

电压检测未校准或电压监测未启用时，不生成电压低/高告警。

电压低/高告警不禁止启动、不触发停机、不进入锁定；它只用于提示用户检查电源、线路、负载和现场供电质量。

## 清除规则

| 告警或故障 | 清除条件 |
|---|---|
| 低液位 | 液位恢复，或用户确认并允许清除 |
| 无水 | 用户确认检查水源、泵、阀、流量计 |
| 高流量 | 用户确认检查管路、脱管、阀门和流量计 |
| 低流量 | 用户确认，或策略允许自动恢复 |
| 待机漏水 | 用户确认检查后，且当前无持续脉冲 |
| 通信丢失 | 从站重新上报安全状态后清除 |
| 地址疑似冲突 | 重新扫描后异常消失 |
| 配置未同步 | 同步成功 |
| 配置无效 | 重新同步有效配置 |
| 电压检测未校准 | 完成校准或用户确认暂不启用 |
| 已启用电压监测并确认电压低/高 | 电压恢复到告警阈值内，或用户确认后清除 |
| 主控时间无效 | 时间恢复有效 |

规则：

- 清除失败必须显示原因。
- 清除后不自动启动。
- 如果清除后条件仍存在，应重新生成告警。

## 事件记录

事件类型：

| 类型 | 例子 |
|---|---|
| `run_started` | 手动或计划启动水路 |
| `run_stopped` | 正常停止或用户停止 |
| `run_faulted` | 本地保护停机 |
| `schedule_triggered` | 计划触发 |
| `schedule_skipped` | 计划项跳过 |
| `config_changed` | 用户保存配置 |
| `config_sync` | 配置同步成功或失败 |
| `fault_cleared` | 用户或维护动作清除故障 |
| `maintenance` | 扫描、识别、恢复配置 |
| `calibration` | 流量计校准和标准流速学习 |
| `firmware_update` | ESP32 OTA、从站 RS485 OTA 成功、失败或中断 |

事件不能被清除故障删除。

## 运行记录

`RunRecord` 必须包含：

```text
run_id
run_type
station_id
zone_index
started_at
ended_at
requested_duration_sec
actual_duration_sec
result
stop_reason
pulse_count
total_ml
flow_summary
config_version
had_lost_contact
disabled_protections
estimation_quality
created_by
```

运行结果：

| 结果 | 含义 |
|---|---|
| `completed` | 按预期完成 |
| `stopped_by_user` | 用户停止 |
| `skipped` | 未启动即跳过 |
| `failed_to_start` | 启动失败 |
| `stopped_by_fault` | 故障或保护停机 |
| `lost_contact` | 最终状态无法确认 |
| `cancelled` | 启动前取消 |

## 计划记录

`ScheduleRunRecord` 必须包含：

```text
schedule_run_id
schedule_id
triggered_at
ended_at
result
station_results
skipped_items
failed_items
run_ids
```

计划结果：

| 结果 | 含义 |
|---|---|
| `completed` | 全部应执行项完成 |
| `partial` | 部分完成，部分跳过或失败 |
| `skipped` | 全部应执行项未启动即跳过 |
| `failed` | 至少尝试启动或运行，但没有任何水路成功完成 |
| `cancelled` | 用户取消 |

规则：

- 计划全部跳过必须显示为 `skipped`，不能显示为 `failed`。
- 跳过项必须保留跳过原因。
- 删除计划不删除计划历史。

## 不支持

当前产品不支持：

- 云端告警推送。
- 手机 App 消息。
- 复杂账号审计。
- 用户自定义脚本规则。
- 从站本地长期保存完整运行历史。
- 清除故障时删除历史证据。
