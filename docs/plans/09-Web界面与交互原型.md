# 09 Web 界面与交互原型

## 文档目标

本文档定义第一版 Web、手机通知与轻操作、LCD2004 和本地按键的结果性交互规则，用于约束当前最终高保真原型和后续实施计划。

本文档不记录中间推进过程、子会话推进记录、前端技术栈、路由文件、组件拆分、CSS 方案、后端类名或代码任务。

当前唯一可点击 Web 原型入口是 `docs/prototypes/web/final-prototype.html`。

## 交互边界

- Web 是完整配置、运行、维护、故障处理和记录查看入口。
- 手机通知与轻操作只提供通知、关键状态和暂停/恢复自动浇水，不提供手动启动、停止、配置、清故障、OTA 或维护入口。
- LCD2004 + 主控本地按键只提供现场基础控制：查看状态、停止当前浇水、发起一次手动浇水、微调本次时长、暂停/恢复自动浇水、清除可恢复故障。
- 从站本地按键只用于维护识别和受控清除可恢复故障，不直接启动灌溉。
- 天气只显示、解释和建议，不自动停机、不禁止启动、不自动跳过计划、不自动改时长、不自动改配置。
- 页面不能提供通用 IO、通用寄存器、任意 RS485 帧透传或脚本规则入口。

## 全局布局

Web 控制台采用分层导航：

1. 日常用户页：总览、运行、计划、暂停、记录、统计、天气、事件。
2. 配置页：设置、水源配置、水路配置。
3. 维护页：从站维护、识别、清除可恢复故障、从站 OTA。
4. 诊断详情：只从维护、事件、状态无法确认、待同步失败、断电恢复摘要等入口进入，不作为普通顶层页。

全局规则：

- 用户控制台优先显示当前运行、水源可用性、今日计划、阻断事件和运行入口。
- 设置围绕真实水源组织，不做系统设置页。
- 维护集中处理现场服务和高影响动作，不做普通运行控制台。
- 诊断详情显示底层证据，不承载维护动作入口。
- 列表页只显示摘要和操作入口，不默认展开完整编辑表单。
- 点击编辑、新增、配置或设置后，进入独立表单页、独立视图、弹窗或抽屉。
- 配置页面保存主控侧配置后，如果影响从站安全配置，必须显示待同步状态。
- 任何未同步、离线、故障、锁定、地址冲突、低液位、待机漏水等阻断状态，都必须在启动入口前可见。
- 电压状态只作为显示、告警和记录，不表现为停机原因、启动禁止原因或入口电源控制。

## 状态语言

| 状态类型 | 显示方式 | 使用场景 |
|---|---|---|
| 正常/可运行 | 绿色小状态点、状态标签、可执行按钮 | 水源可运行、水路可运行、同步成功 |
| 信息/进行中 | 蓝色状态标签、进度条、当前步骤 | 启动中、停止中、同步中、OTA 中 |
| 需注意/保护不完整 | 琥珀色提示条或标签 | 未校准、未学习、天气建议、配置待同步 |
| 阻断/危险 | 红色阻断条、禁用按钮旁原因、高影响确认 | 故障、锁定、低液位、通信丢失、清空配置、OTA |
| 离线/禁用 | 灰色状态标签、弱化对象行 | 从站离线、水路禁用、功能关闭 |

状态规则：

- 颜色只辅助理解，不能作为唯一信息。
- `STOPPING` 必须显示为正在停止，不能提前显示为已停止。
- 状态无法确认优先级高于普通离线提示，必须提供诊断入口。
- 未校准、未学习、未配置显示为保护不完整或功能未启用，不显示为设备故障。
- 结果语言必须区分完成、部分完成、未启动/跳过、失败、取消、保护停机和状态无法确认。

## 动作和表单规则

- 每页最多一个主动作区域。
- 禁用动作保留按钮位置并说明原因，不能只隐藏。
- 危险动作必须二次确认，确认文案必须具体，例如“确认停止全部”“确认删除计划”“确认发起从站 OTA”。
- 配置页没有启动灌溉主动作。
- 维护页没有普通用户日常运行主动作。
- 事件页只处理事件状态，不发起 OTA 或普通灌溉。
- 手机轻入口没有手动启动、停止、配置、清故障或 OTA 动作。
- 表单按业务段落分组，不按硬件字段散列。
- 保存结果必须区分主控保存成功、待同步、同步成功和同步失败。
- 运行中从站允许保存主控侧配置，但同步动作禁用并说明运行结束后再同步。

## 页面职责表

| 页面 | 回答的问题 | 主要内容 | 不做 |
|---|---|---|---|
| 总览 | 现在能不能浇水，下一步是什么 | 当前运行、水源状态、今日计划、阻断摘要、运行入口 | 天气摘要、系统设置、诊断长表、配置表单 |
| 运行 | 本次手动浇哪些水路，能否启动，运行结果是什么 | 水源选择、水路顺序、本次时长、保护摘要、启动确认、运行进度、停止入口 | 保存计划、修改水路配置、跨水源编成一个任务 |
| 计划 | 自动浇水规则是什么，下一次何时执行 | 计划列表、独立计划表单、周期规则、最多 4 个开始时间、水源和水路顺序 | 手动启动、补跑队列、天气自动跳过、事件处理 |
| 暂停 | 是否暂停自动浇水，何时恢复 | 暂停状态、影响范围、恢复时间、暂停/恢复动作 | 停止当前手动运行、修改计划、清故障 |
| 记录 | 已经发生过哪些水路级运行事实 | 水路级记录列表、筛选、记录详情、关联事件入口 | 清除事件、统计大图、处理阻断 |
| 统计 | 正常浇水趋势如何 | 近 30 天水量、时长、日均水量、水路趋势 | 异常处理、保护停机解释、计划编辑 |
| 天气 | 天气对人工判断有什么影响 | 当前天气、短期预报、人工建议、暂停入口 | 自动控制策略、自动跳过、自动改时长 |
| 事件 | 有哪些告警、阻断和系统事件需要处理 | 事件列表、阻断详情、处理建议、清除确认、关联记录 | 浇水记录大表、统计趋势、OTA |
| 设置 | 进入哪个水源配置 | 水源列表、启用状态、地址、主阀、流量计、低液位、自吸泵、水路数量、配置入口 | 设置总览、系统设置、运行入口、维护动作 |
| 水源配置 | 当前真实水源装了什么，保护策略是什么 | 名称、启用、从站地址、主阀、流量计、低液位、自吸泵、电磁阀参数、保护策略、流量计校准入口 | 分支水路长表、OTA、清故障、通用 IO |
| 水路配置 | 当前水源下 8 路阀分别是什么角色 | 阀路角色、名称、启用、主阀占用、标准流速、编辑入口、学习入口 | 手动/计划权限、运行默认值、完整表单同屏 |
| 维护 | 现场服务要做什么 | 从站列表、在线/运行/故障、固件版本、详情、识别、清除故障、从站 OTA | 普通运行、计划编辑、设置表单、完整诊断 |
| 诊断详情 | 发生问题的底层证据是什么 | 通信、状态、配置版本、同步证据、原始状态、导出诊断 | 普通配置入口、维护动作入口、运行入口 |
| 通知 | 手机轻入口需要知道什么 | 当前运行、暂停状态、最近结果、阻断摘要、暂停/恢复 | 完整 App、手动启动、配置、清故障、OTA |
| 本地 LCD | 现场最小闭环怎么完成 | 基础状态、停止、一次手动浇水、时长微调、暂停/恢复、故障提示 | 完整配置、计划编辑、校准、OTA、完整记录 |

## 页面规则

### 总览

- 首屏必须让用户在 5 秒内判断系统是否可运行。
- 显示当前运行、下一次计划、今日已完成或失败摘要、水源可用性和阻断事件摘要。
- 运行入口只能进入运行页，不在水源卡片上直接启动某一路。
- 不显示天气、RTC、RS485、输入输出位图、固件版本、电压 ADC 或配置表单。

### 运行

- 手动运行不保存为计划，不修改水路配置，不改变计划配置。
- 单水源默认选中当前水源；多水源先选择一个水源。
- 只显示分支水路，主阀不作为可运行水路显示。
- 本次时长只用于本次手动运行。
- 启动前显示水源、水路顺序、时长、保护摘要和保护不完整提示。
- 禁止启动时必须说明具体原因，例如离线、未同步、已有运行、锁定、低液位、待机漏水。
- 运行中显示当前水路、剩余时间、估算流量、最近状态更新时间和始终可见的停止按钮。
- 停止后结果以从站上报原因为准，不把所有停止都覆盖成用户停止。

### 计划

- 计划模型是周期规则 + 最多 4 个开始时间 + 水源/水路顺序 + 每路固定时长。
- 周期规则表达为几天一周期、周期内第几天执行、周期起始日期；每天、每周固定日期、浇几天停几天只能作为快捷解释。
- 开始时间使用 `HH:MM`，每个时间有启用状态；未启用或为空的时间不触发。
- 计划表单必须独立于计划列表。
- 保存前摘要显示计划名称、启用状态、周期解释、启用的开始时间、将运行的水源/水路和每路时长。
- 删除计划必须二次确认，说明只删除后续自动安排，不删除历史运行记录和事件。
- 不提供补跑队列、排队延后、天气自动跳过或维护诊断。

### 暂停

- 暂停只影响后续自动计划，不等同于停止当前运行。
- 暂停状态必须显示影响范围、开始时间、恢复条件或恢复时间。
- 恢复自动浇水不补跑暂停期间错过的计划，除非后续方案另行确认。
- 手机轻入口可以暂停/恢复自动浇水，但不能修改计划。

### 记录

- 记录是水路级运行事实，不是事件列表。
- 列表字段包括时间、水源、水路、触发方式、触发来源、请求时长、实际时长、结果、停止原因和详情入口。
- 详情包括开始/结束时间、估算水量、脉冲数、流量摘要、计量质量、配置版本、本次启用和未启用的保护、关联事件和同一计划下其它水路记录。
- 关联事件入口只用于追溯查看，不在记录页处理或清除事件状态。

### 统计

- 默认范围为近 30 天；近 7 天只能作为次级筛选。
- 顶部只放正常浇水摘要：总水量、总浇水时长、日均水量、浇水最多水路。
- 多水源必须平铺展示，不把水源藏在主要下拉选择里。
- 统计主体按水源和水路展示每日估算水量和每日实际浇水时长。
- 统计不作为自动控制输入，不改变计划，不解释阻断处理。

### 天气

- 天气页只显示天气、预报、人工判断建议和可选的暂停自动浇水入口。
- 天气建议可以提示用户手动暂停自动浇水，但不得自动执行。
- 不显示天气自动跳过、天气规则配置、自动改时长或安全策略输入。

### 事件

- 事件包含告警、阻断、配置同步、系统状态、OTA 等状态变化。
- 事件列表字段包括时间、类型、级别、影响对象、当前状态、是否阻断、摘要和详情入口。
- 阻断详情显示触发条件、影响范围、是否禁止运行、处理建议、关联记录和清除入口。
- 清除阻断必须二次确认，说明不删除事件记录、不删除运行记录、不自动启动计划或手动运行。
- 清除失败必须显示原因，例如条件仍存在、从站离线、状态无法确认或配置仍未同步。
- 事件页不发起 OTA，不显示浇水记录大表，不做统计趋势。

### 设置和水源列表

- 顶层入口命名为设置，默认进入水源列表。
- 不制作设置总览页，不制作系统设置页。
- 水源列表用于选择、新增真实水源和查看摘要。
- 列建议为水源名称、启用状态、从站地址、主阀、流量计、低液位、自吸泵、水路、操作。
- 操作只提供编辑配置和水路入口，进入独立页面或视图。
- 不提供启动灌溉、OTA、清空配置或清故障动作。
- 默认层不显示 RS485 原始状态、IN 编号、CRC、API、协议命令、芯片型号或输入输出位图。

### 水源配置

- 从水源列表的配置入口进入，一次只配置一个真实水源。
- 基础信息包括水源名称、启用状态和从站地址。
- 从站地址为必填，范围 `1..15`；地址 `0` 保留。地址配置和地址通信状态分开表达。
- 主阀是水源级硬件装配：无主阀/有主阀；有主阀时选择占用 `VALVE1..VALVE8` 并配置打开后等待和关闭前等待。
- 流量计是水源级固定接口，有源脉冲输入；显示当前参数、修改参数入口和开始校准入口。
- 低液位传感器是水源级可选固定接口，干接点输入；只配置启用/不启用。
- 自吸泵控制是水源级可选固定接口，干接点输出；只配置启用/不启用。
- 电磁阀参数是水源级统一参数，作用于该水源下的主阀和分支水路。
- 保护策略留在水源配置：无水、低流量、高流量、待机漏水。
- 不显示分支水路摘要、输入输出位图、脉冲计数、CRC、API 或协议命令。

### 水路配置

- 从某个水源的水路入口进入，始终处于当前水源上下文。
- 显示当前水源下 `VALVE1..VALVE8` 的阀路状态。
- 被主阀占用的阀路显示为主阀占用，不可编辑为分支水路，不能手动或计划运行，标准流速不适用。
- 分支水路行显示第几路、角色、名称、启用、标准流速和编辑入口。
- 点击编辑后进入独立水路编辑视图或弹出独立编辑层；完整列表和完整编辑表单不得默认同屏。
- 水路编辑一次只编辑一路分支水路：名称、启用、标准流速值、修改标准流速入口和学习标准流速入口。
- 第一版不显示 `manual_allowed`、`schedule_allowed`、`default_duration_sec`、运行默认值、默认时长、手动权限或计划权限。
- 启用的分支水路默认可被手动运行和计划选择，实际能否启动由锁定、阻断、在线状态和统一运行准入决定。

### 维护

- 维护面向现场服务和高影响动作。
- 默认显示从站/水源列表，列建议为选择、从站/水源、地址、在线与运行、故障、固件版本、操作。
- 操作包括详情、识别、清除故障。
- 从站 OTA 从维护页进入：上传固件文件、选择目标从站、确认更新、显示阶段和结果。
- 发起 OTA 前提示目标从站不能运行，全部输出保持关闭。
- 清除故障只用于可恢复故障或锁定；清除不表示真实问题已经修好，不删除事件和运行记录，不自动启动运行，不补跑计划。
- 维护页不提供普通运行、计划编辑、水源配置表单、流量校准表单或完整诊断长表。

### 诊断详情

- 诊断详情面向安装、调试和现场维护人员，不作为普通顶层入口。
- 进入来源包括维护页、事件详情、状态无法确认、待同步失败、断电恢复摘要和导出诊断。
- 可显示通信状态、配置版本、同步证据、原始状态、输入输出位图、协议证据和导出入口。
- 默认用户层不得直接显示 API、CRC、位图、协议命令、SEQ、原始参数名或芯片完整型号。
- 诊断详情不发起 OTA、清故障、普通运行或配置保存。

### 通知和手机轻入口

- 手机轻入口只显示当前运行、自动浇水暂停状态、最近结果和阻断摘要。
- 主要动作只允许暂停/恢复自动浇水。
- 通知可提示运行完成、保护停机、阻断、配置待同步、断电恢复和 OTA 结果。
- 通知不提供手动启动、停止、配置、清故障、OTA 或完整远程控制后台。

### 本地 LCD 和按键

- LCD 文案必须短，优先显示状态、对象、时间或下一步。
- LCD 只显示现场基础控制所需信息，不显示完整配置、计划编辑、校准或 OTA。
- LCD 异常提示必须引导到 Web 查看详情。
- 主控本地按键可以停止当前浇水、发起一次手动浇水、微调本次时长、暂停/恢复自动浇水和清除可恢复故障。
- 从站本地按键只用于识别和维护确认，不直接启动灌溉。

## 关键流程

### 手动运行

1. 用户从总览或运行页进入运行。
2. 选择水源和分支水路顺序，填写本次时长。
3. 页面执行准入检查：在线、未运行、未锁定、配置已同步、低液位正常、无待机漏水、状态可确认。
4. 如果准入失败，保留启动按钮位置并显示具体原因。
5. 用户确认水源、水路、时长、保护状态和保护不完整提示。
6. 主控按顺序启动水路；主阀按水源配置自动配合。
7. 页面显示启动中、运行中、正在停止、完成或失败状态。
8. 每一路实际启动、完成、失败、保护停机、用户停止或状态无法确认，都生成水路级运行记录。
9. 相关告警或阻断生成事件。

### 暂停和恢复自动浇水

1. 用户从暂停页、总览提示或手机轻入口进入。
2. 页面显示暂停影响范围、当前计划影响和恢复条件。
3. 用户确认暂停或恢复。
4. 暂停后，后续自动计划不启动；当前正在运行的任务不因此自动停止。
5. 恢复后，只影响后续触发，不自动补跑暂停期间错过的计划。
6. 暂停、恢复和因暂停跳过的计划结果进入事件或记录中的对应证据。

### 创建和编辑计划

1. 用户从计划列表进入新增或编辑表单。
2. 设置名称、启用状态、周期规则、最多 4 个开始时间。
3. 选择水源和分支水路顺序，并填写每路固定时长。
4. 页面提示系统时间无效、自动浇水暂停、目标水源离线、配置待同步、存在阻断或无可运行水路等影响。
5. 保存前显示摘要。
6. 保存成功后回到计划列表；删除计划必须二次确认。
7. 计划触发后的每一路结果进入运行记录，阻断或异常进入事件。

### 查看记录

1. 用户在记录页选择时间范围、水源、触发方式和结果类型。
2. 页面显示水路级记录列表。
3. 用户进入记录详情查看运行事实、计量质量、保护状态和关联事件。
4. 如果需要处理事件，从关联入口跳转到事件详情；记录页本身不清除事件。

### 事件处理

1. 用户从总览阻断条、事件页、记录详情或通知进入事件详情。
2. 页面显示触发条件、影响范围、是否禁止运行和处理建议。
3. 可清除事件必须显示清除条件和二次确认。
4. 清除动作不删除事件记录、不删除运行记录、不自动启动计划或手动运行。
5. 清除失败时显示具体原因并保留原状态。

### 配置保存和同步

1. 用户在水源配置或水路配置中修改配置。
2. 保存先写入主控侧配置，并显示保存成功或失败。
3. 如影响从站安全配置，页面显示待同步。
4. 同步前检查从站在线、未运行、地址无冲突、配置校验通过。
5. 同步中显示阶段和结果。
6. 同步成功后解除待同步阻断；同步失败显示原因和处理入口。

### 流量计校准

1. 用户从水源配置中的流量计区域点击开始校准。
2. 页面显示当前水源、固定接口、当前参数和准入检查。
3. 用户执行出水、停水、输入实际出水量、保存结果。
4. 校准成功后更新流量计参数，并生成记录或事件证据。
5. 校准失败、用户取消、通信丢失或输入水量非法时，不覆盖原校准值。

### 标准流速学习

1. 用户从水路配置的具体水路编辑视图进入学习标准流速。
2. 页面显示当前水源和当前水路，只针对这一条分支水路。
3. 用户执行出水、停水、保存标准流速。
4. 学习成功后更新该水路标准流速。
5. 学习失败、用户取消或保护停机时，不覆盖原标准流速。

### OTA

1. OTA 只能从维护页进入。
2. 用户上传从站固件文件并选择目标从站。
3. 页面显示影响对象、输出关闭、运行禁用和失败处理说明。
4. 用户二次确认后发起。
5. 页面显示准备、传输、校验、提交、重启确认、成功或失败阶段。
6. 成功、失败、中断和重试都生成事件。
7. 失败或中断后只提示重新升级或现场处理，不自动恢复运行。

### 上电通知

1. 主控上电后显示系统恢复状态。
2. 如果存在上次运行中断、配置待同步、从站状态无法确认或时间无效，通知和总览必须提示。
3. 断电恢复摘要可以进入诊断详情查看证据。
4. 系统不能因为上电恢复摘要自动补跑计划或自动启动手动运行。

## 明确禁止项

- 不做天气自动控制。
- 不做系统设置页。
- 不把系统时间、LCD、SHT30、RS485、记录保留、刷新间隔做成普通配置项。
- 不把 OTA 放入顶层导航、事件页、诊断详情、通知或 LCD。
- 不把事件页做成浇水记录页。
- 不把记录页做成事件处理页。
- 不把统计页做成 BI 仪表盘或异常处理页。
- 不在设置或水路配置中暴露手动/计划独立权限和运行默认值。
- 不把主阀做成可运行分支水路。
- 不把流量计、低液位、自吸泵做成每路水路能力。
- 不默认同屏展示完整列表和完整编辑表单。
- 不显示 REST 路径参数、JSON body、`PUT`、`DELETE` 或通用 RS485 透传能力。
- 不支持多主控、从站主动上报、同站多水路并行或一个从站多个流量计。

## 原型验收点

- 用户能在 5 秒内判断当前是否可运行。
- 用户能找到每页唯一主任务。
- 阻断、保护不完整、待同步、离线和状态无法确认能一眼区分。
- 事件和记录严格分离。
- 设置围绕真实水源组织，不像系统后台。
- 水源配置和水路配置职责清楚。
- 维护和诊断不干扰日常使用。
- OTA 入口只在维护页。
- 单水源和多水源布局都自然。
- 原型没有引入本文档和 `01..08` 未确认的功能边界。
