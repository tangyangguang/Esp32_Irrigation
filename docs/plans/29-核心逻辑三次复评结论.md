# 29 核心逻辑三次复评结论

状态：当前阶段复评结论。本文是在 `28-核心逻辑二次复评结论.md` 之后，再次检查“增强能力未就绪不得阻断核心运行”的残留口径。本文只处理方案逻辑、模型和边界，不写实施计划，不决定工程实现方式。

## 一、总体判断

本轮复评发现 4 个同类残留问题，均属于旧口径残留：把“流量计计量未校准”错误地写成运行阻断条件。

当前已统一修正为：

- 流量计输入必须配置，这是核心条件。
- `pulses_per_liter` 计量校准不是核心运行条件。
- 计量未校准时仍允许普通灌溉运行。
- 计量未校准时水量估算不可用或仅供参考。
- 能检测脉冲时，仍可用于有无流量判断。

当前仍不进入实施计划。

## 二、本轮必须修正的问题

### P1：验证清单把“未校准”写成禁止启动条件

涉及文档：

- `08-产品验证范围清单.md`

问题描述：

- 原文写“主控不能对离线、未同步、故障、锁定、未校准的从站启动运行”。
- 这里的“未校准”容易覆盖流量计计量未校准、电压检测未校准、标准流速未学习。

风险：

- 测试人员会按旧口径验证，导致增强能力未就绪时错误判定不可运行。

修正方式：

- 改为禁止对“未配置流量计输入或已触发明确安全阻断条件”的从站启动。
- 明确计量未校准、标准流速未学习、电压检测未校准不属于启动阻断条件。

### P1：主控配置状态 `runtime_usable` 错误要求流量计已校准

涉及文档：

- `15-主控数据模型与生命周期边界.md`

问题描述：

- 原文把 `runtime_usable` 定义为“从站在线、无锁定、流量计已校准，可参与运行”。

风险：

- 主控会把计量增强能力当成核心准入条件，导致未校准时无法运行。

修正方式：

- 改为“流量计输入已配置、无明确安全阻断条件”。
- 明确流量计计量未校准不影响 `runtime_usable`。

### P1：手动启动检查错误要求“流量计已配置并校准”

涉及文档：

- `07-Web API与数据结构.md`

问题描述：

- 手动启动前检查中写“流量计已配置并校准”。

风险：

- Web/API 会把普通运行误做成必须先完成计量校准。

修正方式：

- 改为“流量计输入已配置”。
- 增加“流量计计量未校准不阻断启动，只影响水量估算质量”。

### P1：手动运行规则错误写成不能绕过流量计校准

涉及文档：

- `19-运行调度与多从站并行模型.md`

问题描述：

- 原文写“手动运行不能绕过流量计校准、故障、锁定、配置同步或地址冲突”。
- 这会把计量校准和故障、锁定、地址冲突放在同一类阻断条件里。

风险：

- 手动运行会被误设计成必须先完成 `pulses_per_liter` 校准。

修正方式：

- 改为“手动运行不能绕过流量计输入配置、故障、锁定、配置同步或地址冲突”。
- 增加“手动运行不要求流量计计量校准完成；计量未校准时水量估算不可用或仅供参考”。

## 三、当前统一口径

| 项目 | 是否核心阻断 | 当前处理 |
|---|---|---|
| 未配置流量计输入 | 是 | 禁止启动，要求完成输入配置 |
| 流量计计量未校准 | 否 | 允许运行，水量估算不可用或仅供参考 |
| 标准流速未学习 | 否 | 允许运行，不启用低/高流量保护 |
| 电压检测未校准 | 否 | 允许运行，不启用电压保护 |
| 低液位已触发 | 是 | 禁止启动或运行中停机 |
| 已启用无水检测且启动/运行无脉冲 | 是 | 停机并按策略锁定 |
| 已学习标准流速后高流量确认 | 是 | 停机并按策略锁定 |
| 通信超时或运行超时 | 是 | 从站本地安全停机 |

## 四、下一轮复评重点

下一轮不需要扩大范围重写，应继续查同类残留：

- 是否还有 `runtime_usable`、启动检查、计划检查、验证标准中把“未校准”作为泛化阻断条件。
- 是否还有“流量计已校准才能普通运行”的表述。
- 是否还有“增强保护未启用”等同于“设备故障”的表述。
- 是否还有跨计划同站冲突“等待、排队、延后执行”的表述。
