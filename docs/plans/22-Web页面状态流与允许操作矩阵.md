# 22 Web 页面状态流与允许操作矩阵

状态：成熟产品方案层文档。本文定义 Web 在关键业务状态下应显示的信息、允许的动作、禁止的动作和提示语义。不定义页面布局、组件样式、前端框架或具体路由实现。

## 核心原则

- Web 是完整业务操作入口。
- Web 不能把不可确认的状态显示成已确认。
- Web 不能允许用户绕过主控业务校验。
- Web 不能直接透传 RS485 帧。
- Web 的按钮可见不等于可执行；不可执行时必须给出明确原因。
- Web 页面刷新、关闭或网络短断不影响已经启动的运行。
- 所有高影响动作必须二次确认。

高影响动作包括：

- 启动水路。
- 停止全部。
- 清除故障。
- 重新同步从站安全配置。
- 恢复灌溉业务配置。
- 流量计校准放水。
- 标准流速学习。

## 状态来源

Web 显示状态来自主控。

主控状态来源分为：

| 来源 | 含义 |
|---|---|
| 主控配置 | 用户保存的业务配置 |
| 主控运行态 | 当前 `RunSession`、`ScheduleRun` 和告警状态 |
| 从站上报 | RS485 轮询得到的从站状态、故障、流量、电压 |
| 主控推断 | 通信超时、状态过期、疑似地址冲突 |

规则：

- 从站输出是否关闭，以从站上报或从站本地安全策略为准，不能只看主控是否发过停止命令。
- 主控推断状态必须标明“未确认”或“状态过期”。
- 状态过期时，Web 应限制启动、配置同步、清除故障等动作。

## 首页状态矩阵

首页按设备整体状态展示。

| 状态 | 显示 | 允许动作 | 禁止动作 |
|---|---|---|---|
| 无运行、无告警 | 从站在线摘要、下次计划、温湿度 | 手动运行、进入配置、进入维护 | 无 |
| 有运行 | 多个运行卡片、每站状态、流量、时长 | 停止单路、停止全部、查看详情 | 修改运行中从站安全配置 |
| 有 `lost_contact` | 明确显示通信丢失、最后状态、最后通信时间 | 停止全部、查看维护、等待恢复 | 对该从站启动新运行、清除未知状态 |
| 有故障/锁定 | 故障摘要、影响范围、处理入口 | 查看故障、清除可恢复故障 | 直接启动受影响对象 |
| 有配置未同步 | 从站和原因 | 重新同步、查看配置 | 启动未同步从站 |
| 有地址疑似冲突 | 冲突地址、排查建议 | 扫描、查看维护说明 | 启动该地址 |

首页排序：

1. 正在运行、启动、停止中的从站。
2. `lost_contact` 从站。
3. 故障或锁定从站。
4. 配置未同步从站。
5. 在线空闲从站。
6. 离线从站。

## 从站状态矩阵

| 从站状态 | 用户含义 | 允许动作 | 禁止动作 |
|---|---|---|---|
| `BOOTING` | 从站正在启动 | 查看状态 | 启动、同步、清除故障 |
| `UNCONFIGURED` | 从站无有效安全配置 | 同步配置、查看配置 | 启动、校准、学习 |
| `IDLE` | 空闲可操作 | 启动、校准、学习、同步、识别 | 无 |
| `STARTING` | 正在启动 | 停止、查看详情 | 再次启动、改安全配置 |
| `RUNNING` | 正在运行 | 停止、查看详情 | 再次启动、改安全配置、校准 |
| `STOPPING` | 正在关闭 | 查看详情、停止全部 | 显示为已停止、再次启动 |
| `FAULT` | 有可处理故障 | 查看故障、按条件清除 | 直接启动 |
| `LOCKED` | 安全锁定 | 查看处理建议、清除允许的锁定 | 直接启动 |
| `OFFLINE_LOCAL_SAFE` | 通信丢失后本地安全停机 | 通信恢复后确认/清除 | 自动继续运行 |
| 离线 | 主控无法联系从站 | 扫描、查看最后状态 | 启动、同步、清除从站故障 |

提示语义：

- `STOPPING` 必须显示“正在停止”，不能显示“已停止”。
- `OFFLINE_LOCAL_SAFE` 必须解释为“从站报告已本地安全停机”，不是普通离线。
- 离线状态不能承诺阀门实际状态，只能显示最后已知状态和从站本地安全策略。

## 手动运行页面矩阵

| 条件 | 启动按钮 | 提示 |
|---|---|---|
| 从站在线、空闲、配置同步、无故障、水路可运行 | 允许 | 显示本次运行参数 |
| 从站离线 | 禁止 | 从站离线，不能确认运行安全 |
| 从站配置未同步 | 禁止 | 请先同步配置 |
| 从站故障或锁定 | 禁止 | 请先处理故障 |
| 同一从站已有运行 | 禁止 | 该从站正在运行 |
| 其他从站正在运行 | 允许 | 多个水源可并行运行 |
| 水路禁用 | 禁止 | 水路未启用 |
| 水路锁定 | 禁止 | 水路已锁定 |
| 流量计计量未校准 | 允许 | 水量估算不可用或仅供参考 |
| 标准流速未学习 | 允许 | 低/高流量保护未启用 |
| 电压检测未校准 | 允许 | 电压保护未启用 |
| 低液位当前触发 | 禁止 | 低液位触发，不能启动 |
| 已启用电压保护并确认电压严重异常 | 禁止 | 电压异常，不能启动 |

停止按钮：

- `STARTING`、`RUNNING`、`STOPPING`、`lost_contact` 都显示停止或状态处理入口。
- 对 `STOPPING` 重复点击停止不应生成新的业务结果，只能显示正在停止。
- 对 `lost_contact` 点击停止，只能记录用户要求停止并继续轮询，不能显示已停止。

## 配置页面矩阵

配置分为：

- 显示类配置。
- 安全相关配置。
- 计划配置。
- 校准和标准流速。

安全相关配置包括：

- 输入角色。
- 低液位配置。
- 自吸泵配置。
- 主阀配置。
- 分支水路启用和角色。
- 阀门参数。
- 通信安全参数。
- 电压检测启用状态，以及启用时的电压阈值。

规则：

| 状态 | 显示类配置 | 安全相关配置 | 保存后动作 |
|---|---|---|---|
| 从站空闲 | 允许 | 允许 | 标记配置待同步 |
| 从站运行中 | 允许 | 禁止 | 无 |
| 从站停止中 | 允许 | 禁止 | 等待停止确认 |
| 从站离线 | 允许 | 允许保存主控配置，但不能同步 | 标记待同步 |
| 从站配置未同步 | 允许 | 允许继续编辑 | 保存后仍待同步 |

保存规则：

- 保存安全相关配置后，Web 必须显示该从站配置待同步。
- 配置待同步的从站不能启动运行。
- 同步成功后才显示可运行。
- 同步失败必须保留失败原因和重试入口。

## 输入输出配置矩阵

| 操作 | 允许条件 | 禁止条件 |
|---|---|---|
| 设置流量计输入 | 从站空闲 | 从站运行中、输入与低液位冲突 |
| 设置低液位输入 | 从站空闲 | 与流量计输入重复 |
| 启用自吸泵 | 从站空闲 | 输出资源无效 |
| 设置主阀 | 从站空闲 | 与分支水路重复 |
| 修改阀门参数 | 从站空闲 | 从站运行中 |

提示：

- 流量计是必选。
- 低液位是可选。
- 自吸泵是可选。
- 自吸泵和低液位互不强制依赖。
- 同时启用自吸泵和低液位时，会启用启动前低液位检查。

## 流量页面矩阵

| 状态 | 校准流量计 | 学习标准流速 | 普通运行 |
|---|---|---|---|
| 流量计计量未校准 | 允许 | 禁止 | 允许运行，但水量估算不可用 |
| 流量计校准中 | 当前流程可停止 | 禁止 | 禁止 |
| 流量计已校准 | 允许重新校准 | 允许 | 允许 |
| 水路标准流速未学习 | 不相关 | 允许 | 允许，但低/高流量保护未启用 |
| 水路学习中 | 禁止其他同站运行 | 当前流程可停止 | 禁止同站普通运行 |
| 从站故障或锁定 | 禁止 | 禁止 | 禁止 |

重新校准提示：

- 重新校准会影响该从站所有水路的水量估算。
- Web 必须提示用户是否需要重新学习各水路标准流速。

标准流速学习提示：

- 学习只影响选中的水路。
- 学习失败不代表流量计校准失效。
- 未学习时无水保护仍有效，低/高流量保护未启用。

## 计划页面矩阵

| 状态 | 允许动作 | 禁止动作 |
|---|---|---|
| 计划未运行 | 新建、编辑、启用、禁用、删除 | 无 |
| 计划运行中 | 查看、停止本次计划、停止单个从站 | 直接修改当前运行快照 |
| 计划禁用 | 编辑、启用 | 自动触发 |
| 计划部分完成 | 查看结果、处理跳过/失败原因 | 显示为完全成功 |
| 时间无效 | 编辑计划 | 自动执行计划 |
| 电压检测未校准 | 编辑计划、执行计划 | 无；但运行提示电压保护未启用 |

修改计划：

- 修改只影响未来触发。
- 如果计划当前正在运行，Web 必须提示“是否停止当前运行”。
- 删除计划不删除历史记录。

计划结果显示：

- `completed`：全部应执行水路完成。
- `partial`：部分完成，部分跳过或失败。
- `failed`：没有水路成功启动或完成。
- `cancelled`：用户取消。

## 故障处理矩阵

| 故障 | Web 清除 | 条件 | 提示 |
|---|---|---|---|
| `LOW_LEVEL` | 允许 | 液位恢复或用户确认 | 检查水源液位 |
| `NO_FLOW_START` | 允许 | 用户确认检查水源/泵/阀/流量计 | 启动无水 |
| `NO_FLOW_RUNNING` | 允许 | 用户确认检查水路 | 运行中断流 |
| `HIGH_FLOW` | 允许 | 用户确认检查破管/脱管 | 高流量锁定 |
| `LOW_FLOW` | 允许 | 按策略 | 低流量 |
| `COMM_TIMEOUT` | 允许 | 通信恢复或从站处于安全状态 | 通信丢失本地停机 |
| `VOLTAGE_LOW` | 允许 | 电压恢复 | 供电电压低 |
| `VOLTAGE_HIGH` | 允许 | 用户确认电源正常 | 供电电压高 |
| `CONFIG_INVALID` | 禁止普通清除 | 重新同步配置 | 配置无效 |
| 疑似地址冲突 | 禁止普通清除 | 重新扫描并排除冲突 | 检查拨码地址 |

规则：

- 清除故障不删除历史记录。
- 清除故障不自动启动运行。
- 清除失败必须显示具体原因。

## 维护页面矩阵

| 动作 | 允许条件 | 说明 |
|---|---|---|
| 扫描从站 | 总线空闲或无高优先级运行轮询 | 不修改从站地址 |
| 从站识别 | 从站在线，非运行中优先 | 可用状态灯辅助现场确认 |
| 重新同步配置 | 从站在线且非运行中 | 同步失败禁止运行 |
| 停止全部 | 任意存在活动运行 | 高影响动作，二次确认 |
| 导出记录 | 任意 | 不影响运行 |
| 恢复业务配置 | 无活动运行 | 高影响动作，恢复后相关从站待同步 |

维护页面不支持：

- 修改从站拨码地址。
- 任意 RS485 帧透传。
- 直接操作单个 GPIO。
- 绕过配置同步启动运行。

## 状态过期规则

状态过期指主控缓存的从站状态超过允许时间未更新。

Web 表达：

- 显示最后更新时间。
- 显示“状态过期”。
- 禁止启动新运行。
- 禁止清除依赖实时条件的故障。

允许：

- 查看最后状态。
- 执行扫描。
- 查看记录。

禁止：

- 启动。
- 同步配置。
- 判断故障已经恢复。

## 文案语义要求

Web 文案必须区分：

- “已发停止命令”与“从站已确认停止”。
- “从站离线”与“通信丢失后本地安全停机”。
- “故障已清除”与“问题已修复”。
- “水路跳过”与“水路运行失败”。
- “标准流速未学习”与“流量计未校准”。
- “配置已保存”与“从站配置已同步”。

## 不支持的 Web 操作

当前产品 Web 不支持：

- 任意 RS485 帧透传。
- 通用寄存器编辑。
- 从站主动上报配置页面。
- 一个从站多个流量计配置。
- 一个从站多水路并行启动。
- 未认证用户执行运行、配置、清除或维护动作。
- 通过 Web 修改从站拨码地址。
