# 08 验证验收清单

## 验收范围

| 类别 | 目标 |
|---|---|
| 基础库能力 | 确认 `Esp32Base` 和 `Stc8hBase` 能支撑所需能力 |
| 单板硬件 | 确认主控板、从站板启用资源可用 |
| 通信协议 | 确认 RS485 帧、CRC、超时、重试和多地址管理正确 |
| 配置同步 | 确认版本、CRC、配置拒绝和重新同步正确 |
| 运行控制 | 确认手动运行、计划运行、停止和状态跟踪正确 |
| 安全保护 | 确认低液位、无水、高流量、通信丢失、运行超时保护可靠 |
| 电压监测 | 确认电压显示、校准、告警和记录可靠，且不误触发停机 |
| Web 交互 | 确认 Web 覆盖所有业务操作 |
| 长时间运行 | 确认稳定性、记录、恢复和无误动作 |

## 硬件启用验证

主控板：

- 12V 输入、5V、3.3V 正常。
- ESP32 启动和复位正常。
- RS485 发送、接收、方向控制正常。
- LCD2004 可显示。
- SHT30 可读取。
- DS3231 可读写时间。
- 主控按键和两个状态灯可用。
- ADS1115、FRAM、主控侧阀输出、干接点、预留输入不被初始化为业务功能。

从站板：

- STC8H 启动和复位后全部输出关闭。
- RS485 通信正常。
- 拨码地址读取正确。
- `IN1..IN4` 可按配置识别。
- 8 路阀输出可单独控制。
- `DRV_SD` 安全关闭有效。
- 干接点输出可控制外接泵继电器。
- 系统电压 ADC 可读取并校准。
- 状态灯和维护按键可用。
- 预留 I2C、预留 ADC、`SPARE_*` 不进入业务逻辑。

## 配置验证

必须验证：

- 参数默认值、最小值、最大值、单位和字段关系符合 `02-配置模型与参数边界.md`。
- 一个从站必须有且只有一个流量计输入。
- 低液位最多一个，可关闭。
- 自吸泵可关闭。
- 主阀最多一个，可关闭。
- 至少一路分支水路启用。
- 同一路输入不能同时配置为流量计和低液位。
- 主阀不能同时作为分支水路。
- 配置版本和 CRC 可校验。
- 配置未同步时禁止相关从站运行。
- 增强能力未校准或未学习不阻断运行，只提示保护不完整。
- 非法参数不能保存，不能下发到从站，也不能被运行命令绕过。

## 通信验证

必须验证：

- `PING`、`GET_INFO`、`GET_STATUS`、`GET_CONFIG_SUMMARY`。
- `SET_SAFE_CONFIG`、`GET_SAFE_CONFIG`。
- `START_ZONE`、`STOP`、`CLEAR_FAULT`。
- `GET_STATUS` 能返回状态、运行 ID、活动水路、停止原因、故障位、输入/输出位图、脉冲计数和电压状态。
- CRC 错误不执行动作。
- `SEQ` 不匹配不被当成有效应答。
- 地址不匹配不被当成有效应答。
- 超时重试有效。
- 多从站轮询稳定。
- 疑似地址冲突能被标记并禁止运行。
- 普通灌溉运行的 `START_ZONE` 不允许关闭无水保护。
- `STOP` 对 `run_id=0` 和重复停止请求保持幂等。
- `CLEAR_FAULT.flags` 未定义位非 0 时必须拒绝。

## 运行验证

手动运行：

- 单从站单水路启动、运行、停止。
- 其他从站运行中仍可启动当前从站。
- 同一从站已有运行时拒绝第二个运行。
- 用户停止和本地保护停机的停止原因不混淆。

计划运行：

- 同一从站多水路按顺序运行。
- 多从站可并行运行。
- 同一从站冲突时后触发计划项跳过并记录。
- 全部跳过记录为 `skipped`，不记录为 `failed`。
- 错过执行不补跑，只记录事件。

## 安全验证

必须验证：

- 上电、复位、看门狗复位后所有输出关闭。
- 通信超时后从站本地安全停机。
- 运行超时后从站本地安全停机。
- 低液位触发后停机。
- 启动无水停机。
- 运行中无水停机。
- 标准流速已学习且高流量确认后停机。
- 电压低/高告警只提示和记录，不自动停机、不禁止启动。
- 从站进入 `OFFLINE_LOCAL_SAFE` 后不自动恢复运行。
- 清除故障后不自动启动。

## 流量验证

必须验证：

- 流量计脉冲计数。
- 计量校准流程。
- 计量未校准时允许运行并提示水量估算不可用或仅供参考。
- 标准流速学习。
- 标准流速未学习时允许运行并提示低/高流量保护未启用。
- 无水保护不依赖标准流速。
- 低流量策略按配置处理。
- 高流量默认停机并锁定。
- 待机漏水可生成告警。

## Web 验证

必须验证：

- 首页能显示系统、从站、运行和告警摘要。
- 手动运行能启动、监控、停止。
- 从站页面能同步配置、识别从站、清除故障。
- 水路页面能配置 8 路输出、主阀和分支水路。
- 输入输出页面只覆盖 `IN1..IN4`、自吸泵和电压监测。
- 流量页面能完成校准和标准流速学习。
- 计划页面能创建、编辑、启停和查看结果。
- 告警与记录页面能查看、确认和导出记录。
- Web API 能保存系统、从站、水路、输入、流量、自吸泵和计划配置。
- 配置类 API 的校验结果和 Web 页面一致。
- 从站安全配置修改后能正确标记待同步。
- Web 不提供通用 IO、通用寄存器、任意 RS485 帧透传。

## 长时间运行验证

第一版交付前至少完成：

- 24 小时空闲轮询。
- 24 小时单从站周期运行。
- 24 小时多从站并行计划运行。

失败条件：

- 主循环卡死。
- 误动作打开输出。
- 配置丢失。
- 记录写入失控。
- 通信恢复后自动误启动。
- 清除故障后自动误启动。

## 量产前工程验证

量产前必须完成：

- 反接验证。
- 过流验证。
- 典型浪涌验证。
- 阀门感性负载验证。
- 电源跌落验证。
- 高低温边界验证。

这些属于硬件量产验证，不要求软件增加超出当前硬件范围的功能；软件必须在异常复位时保持输出关闭，电压监测异常只生成告警和记录。
