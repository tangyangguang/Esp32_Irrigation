## 核心原则

- 核心能力决定系统能否执行最基本的灌溉运行。
- 增强能力用于提前发现异常、提升计量精度、改善维护体验。
- 增强能力未完成配置、学习或校准时，不应成为核心运行的阻断条件。
- 增强能力只有在已经检测到明确危险条件时，才可以阻断运行或触发停机。
- Web 必须清楚提示哪些增强保护未启用，但不能把“未启用增强保护”误写成“系统不可运行”。

## 核心能力

以下属于最小可运行核心能力：

| 能力 | 为什么是核心 |
|---|---|
| 从站在线或可通信 | 主控必须能下发启动/停止并读取基本状态 |
| 从站地址无冲突 | 避免命令发给错误设备 |
| 从站安全配置有效且已同步 | 从站必须知道输入输出映射和安全关闭规则 |
| 目标分支阀启用且未锁定 | 必须知道打开哪一路水路 |
| 同一从站当前空闲 | 一个从站内部不支持多水路并行 |
| 低液位未触发 | 如果配置了低液位且当前触发，属于明确危险条件 |
| 基础运行时长合法 | 防止无边界运行 |
| 通信超时安全停机策略有效 | 主控失联时从站必须能本地关闭 |
| 运行超时安全停机策略有效 | 防止异常长时间放水 |

这些条件不满足时，可以禁止启动。

## 增强能力

以下属于增强能力：

| 能力 | 未完成时的降级策略 |
|---|---|
| 标准流速学习 | 不启用低/高流量保护，仍允许运行 |
| 流量计 `pulses_per_liter` 校准 | 不提供准确水量估算，仍可用脉冲有无做无水判断 |
| 电压检测校准 | 不启用电压告警/停机保护，仍允许运行 |
| 阀门 PWM 降功率保持 | 使用 100% 保持，不阻断运行 |
| SHT30 温湿度 | 不显示或标记传感器不可用，不影响灌溉 |
| LCD 本地显示 | Web 仍可完整操作，不影响灌溉 |
| RTC 高级告警/中断 | 主控时间有效即可，RTC 中断不是运行前提 |
| 待机漏水增强判断 | 未启用时不做该告警，不影响正常启动 |

## 明确危险条件

增强能力一旦已经启用并检测到明确危险条件，可以阻断或停机。

| 条件 | 处理 |
|---|---|
| 低液位已触发 | 禁止启动或运行中停机 |
| 已校准电压检测发现严重低压/高压 | 禁止启动或运行中停机 |
| 已学习标准流速后检测到高流量 | 默认停机并锁定 |
| 已配置无水检测后启动/运行中无脉冲 | 默认停机并锁定 |
| 待机漏水已确认 | 默认禁止该从站新运行，等待用户确认 |

规则：

- “未校准”不是危险条件。
- “未学习”不是危险条件。
- “未启用增强能力”不是危险条件。
- “已启用且检测到异常”才是危险条件。

## 运行准入分层

启动运行前，主控按三层判断。

### 第一层：硬阻断

满足任一条件时禁止启动：

- 从站禁用。
- 从站离线。
- 地址疑似冲突或总线异常。
- 配置无效或未同步。
- 从站已有活动运行。
- 从站处于故障、锁定、未配置、通信丢失安全停机。
- 目标水路禁用或锁定。
- 低液位当前触发。
- 请求运行时长非法。

### 第二层：增强保护未就绪

满足以下条件时允许启动，但必须提示保护不完整：

- 标准流速未学习。
- 流量计计量校准未完成。
- 电压检测未校准。
- 阀门 PWM 降功率未验证，使用 100% 保持。

### 第三层：增强保护已触发异常

满足以下条件时禁止启动或停机：

- 电压检测已校准且电压异常。
- 标准流速已学习且高流量确认。
- 待机漏水确认。
- 低液位确认。

## 记录和提示要求

增强能力未就绪时，运行记录必须标记：

- 哪些增强保护未启用。
- 哪些计量值不可用或仅供参考。
- 本次运行是否缺少低/高流量保护。
- 本次运行是否缺少电压保护。

Web 提示语义：

- “标准流速未学习：本次运行不启用低/高流量保护。”
- “流量计未校准：本次运行水量估算不可用，但仍可根据脉冲判断是否有流量。”
- “电压检测未校准：本次运行不启用电压保护。”
- “保护未启用”不能写成“设备故障”。

## 对现有方案的修正结论

以下策略需要覆盖旧结论：

- 标准流速未学习：允许手动和计划运行，只是不启用低/高流量保护。
- 流量计计量校准未完成：允许运行，但水量估算不可用；是否能做无水判断取决于是否能检测脉冲。
- 电压检测未校准：允许运行，只是不启用电压告警和电压停机保护。
- 增强能力未就绪不生成阻断性告警，只生成保护能力提示。

## 仍需进一步评估的问题

多计划同一从站冲突策略仍需按成熟产品原则再评估：

- 不能过度设计成复杂排队系统。
- 不能让后触发计划无限排队导致执行时间不可解释。
- 也不能简单跳过导致用户误以为计划已经完整执行。

下一步应单独评估计划冲突的默认策略、用户提示和记录方式。
