## 验证目标

- 确认主控板和从站板当前产品启用的硬件资源可用。
- 确认未启用的预留硬件不会被软件误用。
- 确认 ESP32 主控固件能基于 `Esp32Base` 稳定运行。
- 确认 STC 从站固件能基于 `/Users/tyg/dir/codex_dir/Stc8hBase` 稳定运行。
- 确认 RS485 协议在多个从站、并行运行和异常情况下行为可控。
- 确认从站任何异常路径都能安全关闭阀、主阀和自吸泵。
- 确认 Web 能完成全部配置、操作、维护、告警和记录查看。

## 验证分层

测试按以下层级推进：

| 层级 | 目标 |
|---|---|
| 基础库能力验证 | 确认方案依赖的 `Esp32Base` 和 `Stc8hBase` 通用能力可用 |
| 单板硬件验证 | 确认主控板、从站板启用资源可用 |
| 模块级验证 | 确认配置、协议、状态机、输入输出驱动正确 |
| 主从通信验证 | 确认 RS485 命令、重试、CRC、超时和多地址管理正确 |
| 系统功能验证 | 确认 Web、调度、计划、校准、学习、记录可用 |
| 安全验证 | 确认低液位、无水、通信丢失、电压异常等保护可靠 |
| 长时间运行验证 | 确认稳定性、记录、存储和恢复策略可接受 |

## ESP32 主控验证

基础验证：

- 固件能基于 `Esp32Base` 编译。
- Web 认证、基础页面、系统诊断和 OTA 能正常工作。
- 业务页面使用 `Esp32Base` 的 Web/UI 基线，不另起独立 Web 框架。
- 主控侧能可靠保留主控系统配置、从站业务配置、计划、校准参数。
- 主控侧能可靠保留事件记录、运行记录和导出数据。

硬件验证：

- 12V 输入、5V、3.3V 电源稳定。
- ESP32 启动稳定。
- RS485 收发正常。
- 主控按键可读取。
- 两个软件状态灯可控制。
- LCD2004 能完成最小状态显示。
- SHT30 能读取温湿度。
- DS3231 RTC 能读取和校时。

不启用项检查：

- 不读取 ADS1115。
- 不使用主控板 ADC、电压检测、NTC、模拟输入。
- 不使用主控板流量/传感器输入。
- 不使用主控板 2 路 NMOS 阀输出。
- 不使用主控板干接点继电器。
- 不依赖 FRAM。

通过标准：

- 主控连续重启后配置不丢失。
- 网络异常不影响本地 RS485 调度主循环持续运行。
- Web 操作不会阻塞正在运行从站的轮询。
- 主控掉电重启后不恢复上次运行命令。

## STC 从站验证

基础验证：

- 固件能基于 `/Users/tyg/dir/codex_dir/Stc8hBase` 编译。
- GPIO、UART、定时器、PWM、ADC、持久化、看门狗、CRC 等基础能力接入正确。
- 未使用模块不编译、不初始化、不占用资源。

硬件验证：

- 拨码地址读取正确，地址范围 `1..15`。
- RS485 收发正常。
- `IN1..IN4` 均可读取数字输入。
- 流量计输入能稳定计数。
- 低液位输入能按有效电平和确认时间判断。
- 系统电压 ADC 读数与实测电压误差在可接受范围内。
- 8 路阀输出可独立控制。
- 阀门全功率吸合和 PWM 保持可工作。
- 自吸泵干接点输出可控制。
- 3 个软件状态灯可显示通信、运行、故障状态。
- 本地维护/识别按键可读取。

通过标准：

- 上电、复位、看门狗复位后全部输出默认关闭。
- 无有效配置时不允许运行。
- 通信丢失后能按 `comm_timeout_ms` 本地安全停机。
- 从站本地状态机不会因为 RS485 收发阻塞流量计计数。

## RS485 通信验证

基础协议：

- 帧头、版本、地址、序号、命令、长度、CRC 校验正确。
- CRC 错误帧被丢弃。
- 长度异常帧被丢弃。
- 地址不匹配帧不响应。
- 未知命令返回明确错误。

命令验证：

- `PING`
- `GET_INFO`
- `GET_STATUS`
- `GET_CONFIG_SUMMARY`
- `SET_SAFE_CONFIG`
- `GET_SAFE_CONFIG`
- `START_ZONE`
- `STOP`
- `CLEAR_FAULT`
- `FACTORY_RESET_CONFIG`

多从站验证：

- 地址 `1..15` 能被扫描。
- 疑似地址冲突或总线冲突能被主控标记，并禁止相关地址运行。
- 多从站轮询不会串包。
- 一个从站超时不会阻塞其他从站持续轮询。
- 多个从站同时运行时，主控能按优先级轮询运行中的从站。

通过标准：

- 默认 `19200 bps` 下通信稳定。
- 在典型线长、典型从站数量下，运行状态刷新满足页面显示需要。
- 发生连续通信失败时，主控标记异常，从站本地超时停机。

## 配置验证

主控配置：

- 从站列表、名称、水源名称可保存。
- 水路名称、启用状态、默认时长可保存。
- 输入输出映射可保存。
- 流量计校准参数可保存。
- 标准流速可保存。
- 计划可保存。

从站安全配置：

- 必须有且只有一个流量计输入。
- 低液位最多一个。
- 自吸泵可选。
- 主阀可选且最多一个。
- 至少一路分支水路启用。
- 电压阈值、超时时间、阀门 PWM 参数可下发。
- 配置版本和 CRC 可校验。

异常配置：

- 重复输入角色应拒绝保存。
- 主阀和分支水路复用同一路输出应拒绝保存。
- 没有流量计应拒绝运行。
- 流量计计量未校准时应允许运行，但水量估算不可用或仅供参考；能检测脉冲时仍应验证无水判断。
- 配置未同步应拒绝运行。

## 运行状态机验证

启动流程：

- 清零脉冲计数。
- 如果启用自吸泵和低液位，先确认低液位未触发。
- 按配置启动自吸泵。
- 按配置启动主阀。
- 按配置启动分支阀。
- 阀门完成全功率吸合后进入 PWM 保持。
- 进入 `RUNNING` 后能持续更新流量、时间、输出状态。

停止流程：

- 用户停止。
- 计划时间结束。
- 通信超时。
- 低液位触发。
- 首次无水。
- 运行中无水。
- 电压严重异常。
- 运行超时。

通过标准：

- 任意停止原因下，分支阀、主阀和自吸泵关闭顺序符合配置。
- 停止原因不被主控错误覆盖。
- 通信恢复后不会自动继续上次运行。
- 故障清除前，不允许运行被锁定对象。

## Web/API 验证

页面验证：

- 首页能显示系统状态、从站状态、运行状态和告警摘要。
- 手动运行能启动、监控、停止水路。
- 从站页面能管理启用状态、名称、水源名称、同步配置、识别从站、清除故障。
- 水路页面能配置 8 路输出、主阀和分支水路。
- 输入输出页面能配置流量计、低液位、自吸泵、阀门 PWM 和电压阈值。
- 流量页面能完成流量计校准和标准流速学习。
- 计划页面能创建、编辑、启停和执行计划。
- 告警与记录页面能查看、确认和导出记录。
- 维护页面能扫描从站、重新同步、停止全部运行。

API 验证：

- 所有修改配置和运行控制 API 都需要认证。
- API 不输出 Web Auth 密码、Wi-Fi 密码。
- 运行控制 API 能返回稳定错误码。
- 配置保存失败、同步失败、从站离线、从站忙等状态能被页面正确显示。
- 页面刷新使用主控缓存状态，不直接访问 RS485。

## 安全验证

必须验证的安全场景：

- 主控断电。
- 从站断电。
- RS485 A/B 断线。
- RS485 干扰导致 CRC 错误。
- 从站通信连续超时。
- 低液位启动前触发。
- 低液位运行中触发。
- 自吸泵启用但低液位未配置。
- 低液位配置但自吸泵未配置。
- 流量计无脉冲。
- 流量计脉冲异常偏低。
- 流量计脉冲异常偏高。
- 阀门输出运行中停止命令。
- 电压低告警。
- 电压严重异常停机。
- 看门狗复位。
- 主控 Web 页面重复点击启动。
- 多个从站同时运行时，其中一个从站故障。

通过标准：

- 安全问题默认只影响对应从站，不停止其他独立水源的从站。
- 任何从站异常路径都不能保持阀门或自吸泵错误开启。
- 主控不能对离线、未同步、故障、锁定、未配置流量计输入或已触发明确安全阻断条件的从站启动运行。
- 流量计计量未校准、标准流速未学习、电压检测未校准不属于启动阻断条件；验证时应确认系统允许运行并提示对应保护或计量能力未启用。
- 用户能在 Web 上看到可解释的故障原因和处理入口。

## 长时间运行验证

建议验证：

- 多从站空闲轮询 24 小时。
- 单从站周期运行 24 小时。
- 多从站并行计划运行 24 小时。
- Web 长时间打开运行页。
- 事件记录持续写入和覆盖。
- 断电重启后配置、计划、校准参数保留。
- RTC 时间在断网情况下仍可用。

通过标准：

- 不出现主循环卡死。
- 不出现从站误动作。
- 不出现运行记录错乱。
- 不出现配置丢失。
- 不出现存储写入失控。

## 实现依据和验收补充

- 系统电压误差、电压换算、阀门吸合、PWM 保持、流量计校准和标准流速学习采样见 `16-参数默认值与校验范围.md`。
- 长时间运行测试第一版按本文 24 小时空闲轮询、24 小时单从站周期运行、24 小时多从站并行计划运行执行；如果 24 小时内出现任何主循环卡死、误动作、配置丢失、存储写入失控，不能进入交付。
- 量产前必须至少完成反接、过流、典型浪涌、阀门感性负载、电源跌落和高低温边界的工程验证；这些属于硬件量产验证，不要求软件增加超出当前硬件范围的功能，但软件必须在异常复位和电压异常时保持输出关闭。
