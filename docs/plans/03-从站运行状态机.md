# 03 从站运行状态机

状态：成熟产品状态机草案。本文定义从站在上电、配置、空闲、运行、停机、故障和通信异常时的行为边界。

关联文档：

- `01-新版硬件启用范围与系统边界.md`
- `02-RS485通讯协议设计草案.md`
- `11-统一枚举与错误码表.md`

## 设计目标

- 从站负责一个真实水源的本地执行和安全保护。
- 从站任何时候都不能因为主控离线、通信丢失或自身复位而保持阀门或泵持续打开。
- 从站只接受主控命令启动运行；不会自行恢复上一次运行。
- 从站运行中需要独立检测通信超时、运行超时、低液位、流量异常和电压异常。
- 从站状态机必须简单、可验证、可在 STC8H8K64U 上稳定实现。

## 状态定义

从站状态沿用通讯协议中的状态码。

| 状态 | 含义 | 输出要求 |
|---|---|---|
| `BOOTING` | 上电或复位后的初始化阶段 | 全部输出关闭 |
| `UNCONFIGURED` | 无有效安全配置或配置校验失败 | 全部输出关闭 |
| `IDLE` | 空闲，配置有效，可接收运行命令 | 全部输出关闭 |
| `STARTING` | 已接受运行命令，正在按顺序启动泵、主阀和分支阀 | 只允许按启动步骤打开输出 |
| `RUNNING` | 分支水路正在运行 | 允许当前运行相关输出保持打开 |
| `STOPPING` | 正在执行关闭顺序 | 按停止步骤关闭输出 |
| `FAULT` | 出现故障，但可能可恢复 | 全部输出关闭 |
| `LOCKED` | 安全锁定，主控处理前不能运行 | 全部输出关闭 |
| `OFFLINE_LOCAL_SAFE` | 运行中通信超时，从站已本地安全停机 | 全部输出关闭 |

## 上电流程

从站上电或复位后必须执行：

1. 立即关闭所有阀输出、主阀输出和自吸泵干接点输出。
2. 初始化 GPIO、定时器、ADC、UART、RS485 方向控制和状态灯。
3. 读取 4 位拨码地址。
4. 读取本地保存的安全配置。
5. 校验安全配置版本、CRC 和字段合法性。
6. 配置有效则进入 `IDLE`。
7. 配置无效则进入 `UNCONFIGURED`。

上电后不得恢复复位前的运行状态。

## 配置校验规则

从站收到 `SET_SAFE_CONFIG` 后，需要校验：

- 从站地址匹配。
- 配置 CRC 正确。
- 流量计输入必须在 `IN1..IN4` 中。
- `IN1..IN4` 中只能有一路配置为流量计。
- 低液位输入可为 `0`，表示不启用。
- 低液位输入启用时，不能和流量计输入重复。
- 自吸泵输出可不启用。
- 自吸泵输出启用时，不强制要求低液位输入也启用。
- 低液位输入启用时，不强制要求自吸泵输出也启用。
- 主阀可为 `0`，表示无主阀。
- 主阀启用时，必须在 `VALVE1..VALVE8` 中。
- 分支水路启用位不能和主阀重复。
- 至少需要有一路分支水路可用，否则拒绝运行配置。
- 通信超时时间必须在允许范围内。
- 单次最大运行时间必须在允许范围内。
- 阀门吸合时间、保持比例、PWM 频率必须在允许范围内。

配置校验失败时：

- 不保存新配置。
- 保持原配置不变。
- 返回对应错误码。
- 如果原配置也无效，保持 `UNCONFIGURED`。

## 空闲状态

`IDLE` 状态下：

- 所有阀输出关闭。
- 自吸泵干接点关闭。
- 脉冲计数器可保持最近一次运行值，也可在新运行开始时清零。
- 继续检测输入状态，用于状态上报。
- 可以响应 `PING`、`GET_STATUS`、`GET_INFO`、`SET_SAFE_CONFIG`、`START_ZONE`、`CLEAR_FAULT`。

空闲状态下如果检测到流量脉冲：

- 记录为待机流量事件。
- 是否立即锁定由主控配置和后续流量策略决定。
- 从站可以在 `fault_flags` 中上报待机流量相关标志，具体标志后续在流量策略文档中定义。

## 启动运行

从站收到 `START_ZONE` 后，只有满足以下条件才接受：

- 当前状态为 `IDLE`。
- 本地安全配置有效。
- `config_version` 与本地配置一致。
- `zone_index` 对应的分支水路已启用。
- `zone_index` 没有被配置为主阀。
- 没有不可恢复故障或安全锁定。
- 请求运行时长不超过从站本地最大运行时间。

接受后进入 `STARTING`。

启动顺序：

1. 清零本次运行脉冲计数。
2. 记录 `run_id`、`zone_index`、计划停止时间。
3. 如果同时启用自吸泵和低液位输入，先确认低液位未触发。
4. 如果启用自吸泵，打开自吸泵干接点。
5. 如果配置主阀，打开主阀。
6. 等待主阀启动延时。
7. 打开分支阀。
8. 分支阀先全功率吸合 `valve_pull_in_ms`。
9. 切换到 PWM 保持。
10. 进入 `RUNNING`。

如果启动过程中出现低液位、配置异常、输出异常或主控停止命令：

- 进入 `STOPPING`。
- 按安全关闭顺序关闭输出。
- 上报对应停止原因。

## 运行状态

`RUNNING` 状态下，从站持续执行：

- 维护当前分支阀 PWM 保持。
- 如果配置主阀，保持主阀打开。
- 如果启用自吸泵，保持干接点输出。
- 统计流量计脉冲。
- 计算实时流速。
- 检测低液位输入。
- 检测系统电压。
- 检测运行超时。
- 检测通信超时。
- 响应主控 `GET_STATUS` 和 `STOP`。

运行中不接受新的 `START_ZONE`。如果收到新的 `START_ZONE`，返回 `ERR_BUSY`。

## 停止顺序

进入 `STOPPING` 后必须按安全顺序关闭：

1. 关闭当前分支阀 PWM 输出。
2. 关闭当前分支阀。
3. 如果配置主阀，等待分支阀关闭延时后关闭主阀。
4. 如果启用自吸泵，等待泵关闭延时后关闭自吸泵干接点。
5. 确认所有输出关闭。
6. 保存本次停止原因和最近运行摘要。
7. 根据停止原因进入 `IDLE`、`FAULT`、`LOCKED` 或 `OFFLINE_LOCAL_SAFE`。

停止原因和目标状态：

| 停止原因 | 目标状态 |
|---|---|
| `DURATION_DONE` | `IDLE` |
| `USER_STOP` | `IDLE` |
| `LOW_LEVEL` | `FAULT` 或 `LOCKED`，由策略决定 |
| `NO_FLOW_START` | `LOCKED`，除非策略明确允许只告警 |
| `NO_FLOW_RUNNING` | `LOCKED`，除非策略明确允许只告警 |
| `HIGH_FLOW` | `LOCKED`，除非策略明确允许只告警 |
| `LOW_FLOW` | `FAULT` 或 `IDLE`，由策略决定 |
| `COMM_TIMEOUT` | `OFFLINE_LOCAL_SAFE` |
| `RUN_TIMEOUT` | `FAULT` |
| `VOLTAGE_LOW` | `FAULT` 或 `LOCKED`，由严重程度决定 |
| `VOLTAGE_HIGH` | `FAULT` 或 `LOCKED`，由严重程度决定 |
| `CONFIG_INVALID` | `UNCONFIGURED` 或 `FAULT` |
| `WATCHDOG_RESET` | 上电后重新进入 `BOOTING`，并保留最近停止原因 |
| `LOCAL_FAULT` | `FAULT` 或 `LOCKED`，由严重程度决定 |
| `UNKNOWN` | `FAULT` |

## 通信超时

通信超时只在从站处于 `STARTING` 或 `RUNNING` 时触发安全停机。

规则：

- 从站记录最近一次有效主控请求时间。
- 有效请求包括发给本地址、CRC 正确、`SEQ/CMD` 合法的请求。
- 超过 `comm_timeout_ms` 未收到有效请求，触发 `COMM_TIMEOUT`。
- 触发后立即进入 `STOPPING`。
- 停止完成后进入 `OFFLINE_LOCAL_SAFE`。
- 通信恢复后，不自动继续运行。
- 主控需要重新读取状态、记录事件，并按策略重新下发运行命令。

## 低液位

低液位输入可选。

启用低液位时：

- 从站按配置的输入号读取 `IN1..IN4`。
- 支持有效电平配置。
- 需要确认时间，避免抖动误判。
- 低液位确认成立后，立即进入 `STOPPING`，停止原因为 `LOW_LEVEL`。
- 如果同时启用自吸泵，低液位保护同时用于自吸泵启动前检查和运行中保护。

低液位未配置时：

- 从站不做低液位保护。
- 状态包中仍可上报输入原始状态。

## 流量计和流量异常

一个从站只支持一个流量计。

运行中从站至少负责：

- 统计累计脉冲。
- 按窗口计算实时流速。
- 判断是否长时间无脉冲。
- 上报累计脉冲和实时流速。

流量异常策略：

- 无水或无脉冲：从站可以直接触发停机，因为它属于本地安全条件。
- 高流量：可由从站根据主控下发参数触发停机，避免主控轮询延迟。
- 低流量：可按策略只告警或停机。
- 待机流量：空闲时检测到脉冲，应上报主控，默认由主控生成水源级告警并禁止新运行，具体规则见 `20-流量计校准与流量保护算法.md`。

## 电压检测

从站需要检测系统电压。

电压检测规则：

- 电压值随 `GET_STATUS` 上报。
- 低于最低安全电压时，不允许启动新运行。
- 运行中电压严重异常时，按阈值触发 `VOLTAGE_LOW` 或 `VOLTAGE_HIGH` 并停机。
- 电压阈值应作为配置项保存到从站安全配置中。

## 故障和锁定

`FAULT` 表示出现故障，但可能可以由主控清除。

`LOCKED` 表示安全锁定，需要主控明确处理，不允许直接运行。

锁定场景：

- 高流量停机。
- 无水停机且策略要求锁定。
- 配置冲突或配置损坏。
- 多次运行中反复出现同类安全故障。

可恢复故障可以通过 `CLEAR_FAULT` 清除，但清除前从站必须满足：

- 当前所有输出关闭。
- 低液位不再触发。
- 电压恢复正常。
- 配置有效。
- 主控命令 CRC 正确且地址匹配。

## 本地按键

从站产品默认只使用 1 个本地按键，另 1 个按键保留。

默认用途：

- 短按：进入从站识别模式，用状态灯显示当前从站地址和通信状态，方便现场确认是哪一块从站板。
- 运行中短按：只显示状态，不改变运行。
- 长按：进入维护确认流程，可用于清除可恢复故障或退出 `OFFLINE_LOCAL_SAFE`。
- 长按必须有确认动作，例如持续按住到故障灯快闪完成，避免误触。

本地按键不得直接启动阀门运行，避免现场误触导致放水。

## 状态灯

从站使用 3 个软件状态灯。

默认定义：

- 通信灯：主控轮询正常时闪烁。
- 运行灯：有阀或泵输出时亮或闪烁。
- 故障灯：`FAULT`、`LOCKED`、`OFFLINE_LOCAL_SAFE` 时亮或按模式闪烁。

另 1 个常亮电源灯由硬件供电指示，不进入软件状态机。

## 不允许的行为

- 从站上电后自动恢复上次运行。
- 通信恢复后自动继续通信丢失前的运行。
- 本地按键直接启动灌溉。
- 配置无效时允许启动阀门。
- 一个输入同时作为流量计和低液位。
- 一个阀输出同时作为主阀和分支阀。
- 从站运行中接受第二个运行命令覆盖当前运行。

## 方案依据和验证边界

- 主阀启动延时、关闭延时、自吸泵延时、流量异常阈值和电压阈值字段见 `13-RS485详细Payload与主控重试规则.md`。
- 流量计校准、标准流速学习、无水、低流量、高流量和待机漏水规则见 `20-流量计校准与流量保护算法.md`。
- 参数默认值、校验范围、电压换算实测步骤和测试前保守策略见 `16-参数默认值与校验范围.md`。
- `CLEAR_FAULT` 可清除和不可清除故障列表见 `17-故障清除与本地维护策略.md`。
- 状态灯属于从站本地维护体验：通信灯表示有效轮询，运行灯表示有输出，故障灯表示 `FAULT`、`LOCKED`、`OFFLINE_LOCAL_SAFE`。具体闪烁频率属于实施阶段设计，不能作为业务状态判断的唯一依据。
