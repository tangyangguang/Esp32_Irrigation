# 05 Web 交互模型

状态：成熟产品 Web 交互模型草案。本文定义主控侧 Web 页面、操作流程和维护入口。LCD2004 和本地按键只作为最小现场辅助，不承载完整产品操作。

关联文档：

- `01-新版硬件启用范围与系统边界.md`
- `02-RS485通讯协议设计草案.md`
- `03-从站运行状态机.md`
- `04-配置模型.md`

## 设计目标

- Web 是完整配置、操作、维护、故障处理和记录查看入口。
- 所有产品逻辑都必须能通过 Web 完成。
- LCD2004 只做最小范围的本地状态显示。
- 主控按键只做最小本地辅助操作，不替代 Web 配置。
- 主控通过 RS485 管理从站，所有从站状态需要能在 Web 上被理解和处理。
- 交互设计要面向成熟产品，避免把调试页面直接变成用户页面。

## 主控交互职责

主控负责：

- 提供 Web 页面和 API。
- 管理从站列表和配置。
- 执行自动计划和手动运行。
- 同步从站安全配置。
- 读取从站状态并展示。
- 记录运行、故障、校准、配置变更事件。
- 驱动 LCD2004 显示最小现场状态。
- 读取主控按键。
- 读取 SHT30 温湿度。
- 使用 RTC 提供可靠时间。

主控不负责：

- 直接驱动从站阀输出。
- 在通信丢失后继续假定从站仍在运行。
- 用 LCD 完成配置。
- 用本地按键修改水路、输入输出、协议或计划配置。

## Web 页面结构

Web 页面建议按以下模块组织：

| 页面 | 职责 |
|---|---|
| 首页 | 总览设备状态、当前运行、告警、从站在线状态 |
| 手动运行 | 选择从站、水路、运行时长，启动或停止 |
| 水源/从站 | 管理从站、地址、在线状态、水源名称 |
| 水路 | 配置主阀、分支水路、启用状态、名称、默认时长 |
| 输入输出 | 配置 `IN1..IN4` 的流量计/低液位/预留占位，以及自吸泵 |
| 流量 | 流量计校准、标准流速学习、流量保护参数 |
| 计划 | 自动计划、顺序、跳过策略、启用状态 |
| 告警与记录 | 查看运行记录、故障记录、配置变更和维护记录 |
| 系统设置 | 网络、时间、SHT30、RS485、备份恢复 |
| 维护 | 从站识别、清除故障、重新同步配置、诊断状态 |

## 首页

首页显示：

- 当前时间和 RTC 状态。
- 网络状态。
- SHT30 温湿度。
- 当前运行状态。
- 当前运行从站、水源和水路。
- 当前流速、累计水量和运行时长。
- 从站在线数量。
- 未处理告警数量。
- 最近一次故障摘要。

首页动作：

- 跳转到手动运行。
- 停止当前运行。
- 查看当前故障。
- 查看从站状态。

首页不做复杂配置。

## 手动运行

手动运行流程：

1. 选择从站或水源。
2. 选择已启用且未锁定的分支水路。
3. 输入或使用默认运行时长。
4. 主控检查从站在线、配置同步、故障状态。
5. 主控下发 `START_ZONE`。
6. 页面进入运行监控。

手动运行页面显示：

- 从站地址和名称。
- 水路名称。
- 计划运行时长。
- 已运行时长。
- 实时流速。
- 累计脉冲和估算水量。
- 低液位状态。
- 电压状态。
- 主阀、自吸泵、分支阀输出状态。

停止动作：

- 用户点击停止时，主控下发 `STOP`。
- 停止后记录停止原因。
- 如果从站已因本地安全策略停机，页面应显示从站上报的停止原因，而不是覆盖成用户停止。

## 从站和水源配置

从站配置页面显示：

- 从站启用状态。
- RS485 地址。
- 拨码地址读取值。
- 在线/离线状态。
- 固件版本。
- 配置版本和 CRC。
- 水源名称。
- 最近通信时间。
- 最近故障。

从站配置动作：

- 启用或停用从站。
- 修改水源名称。
- 重新同步安全配置。
- 触发从站识别。
- 清除可恢复故障。

规则：

- 从站地址来自拨码，不通过 Web 改地址。
- 主控检测到地址冲突时，冲突从站不能运行。
- 修改从站安全相关配置后，主控必须重新下发 `SET_SAFE_CONFIG`。

## 水路配置

水路配置页面按从站显示 `VALVE1..VALVE8`。

每一路可配置：

- 禁用。
- 分支水路。
- 主阀。

分支水路配置：

- 名称。
- 启用状态。
- 默认运行时长。
- 是否允许手动运行。
- 是否允许计划运行。
- 标准流速。
- 锁定状态。

主阀配置：

- 选择哪一路作为主阀。
- 主阀启动延时。
- 主阀关闭延时。

规则：

- 一个从站最多一个主阀。
- 主阀不作为普通水路显示在手动运行列表中。
- 被锁定水路不参与自动计划。
- 未启用水路不参与手动和计划。

## 输入输出配置

输入配置：

- `IN1..IN4` 用途：禁用、流量计、低液位、预留。
- 输入有效电平。
- 低液位确认时间。
- 流量计边沿。

输出配置：

- 自吸泵干接点启用/关闭。
- 自吸泵启动延时。
- 自吸泵关闭延时。
- 阀门 PWM 吸合时间。
- 阀门保持比例。
- 阀门 PWM 频率。

规则：

- 每个从站必须配置一个流量计输入。
- 低液位可选。
- 自吸泵可选。
- 自吸泵和低液位互不强制依赖。
- 同时启用自吸泵和低液位时，启动自吸泵前从站必须检查低液位未触发。
- “预留”只适用于 `IN1..IN4` 中暂不参与当前逻辑的输入占位，不代表启用 H6/H8/H9 或其他预留接口。
- 当前 Web 不提供 H6/H8/H9、`SPARE_*`、通用寄存器或任意 IO 映射配置。

## 流量页面

流量页面包含：

- 流量计校准。
- 每路标准流速学习。
- 流量保护参数。
- 校准记录。
- 最近运行流量曲线。

流量计校准：

- 选择用于放水的分支水路。
- 开始维护运行。
- 记录脉冲。
- 输入实际接水量。
- 生成 `pulses_per_liter`。
- 保存为该从站流量计参数。

标准流速学习：

- 选择已启用、未锁定的分支水路。
- 运行水路。
- 等待首次脉冲和稳定时间。
- 采样固定时间窗口。
- 保存 `baseline_ml_min` 到该水路。

规则：

- 流量计计量未校准的从站仍可参与正常计划运行，但水量估算不可用或仅供参考。
- 未学习标准流速的水路仍可运行，但不做低/高流量判断。
- 无水、低液位、通信超时和运行超时不依赖标准流速。

## 计划页面

计划配置包含：

- 计划名称。
- 启用状态。
- 执行时间。
- 重复规则。
- 目标从站和水路顺序。
- 每路运行时长。
- 是否跳过锁定水路。

运行规则：

- 同一从站同一时间只运行一路分支水路。
- 同一个水源下按顺序运行多个水路。
- 不同从站代表不同真实水源，可以同时运行。
- 主控调度器需要支持多个从站并行运行，但每个从站内部仍保持单任务。
- 任一从站的故障默认只影响该从站，不自动停止其他从站；全局停止只能来自用户明确动作。
- 计划遇到离线从站、未同步配置或故障锁定时，跳过或失败并记录事件。
- 计划调度、错过执行、重叠计划、手动冲突和跳过规则见 `21-计划调度与运行冲突规则.md`。

## 告警与记录

记录类型：

- 手动运行记录。
- 自动计划记录。
- 流量计校准记录。
- 标准流速学习记录。
- 从站上线/离线记录。
- 配置同步记录。
- 低液位、无水、高流量、低流量、已启用电压保护并确认电压严重异常、通信超时记录。

告警处理：

- 可恢复故障支持清除。
- 安全锁定需要用户确认处理后清除。
- 清除故障不删除历史记录。

## 本地 LCD 和按键边界

LCD2004 和主控按键不作为当前交互模型的重点。

LCD2004 只承担最小状态显示：

- 设备是否在线。
- 当前是否有运行中的从站。
- 是否有未处理故障。
- 当前时间。
- 必要的简单故障提示。

主控按键只承担最小本地辅助：

- 查看或切换有限状态信息。
- 必要时停止当前运行。
- 必要时触发从站识别。

本地 LCD 和按键不承担：

- 创建或编辑计划。
- 修改从站输入输出映射。
- 修改主阀、自吸泵、低液位、流量计配置。
- 修改阀门 PWM 参数。
- 修改 RS485 或网络配置。
- 完整故障处理流程。

所有完整配置、运行、维护、故障处理和记录查看都必须通过 Web 支持。

## 实现依据

- Web 页面字段默认值和校验范围见 `16-参数默认值与校验范围.md`。
- 故障清除、用户提示和维护入口见 `17-故障清除与本地维护策略.md`。
- 多从站同时运行时，Web 首页按“运行中从站、故障从站、配置未同步从站、在线空闲从站、离线从站”排序；同类内按 `rs485_address` 升序。
- LCD 只显示当前时间、运行摘要、未处理故障数量和最短故障提示；主控按键只允许查看状态、停止当前运行或触发识别，不进入配置流程。
