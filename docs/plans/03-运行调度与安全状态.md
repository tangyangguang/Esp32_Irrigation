# 03 运行调度与安全状态

## 核心运行模型

- 主控不是全局单任务系统。
- 多个从站代表多个独立水源，可以同时运行。
- 一个从站同一时间只能有一个活动运行。
- 一个从站内部不支持多水路并行。
- 从站运行只能由主控命令启动。
- 从站本地安全策略可以独立停机。
- 主控通信丢失时，不能假定从站已经停机。
- 任一从站故障默认只影响该从站，不自动停止其他从站。
- 全局急停是用户明确动作，不是普通故障的默认联动。

## 从站状态

| 状态 | 含义 | 输出要求 |
|---|---|---|
| `BOOTING` | 上电或复位初始化 | 全部输出关闭 |
| `UNCONFIGURED` | 无有效安全配置 | 全部输出关闭 |
| `IDLE` | 空闲可运行 | 全部输出关闭 |
| `STARTING` | 正在启动泵、主阀或分支阀 | 只允许按启动步骤打开输出 |
| `RUNNING` | 正在运行水路 | 仅保持本次运行相关输出 |
| `STOPPING` | 正在关闭输出 | 按停止步骤关闭输出 |
| `FAULT` | 可恢复或待处理故障 | 全部输出关闭 |
| `LOCKED` | 安全锁定 | 全部输出关闭 |
| `OFFLINE_LOCAL_SAFE` | 通信超时后本地安全停机 | 全部输出关闭 |

`STOPPING` 必须显示为正在停止，不能显示为已停止。

## 停止原因

停止原因只回答“本次从站运行为什么停止”。计划整体跳过、计划取消、用户确认告警等不作为从站停止原因。

| 原因 | 含义 | 停止后默认状态 |
|---|---|---|
| `DURATION_DONE` | 时长正常结束 | `IDLE` |
| `USER_STOP` | 用户或主控主动停止 | `IDLE` |
| `LOW_LEVEL` | 低液位触发 | `FAULT` 或 `LOCKED` |
| `NO_FLOW_START` | 启动后首次流量超时 | `LOCKED` |
| `NO_FLOW_RUNNING` | 稳定运行中流量中断 | `LOCKED` |
| `HIGH_FLOW` | 高流量保护 | `LOCKED` |
| `LOW_FLOW` | 低流量策略要求停止 | `FAULT` 或 `IDLE` |
| `COMM_TIMEOUT` | 通信超时，本地安全停机 | `OFFLINE_LOCAL_SAFE` |
| `RUN_TIMEOUT` | 超过最大运行时间 | `FAULT` |
| `CONFIG_INVALID` | 运行中配置异常或丢失 | `UNCONFIGURED` 或 `FAULT` |
| `WATCHDOG_RESET` | 看门狗复位导致停止 | 重新判定 |
| `LOCAL_FAULT` | 本地输出、存储或其他故障 | `FAULT` 或 `LOCKED` |
| `UNKNOWN` | 未知原因 | `FAULT` |

## 启动准入

禁止启动条件：

- 从站禁用。
- 从站离线。
- 疑似地址冲突或总线异常。
- 配置无效或未同步。
- 从站已有活动运行。
- 从站处于 `FAULT`、`LOCKED`、`UNCONFIGURED`、`OFFLINE_LOCAL_SAFE`。
- 目标水路禁用或锁定。
- 请求运行时长非法。
- 已配置低液位且当前确认触发。
- 存在未清除的启动无水、运行中无水故障或锁定。
- 已启用高流量保护并确认高流量。
- 已启用低流量保护，并确认达到停机策略。

允许启动但必须提示保护不完整：

- 标准流速未学习。
- 流量计计量校准未完成。
- 电压检测未校准。
- 主阀未配置。
- 低液位未配置。
- 自吸泵未配置。

## 运行对象

`RunSession` 至少包含：

```text
run_id
run_type
station_id
rs485_address
zone_index
requested_duration_sec
state
started_at
deadline_at
source
schedule_id
config_version
stop_reason
last_seen_at
had_lost_contact
```

`RunSession` 状态：

| 状态 | 含义 |
|---|---|
| `pending` | 主控已创建，尚未发出启动 |
| `starting` | 已发启动命令，等待从站确认 |
| `running` | 从站确认运行 |
| `stopping` | 已发停止命令或等待从站关闭输出 |
| `stopped` | 已确认停止，正常结束 |
| `faulted` | 因故障或保护结束 |
| `lost_contact` | 主控无法确认从站状态 |
| `failed_to_start` | 启动请求被拒绝或启动失败 |
| `cancelled` | 启动前取消 |

## 从站本地安全保护

从站运行中必须独立处理：

- 低液位。
- 无水。
- 高流量。
- 通信超时。
- 运行超时。
- 配置失效。
- 本地输出异常。

规则：

- 安全保护触发后，从站先关闭输出，再上报停止原因。
- 主控必须接受从站上报的停止原因，不能覆盖成用户停止。
- 主控超时未收到停止确认时，进入 `lost_contact`，不能直接写成 `stopped`。
- 电压低/高告警不属于从站本地安全停机条件，只提示、记录和显示。

## 手动运行

手动运行只针对单个从站、单个分支水路。

规则：

- 手动运行不能绕过流量计输入配置、故障、锁定、配置同步或地址冲突。
- 手动运行可以在其他从站运行时启动。
- 手动运行不能抢占同一从站正在执行的计划水路。

## 自动计划

自动计划由主控执行，从站不保存计划。

规则：

- 一个计划可以包含多个从站。
- 同一从站下多个水路按配置顺序执行。
- 不同从站计划项可以并行执行。
- 自动计划不抢占手动运行。
- 自动计划不自动清除故障。
- 自动计划不自动修改配置。
- 自动计划不自动重启因故障停止的水路。

## 计划冲突

同一从站冲突时：

- 如果从站已有活动运行，后触发计划项跳过并记录 `station_busy`。
- 不做跨计划排队。
- 不做隐藏延后执行。
- 用户需要顺序运行时，应把水路合并到同一个计划。

多个计划同时触发且包含同一从站时：

1. 触发时间更早者优先。
2. 触发时间相同，`schedule_id` 更小者优先。
3. 其他涉及该从站的计划项跳过并记录冲突。

## 计划结果

| 结果 | 含义 |
|---|---|
| `completed` | 所有应执行水路都正常完成 |
| `partial` | 至少一路完成，且至少一路跳过或失败 |
| `skipped` | 全部应执行水路都未启动即跳过 |
| `failed` | 至少尝试启动或运行，但没有任何水路成功完成 |
| `cancelled` | 用户取消 |

规则：

- `skipped` 是准入或冲突结果，不是运行失败。
- `failed` 必须表示已经尝试启动或运行。
- 被跳过的水路不能生成从站停止原因，因为它没有启动。
- 跳过原因必须记录在计划记录中。

## 错过执行

- 主控断电、时间无效或系统刚启动导致计划错过时，不自动补跑。
- 主控恢复后记录错过事件。
- 下一次符合重复规则的时间再执行。

当前产品不提供补跑配置。

## 本地按键

从站默认使用 1 个维护/识别按键：

- 短按：进入从站识别模式，用状态灯辅助现场确认。
- 长按：进入维护确认流程，可用于清除可恢复故障或退出 `OFFLINE_LOCAL_SAFE`。
- 长按必须有确认动作，避免误触。
- 本地按键不能启动灌溉。
