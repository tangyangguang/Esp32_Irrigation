# 15 主控数据模型与生命周期边界

状态：成熟产品方案层文档。本文定义主控侧业务数据模型、生命周期、版本关系和边界，不定义 NVS、LittleFS、文件路径、备份算法或具体存储结构。

关联文档：

- `05-Web交互模型.md`
- `06-固件架构.md`
- `07-Web API与数据结构.md`

## 设计目标

主控数据模型必须支持：

- Web 完整配置。
- 多从站管理。
- 多从站并行运行。
- 同一从站单任务运行。
- 配置同步。
- 手动运行。
- 自动计划。
- 流量校准。
- 标准流速学习。
- 故障和锁定处理。
- 运行记录和事件解释。

本文只定义数据的业务含义和生命周期。

## 核心对象

| 对象 | 说明 | 主控是否保存 | 从站是否保存 |
|---|---|---|---|
| `SystemConfig` | 设备级业务设置 | 是 | 否 |
| `StationConfig` | 从站、水源、地址、启用状态 | 是 | 否 |
| `ZoneConfig` | 8 路阀输出的业务角色和水路配置 | 是 | 安全投影 |
| `InputConfig` | 输入用途、有效电平、边沿 | 是 | 安全投影 |
| `PumpConfig` | 自吸泵启用和延时 | 是 | 安全投影 |
| `FlowMeterConfig` | 流量计输入和校准参数 | 是 | 部分运行参数 |
| `FlowBaseline` | 每路标准流速 | 是 | 运行命令携带 |
| `Schedule` | 自动计划 | 是 | 否 |
| `RunSession` | 当前运行任务 | RAM 运行态 | 当前运行态 |
| `FaultRecord` | 故障和锁定记录 | 是 | 当前故障标志 |
| `EventRecord` | 用户可解释事件 | 是 | 否 |

## 配置生命周期

主控配置生命周期：

```text
draft -> validated -> saved -> sync_pending -> synced -> runtime_usable
```

含义：

- `draft`：Web 页面正在编辑，不能影响运行。
- `validated`：通过主控业务校验。
- `saved`：主控已接受为当前业务配置。
- `sync_pending`：涉及从站安全配置，尚未同步到从站。
- `synced`：从站安全配置版本一致。
- `runtime_usable`：从站在线、无锁定、流量计输入已配置、无明确安全阻断条件，可参与运行。
- 流量计计量未校准不影响 `runtime_usable`，只表示水量估算不可用或仅供参考。

任何从 `saved` 变更为影响安全配置的动作，都必须把对应从站重新标记为 `sync_pending`。

## 运行生命周期

运行任务生命周期：

```text
created -> start_requested -> starting -> running -> stopping -> stopped
                                             -> fault
                                             -> lost_contact
```

规则：

- `created` 只存在于主控调度器内。
- `start_requested` 表示主控已准备向从站发送运行命令。
- 从站接受后进入 `starting`。
- 从站上报 `RUNNING` 后进入 `running`。
- 用户停止、时长结束或本地保护触发后进入 `stopping`。
- 停止完成后进入 `stopped` 或 `fault`。
- 主控通信丢失但不能确认从站状态时，进入 `lost_contact`，不能假定从站已停机。

## 多从站并行模型

主控允许多个 `RunSession` 同时存在，但必须满足：

- 每个从站最多一个活动 `RunSession`。
- 不同从站可同时运行。
- 某个从站故障默认只影响该从站。
- 同一计划中同一从站的多个水路按顺序执行。
- 不同从站的计划项可以并行执行。

## 配置版本模型

配置版本用于回答：

- 主控当前认为的从站安全配置是哪一版。
- 从站当前持有的安全配置是哪一版。
- 运行命令基于哪一版安全配置生成。

规则：

- 影响从站安全行为的配置变化必须生成新版本。
- 不影响从站安全行为的显示名称、备注、页面排序不需要改变从站安全配置版本。
- `START_ZONE` 必须携带配置版本。
- 从站版本不匹配时拒绝运行。

## 记录边界

运行记录回答“发生过什么运行”。

事件记录回答“为什么发生这个结果”。

告警回答“用户现在需要处理什么”。

三者不能混用：

- 一次运行可以产生一条运行记录。
- 一次运行可以产生多个事件。
- 一个未处理故障可以对应一个告警。
- 用户确认告警不删除运行记录和事件。

## 数据损坏语义

当前阶段不规定数据如何保存，只规定损坏后业务语义：

- 主控业务配置损坏：禁止自动计划运行，Web 显示配置错误。
- 从站配置损坏：从站进入 `UNCONFIGURED`，主控禁止启动。
- 计划损坏：禁用计划，不影响手动运行。
- 校准参数损坏：对应从站标记计量未校准，正常运行允许继续，但水量估算不可用或仅供参考。
- 标准流速损坏：对应水路不做低/高流量判断，但仍保留无水保护。
- 运行记录损坏：不影响实时安全控制。

具体存储介质、文件格式、CRC 算法和恢复策略进入实施阶段再设计。
