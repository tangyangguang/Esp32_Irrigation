# 34 Web 配置范围非通用化复评结论

状态：当前阶段复评结论。本文在 `33-预留接口与安全配置边界复评结论.md` 之后，继续检查 Web 配置是否有通用 IO、通用寄存器、预留接口扩张的倾向。本文不写实施计划，不决定代码结构。

关联文档：

## 一、总体判断

当前 Web 方案没有把产品做成通用 IO 平台，也没有把预留接口纳入当前必填配置。

本轮只发现 1 个 P2 表述问题，已修正：

- `05-Web交互模型.md` 页面结构中“预留输入”表述偏泛，已改成 `IN1..IN4` 的预留占位。

当前仍不进入实施计划。

## 二、本轮修正的问题

### P2：Web 页面结构中的“预留输入”表述偏泛

涉及文档：

- `05-Web交互模型.md`

问题描述：

- 页面结构中原文写“配置流量计输入、低液位输入、自吸泵、预留输入”。
- 虽然后文已经说明“预留”只适用于 `IN1..IN4`，但页面结构摘要仍可能被误读为所有预留接口都进入 Web 配置。

风险：

- 后续页面设计时可能把 H6 I2C、H8/H9 ADC、`SPARE_*` 扩进输入输出配置。

修正方式：

- 改为“配置 `IN1..IN4` 的流量计/低液位/预留占位，以及自吸泵”。
- 增加规则：当前 Web 不提供 H6/H8/H9、`SPARE_*`、通用寄存器或任意 IO 映射配置。

## 三、当前 Web 配置范围

Web 当前应覆盖：

- 从站启用、名称、水源名称、拨码地址显示和同步状态。
- `VALVE1..VALVE8` 的分支水路、主阀、启用、名称、默认时长、手动/计划允许状态。
- `IN1..IN4` 中 1 路流量计输入。
- `IN1..IN4` 中可选 1 路低液位输入。
- `IN1..IN4` 中未使用输入的 `disabled` 或 `reserved` 占位。
- 可选自吸泵干接点输出 `dry_contact_1`。
- 阀门 PWM 参数。
- 电压检测启用状态；启用时配置阈值。
- 流量计校准、标准流速学习和保护提示。

Web 当前不覆盖：

- H6 预留 I2C 配置。
- H8/H9 预留 ADC 配置。
- `SPARE_*` 引脚配置。
- 主控板预留 ADC、脉冲输入、干接点、阀输出配置。
- 通用寄存器读写。
- Web 任意 RS485 帧透传。
- 通用 IO 映射 UI。

## 四、当前判断

本轮复评后，Web 配置范围和当前产品边界一致：

- Web 是完整业务入口，但不是通用硬件调试台。
- Web 配置只覆盖当前产品需要的从站、水源、水路、输入输出、安全配置、计划、记录和维护动作。
- 预留硬件可以在文档中被识别，但不进入当前产品用户配置。
