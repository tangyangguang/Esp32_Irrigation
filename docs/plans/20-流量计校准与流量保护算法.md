# 20 流量计校准与流量保护算法

状态：成熟产品方案层文档。本文定义一个从站一个流量计的校准、标准流速学习、运行中流量保护、待机漏水判断和故障解释规则。不定义中断实现、滤波代码、存储格式或具体硬件采样细节。

## 核心原则

- 一个从站必须配置且只能配置一个流量计。
- 一个从站对应一个真实水源，流量计表示该水源当前总出水流量。
- 流量计不是每一路分支独立配置的资源。
- 分支水路的标准流速是基于同一个流量计学习出来的水路特征值。
- 无水保护不依赖标准流速。
- 低/高流量保护依赖标准流速。
- 流量计输入是核心必配；`pulses_per_liter` 校准是计量增强能力，未校准时仍可执行普通灌溉，但水量估算不可用或仅供参考。
- 流量计校准和标准流速学习是维护运行，不等同于普通灌溉。

## 流量数据语义

从站上报的流量相关数据至少包括：

```text
pulse_count
flow_rate_ml_min
total_ml
flow_state
```

含义：

| 字段 | 含义 |
|---|---|
| `pulse_count` | 本次运行累计脉冲数 |
| `flow_rate_ml_min` | 当前窗口估算流速 |
| `total_ml` | 本次运行估算累计水量 |
| `flow_state` | 正常、未校准、等待首次流量、无水、低流量、高流量、待机漏水 |

换算关系：

```text
total_liter = pulse_count / pulses_per_liter
total_ml = total_liter * 1000
```

规则：

- `pulses_per_liter` 必须大于 0 才表示已校准。
- 当前流速允许有短时间波动，不能用单个瞬时脉冲间隔直接决定故障。
- 运行记录中同时保留脉冲数和估算水量，方便后续复核。

## 校准状态

从站流量计校准状态：

| 状态 | 含义 | 普通计划运行 |
|---|---|---|
| `uncalibrated` | 未设置有效 `pulses_per_liter` | 允许，但水量估算不可用 |
| `calibrating` | 正在执行校准流程 | 禁止其他同站运行 |
| `calibrated` | 已有有效 `pulses_per_liter` | 允许 |
| `calibration_failed` | 最近校准失败 | 允许，但提示计量不可用 |

规则：

- 流量计校准属于从站级能力，不属于某一路水路。
- 重新校准会影响该从站所有水路的水量估算。
- 重新校准后，已有各水路标准流速应标记为“建议重新学习”，但不强制立即失效；是否继续使用由 Web 明确提示用户选择。

## 流量计校准流程

校准目标：

- 得到该从站流量计的 `pulses_per_liter`。

前置条件：

- 从站在线。
- 从站配置同步。
- 从站没有活动运行。
- 至少有一路可用于放水的分支水路。
- 目标水路未锁定。
- 用户准备实际接水容器并能输入实际水量。

流程：

1. 用户在 Web 选择从站和用于放水的分支水路。
2. Web 明确提示该流程会实际打开水路。
3. 主控创建 `flow_calibration` 运行。
4. 从站按普通启动顺序打开自吸泵、主阀和分支阀。
5. 从站累计校准期间的脉冲数。
6. 用户停止校准运行，或达到校准运行时长上限自动停止。
7. 用户输入实际接水量。
8. 主控计算 `pulses_per_liter`。
9. 用户确认保存后，该从站流量计变为 `calibrated`。

计算：

```text
pulses_per_liter = pulse_count / actual_liter
```

保存条件：

- `pulse_count > 0`。
- `actual_liter > 0`。
- 计算出的 `pulses_per_liter` 在允许范围内。
- 用户确认实际接水量输入无误。

失败条件：

| 条件 | 结果 |
|---|---|
| 校准期间无脉冲 | 校准失败，提示检查流量计接线、水源、阀门 |
| 用户输入水量为 0 或超范围 | 不保存 |
| 校准运行因低液位、通信超时、电压异常停止 | 不保存，记录故障原因 |
| 从站通信丢失且最终状态未知 | 不保存 |

## 标准流速学习

标准流速是每一路分支水路的正常参考流量。

目标：

- 得到某一路分支水路在当前水源、阀门和管路条件下的 `baseline_ml_min`。

前置条件：

- 从站流量计已校准。
- 目标水路已启用。
- 目标水路不是主阀。
- 目标水路未锁定。
- 从站没有活动运行。

流程：

1. 用户在 Web 选择从站和分支水路。
2. 主控创建 `baseline_learning` 运行。
3. 从站启动该水路。
4. 等待首次流量。
5. 经过 `stabilize_sec` 稳定时间。
6. 在 `baseline_sample_sec` 内采样平均流速。
7. 保存该水路 `baseline_ml_min`。

学习成功条件：

- 学习期间有稳定脉冲。
- 没有触发低液位、无水、电压异常、通信超时。
- 采样平均流速大于 0。

学习失败条件：

| 条件 | 结果 |
|---|---|
| 等待首次流量超时 | 学习失败，可能同时触发无水 |
| 采样窗口内流量为 0 | 学习失败 |
| 本地保护停机 | 学习失败，保留故障原因 |
| 通信丢失最终状态未知 | 学习失败 |

规则：

- 学习失败不自动锁定水路，除非失败原因本身要求锁定。
- 未学习标准流速的水路仍可手动和计划运行。
- 未学习标准流速的水路不做低/高流量保护。
- 计划运行默认允许未学习标准流速的水路，但必须提示保护不完整。

## 运行阶段划分

一次普通运行的流量判断分为四段：

| 阶段 | 起点 | 终点 | 使用的保护 |
|---|---|---|---|
| 启动等待 | 分支阀打开后 | 首次脉冲或 `first_flow_timeout_sec` | 无水启动保护 |
| 稳定等待 | 首次脉冲后 | `stabilize_sec` 到达 | 无水运行保护 |
| 稳定运行 | 稳定等待结束后 | 停止前 | 无水、低流量、高流量 |
| 停止阶段 | 收到停止或保护触发 | 全部输出关闭 | 不再判定新的低/高流量 |

规则：

- `stabilize_sec` 不屏蔽无水保护。
- `stabilize_sec` 只屏蔽低/高流量判断。
- 首次脉冲出现后，后续长时间无脉冲按运行中无水处理。

## 无水保护

无水保护不依赖标准流速，只依赖脉冲是否出现。

### 启动无水

触发条件：

- 分支阀打开后，超过 `first_flow_timeout_sec` 仍没有流量脉冲。

默认结果：

- 从站本地停机。
- 停止原因 `NO_FLOW_START`。
- 默认进入 `LOCKED`。
- Web 提示检查水源、自吸泵、阀门、流量计接线和方向。

### 运行中无水

触发条件：

- 已出现过首次脉冲。
- 运行中连续超过 `no_flow_timeout_sec` 没有脉冲。

默认结果：

- 从站本地停机。
- 停止原因 `NO_FLOW_RUNNING`。
- 默认进入 `LOCKED`。
- Web 提示运行中断流。

说明：

- 无水默认属于安全风险，不建议只告警。
- 如果现场某些低流量水路会长间隔出脉冲，应通过合理校准和参数设置解决，不应关闭无水保护。

## 低流量保护

低流量用于识别堵塞、供水不足、阀门未完全打开或过滤器问题。

前置条件：

- 流量计已校准。
- 当前水路已学习 `baseline_ml_min`。
- 当前运行已进入稳定运行阶段。

触发条件：

```text
flow_rate_ml_min < baseline_ml_min * low_flow_percent / 100
```

并持续达到 `abnormal_confirm_sec`。

默认结果：

- 默认只告警，不停机。
- 如果该水路配置 `low_flow_action=stop`，则从站停机，停止原因 `LOW_FLOW`。
- 是否锁定水路由 `lock_on_low_flow` 决定。

规则：

- 低流量告警不自动停止其他从站。
- 低流量告警不自动重学标准流速。
- 用户可以重新学习标准流速，也可以检查过滤器、阀门和管路。

## 高流量保护

高流量用于识别破管、脱管、阀后异常放水或流量计异常。

前置条件：

- 流量计已校准。
- 当前水路已学习 `baseline_ml_min`。
- 当前运行已进入稳定运行阶段。

触发条件：

```text
flow_rate_ml_min > baseline_ml_min * high_flow_percent / 100
```

并持续达到 `abnormal_confirm_sec`。

默认结果：

- 从站本地停机。
- 停止原因 `HIGH_FLOW`。
- 默认进入 `LOCKED`。
- Web 提示检查管路破裂、脱管、阀门异常或流量计异常。

规则：

- 高流量默认比低流量更严重。
- 高流量不建议只告警。
- 高流量锁定只影响该从站或该水路，不默认停止其他从站。

## 待机漏水

待机漏水指从站空闲且所有输出应关闭时，仍检测到持续流量脉冲。

触发条件：

- 从站处于 `IDLE`。
- 当前无活动运行。
- 持续检测到流量脉冲超过 `standby_leak_confirm_sec`。

默认结果：

- 从站上报待机流量标志。
- 主控生成水源级告警。
- 默认不由从站直接进入运行相关停机状态，因为本来没有运行。
- Web 提示检查阀门渗漏、管路倒流、流量计干扰或外部水流。

是否锁定：

- 默认锁定该从站的新运行，直到用户确认处理。
- 用户确认后可以清除告警。
- 如果清除后仍有持续脉冲，应再次触发。

## 流量异常职责边界

从站本地必须能处理：

- 启动无水。
- 运行中无水。
- 高流量停机。
- 配置为停机的低流量。
- 运行累计脉冲和当前流速上报。

主控负责处理：

- 校准流程。
- 标准流速学习流程。
- 低流量只告警策略。
- 待机漏水告警和锁定。
- Web 提示、记录、用户清除。
- 标准流速是否需要重新学习的提示。

原因：

- 无水和高流量需要快速本地保护，不能依赖主控轮询延迟。
- 校准、学习、告警解释和用户确认属于主控业务逻辑。

## 与主阀、自吸泵、低液位的关系

- 流量计看到的是该从站当前水源总流量。
- 有主阀时，流量保护仍针对当前分支水路解释。
- 有自吸泵时，无水可能表示水源无水、泵未启动、泵故障、低液位或阀未开。
- 同时配置低液位和自吸泵时，启动自吸泵前先检查低液位；低液位触发优先于无水解释。
- 未配置低液位时，无水保护仍然有效。

停止原因优先级：

| 同时满足条件 | 停止原因 |
|---|---|
| 低液位确认 + 无水 | `LOW_LEVEL` |
| 通信超时 + 流量异常 | 以从站最先确认并执行的本地保护为准 |
| 电压严重异常 + 流量异常 | 以从站最先确认并执行的本地保护为准 |

## 参数默认策略

默认策略应保守：

- `NO_FLOW_START`：停机并锁定。
- `NO_FLOW_RUNNING`：停机并锁定。
- `HIGH_FLOW`：停机并锁定。
- `LOW_FLOW`：告警，是否停机由配置决定。
- `standby_leak`：告警并默认禁止新运行，用户确认后可清除。

如果用户修改为更宽松策略，Web 必须提示会降低保护能力。

## Web 交互要求

Web 必须能表达：

- 从站流量计是否已校准。
- 每个水路标准流速是否已学习。
- 哪些保护当前启用，哪些保护因未学习标准流速而未启用。
- 最近一次流量异常原因。
- 清除异常前需要用户检查什么。
- 重新校准或重新学习会影响哪些水路。

Web 不应表达为：

- “流量计计量未校准但水量仍然准确”。
- “低/高流量保护未启用但仍完全安全”。
- “清除告警等于问题已经修好”。

## 不支持的能力

当前产品不支持：

- 一个从站多个流量计。
- 每个分支水路独立流量计。
- 从站内部多个水路同时运行后再按流量拆分。
- 云端学习流量模型。
- 自动根据历史数据无确认地改写标准流速。
- 未经用户确认的流量异常自动重启。
