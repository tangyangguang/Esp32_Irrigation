## 一、总体判断

当前方案已经形成正确方向：

- 从站流量计输入是当前产品必需能力。
- 标准流速学习、流量计计量校准、电压检测校准属于增强能力或增强保护。
- 增强能力未完成时，应降级为“不启用该增强保护”，不得阻断核心灌溉运行。
- 增强能力只有在已启用、条件已满足并确认异常时，才可以阻断启动或触发停机。

本轮发现 3 个表述层问题，均已修正。当前仍不进入实施计划。

## 二、本轮修正的问题

### P1：电压异常表述过宽，容易被实现成默认阻断

涉及文档：

- `03-从站运行状态机.md`
- `05-Web交互模型.md`
- `06-固件架构.md`
- `07-Web API与数据结构.md`
- `08-产品验证范围清单.md`
- `13-RS485详细Payload与主控重试规则.md`
- `19-运行调度与多从站并行模型.md`
- `20-流量计校准与流量保护算法.md`
- `22-Web页面状态流与允许操作矩阵.md`
- `23-告警事件与运行记录生命周期.md`
- `25-核心运行与增强保护降级原则.md`
- `26-核心逻辑复评提示词.md`
- `28-核心逻辑二次复评结论.md`

问题描述：

- 多处写了“电压异常”或“电压严重异常”。
- 如果不限定启用和校准前提，后续实现可能把电压检测未校准、未启用、读数不可用也当作阻断条件。

风险：

- 增加电压检测能力后，反而导致原本可运行的基础灌溉被错误禁止。
- 用户未启用电压保护时，系统出现不符合预期的启动限制。

修正方式：

- 改为“已启用电压保护并确认电压严重异常”。
- 明确电压检测未校准或未启用时，只提示电压保护未启用，不生成阻断性电压异常告警。

### P1：低/高流量告警缺少启用前提说明

涉及文档：

- `23-告警事件与运行记录生命周期.md`

问题描述：

- 告警表中列出高流量、低流量，但没有在同一处说明生成前提。
- `20`、`21`、`25` 已经说明标准流速未学习时仍允许运行，但告警表可能被单独引用并误读。

风险：

- 标准流速未学习时，系统可能错误生成低/高流量告警。
- 增强保护未启用被误当成故障，影响成熟产品体验。

修正方式：

- 增加规则：低/高流量告警只在流量计输入有效、目标水路标准流速已学习、对应保护已启用后生成。
- 标准流速未学习时，只记录“低/高流量保护未启用”，不生成阻断性低/高流量告警。

### P2：电压配置项表述不完整

涉及文档：

- `03-从站运行状态机.md`
- `08-产品验证范围清单.md`
- `13-RS485详细Payload与主控重试规则.md`
- `22-Web页面状态流与允许操作矩阵.md`

问题描述：

- 原表述为“电压阈值”，没有体现电压保护可启用/禁用，以及未启用时不作为运行准入条件。

风险：

- 配置、协议和 Web 可能被设计成必须填写电压阈值，间接把电压保护变成必选能力。

修正方式：

- 改为“电压检测启用状态，以及启用时的电压阈值”。

## 三、统一后的运行准入口径

禁止启动或必须停机的核心条件：

- 从站离线、禁用、地址疑似冲突或总线异常。
- 从站配置无效、未同步、未配置或处于锁定/故障状态。
- 同一从站已有活动运行。
- 目标水路禁用或锁定。
- 运行时长非法。
- 已配置低液位且当前确认触发。
- 通信超时、运行超时、本地安全停机已确认。
- 已启用电压保护、检测已校准，并确认严重低压或高压。
- 已启用对应流量保护，并确认无水、高流量或按策略需要停机的低流量。

允许启动但必须提示保护不完整的条件：

- 流量计计量校准未完成。
- 标准流速未学习。
- 电压检测未校准或电压保护未启用。
- SHT30、LCD、待机漏水等增强诊断或显示能力不可用。
- 主阀未配置。
- 低液位未配置。
- 自吸泵未配置。

必须坚持的原则：

- 必需能力缺失才阻断运行。
- 可选能力未配置不阻断运行。
- 增强保护未学习、未校准、未启用不阻断运行。
- 增强保护确认危险后才能阻断运行。
- Web 可以提示风险，但不能把“保护未启用”写成“设备故障”。
