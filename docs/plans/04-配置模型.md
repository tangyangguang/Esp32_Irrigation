# 04 配置模型

状态：成熟产品配置模型草案。本文定义主控和从站分别保存哪些配置，以及配置之间的校验规则。

关联文档：

- `01-新版硬件启用范围与系统边界.md`
- `02-RS485通讯协议设计草案.md`
- `03-从站运行状态机.md`
- `12-从站安全配置语义与持久化边界.md`

## 设计目标

- 主控保存完整业务配置，用于 Web/API、LCD 显示、计划调度、记录和从站管理。
- 从站保存必要安全配置，用于本地执行和安全停机。
- 从站安全配置必须足够让从站在主控离线时安全关闭输出。
- 配置项不做通用化抽象，只覆盖本灌溉产品需要的资源和策略。
- 配置需要可校验、可版本化、可同步到从站。

## 配置分层

配置分为四类：

| 配置类别 | 保存位置 | 用途 |
|---|---|---|
| 主控系统配置 | 主控 | 网络、时间、LCD、SHT30、全局策略 |
| 从站业务配置 | 主控 | 从站名称、水源名称、水路名称、启用状态、计划引用 |
| 从站安全配置 | 主控 + 从站 | 输入/输出映射、本地安全参数、阀门参数 |
| 运行时参数 | 主控下发，部分从站临时保存 | 单次运行时长、标准流速、流量保护参数 |

## 主控系统配置

主控系统配置由 ESP32 保存。

包含：

- 设备名称。
- Web/API 访问配置。
- Wi-Fi 或网络配置。
- RTC 时间同步状态。
- LCD2004 显示配置。
- SHT30 温湿度显示和记录配置。
- RS485 波特率，默认 `19200 bps`。
- 从站轮询周期。
- 事件记录保留策略。

主控板启用 DS3231 RTC 作为可靠时间源。

主控板不启用 FRAM。主控侧负责配置、计划、校准参数和记录的生命周期管理；具体存储介质属于实施阶段设计。

## 从站业务配置

从站业务配置只保存在主控。

包含：

```text
station_id
rs485_address
enabled
station_name
water_source_name
notes
```

规则：

- `rs485_address` 来自从站拨码地址，范围 `1..15`。
- 主控不通过配对设置地址。
- 主控需要检测地址冲突。
- 未启用的从站不参与计划和手动运行。

## 水路配置

水路配置只保存在主控。

每个从站最多 8 路阀输出，对应 `VALVE1..VALVE8`。

每一路阀输出可以是：

- 禁用。
- 分支水路。
- 主阀。

水路配置包含：

```text
zone_index             1..8
role                   disabled / branch_zone / master_valve
enabled                true / false
name
default_duration_sec
lock_state
manual_allowed
schedule_allowed
baseline_ml_min
baseline_status
```

规则：

- 一个从站最多配置 1 路主阀。
- 主阀不能同时作为分支水路。
- 分支水路至少需要 1 路启用，否则从站不允许进入可运行配置。
- 启用的分支水路才出现在手动运行和计划配置中。
- 主阀不单独作为用户可运行水路。
- 标准流速 `baseline_ml_min` 属于分支水路，不属于主阀。

## 输入配置

从站有 4 路输入，记为 `IN1..IN4`。

每路输入可以配置为：

- 禁用。
- 流量计。
- 低液位。
- 预留输入。

输入配置包含：

```text
input_index             1..4
role                    disabled / flow_meter / low_level / reserved
active_level            low / high
debounce_ms
```

规则：

- 一个从站必须且只能配置 1 路流量计。
- 低液位可选。
- 自吸泵和低液位不互相强制依赖。
- 同一路输入不能同时承担多个角色。
- 低液位输入启用时，需要配置有效电平和确认时间。
- 流量计输入使用边沿计数，不使用普通去抖。

## 流量计配置

一个从站只支持一个流量计。

流量计配置包含：

```text
input_index
edge                    rising / falling
pulses_per_liter
sample_window_sec
display_refresh_ms
first_flow_timeout_sec
no_flow_timeout_sec
stabilize_sec
standby_leak_confirm_sec
```

规则：

- `pulses_per_liter` 来自流量计校准。
- 未完成流量计校准时，从站不应参与正常灌溉运行。
- 运行开始后，超过 `first_flow_timeout_sec` 没有脉冲，判断无水。
- 稳定运行后，连续超过 `no_flow_timeout_sec` 无脉冲，判断运行中无水。
- `stabilize_sec` 只屏蔽低/高流量判断，不屏蔽无水和低液位保护。
- 空闲时检测到连续流量超过 `standby_leak_confirm_sec`，上报待机漏水。

## 低液位配置

低液位配置可选。

配置包含：

```text
enabled
input_index             0=不启用，1..4=输入
active_level            low / high
confirm_ms
action                  stop_current_source
```

规则：

- 未配置低液位时，从站不做低液位保护。
- 配置低液位后，运行中触发低液位必须停机。
- 同时配置自吸泵和低液位时，启动自吸泵前必须确认低液位未触发。
- 低液位停机默认不锁定具体水路，但会记录水源级故障。
- 低液位是否需要人工清除，由故障策略配置决定。

## 自吸泵配置

自吸泵输出可选。

配置包含：

```text
enabled
output                  dry_contact_1
start_delay_ms
stop_delay_ms
```

规则：

- 未启用自吸泵时，从站运行水路不控制干接点输出。
- 启用自吸泵时，运行任意分支水路都会联动自吸泵。
- 如果同时启用低液位，启动自吸泵前必须先检查低液位。
- 自吸泵输出不用于普通阀控制。

## 主阀配置

主阀可选。

配置包含：

```text
enabled
valve_index             0=无主阀，1..8=主阀
start_delay_ms
stop_delay_ms
```

规则：

- 未启用主阀时，所有启用阀都作为分支水路。
- 启用主阀时，该阀不再作为普通分支水路。
- 运行任意分支水路时，主阀先于分支水路打开。
- 停止时，分支水路先关闭，主阀后关闭。

## 阀门 PWM 配置

阀门 PWM 配置保存到从站安全配置。

配置包含：

```text
pull_in_ms
hold_percent
pwm_hz
```

规则：

- 分支阀打开后先全功率吸合 `pull_in_ms`。
- 吸合结束后切换到 `hold_percent` 保持。
- `hold_percent=100` 表示不降功率。
- PWM 参数影响阀门发热、噪声和吸合可靠性，修改后需要实机验证。

## 电压检测配置

从站需要检测系统电压。

配置包含：

```text
voltage_scale
voltage_offset_mv
voltage_calibration_state
voltage_low_warn_mv
voltage_low_stop_mv
voltage_high_stop_mv
```

规则：

- 电压值随状态包上报。
- `voltage_calibration_state` 可为 `uncalibrated`、`calibrated`、`invalid`。
- `uncalibrated` 时，电压值只能作为维护参考，不启用电压告警和电压停机，但不阻断基本灌溉运行。
- `invalid` 表示电压检测能力异常；默认不启用电压保护，并在 Web 显示维护提示。只有已经确认电压处于危险范围时，才禁止运行。
- `calibrated` 时，低于 `voltage_low_warn_mv` 上报告警。
- `calibrated` 时，低于 `voltage_low_stop_mv` 不允许启动新运行。
- `calibrated` 时，运行中低于 `voltage_low_stop_mv` 或高于 `voltage_high_stop_mv`，触发电压故障停机。

## 流量保护配置

流量保护配置由主控保存，并在运行命令中下发本次运行所需参数。

配置包含：

```text
low_flow_percent
high_flow_percent
abnormal_confirm_sec
low_flow_action          warn / stop
high_flow_action         stop
lock_on_no_flow          true / false
lock_on_high_flow        true / false
lock_on_low_flow         true / false
```

规则：

- 高流量默认停机。
- 无水默认停机。
- 低流量可以只告警，也可以停机。
- 高流量通常用于识别脱管、破管或阀后异常放水。
- 低流量通常用于识别堵塞、供水不足或过滤器问题。
- 未学习标准流速的水路，不做低/高流量判断，但仍做无水、低液位和超时保护。
- 增强保护未就绪时的降级原则见 `25-核心运行与增强保护降级原则.md`。

## 通信安全配置

通信安全配置保存到从站。

包含：

```text
comm_timeout_ms
max_run_sec
```

规则：

- 运行中超过 `comm_timeout_ms` 未收到有效主控请求，从站本地安全停机。
- 单次运行不能超过 `max_run_sec`。
- `START_ZONE` 请求运行时长不能超过 `max_run_sec`。
- 通信恢复后，从站不自动恢复运行。

## 本地维护配置

从站产品默认使用 1 个维护/识别按键。

配置包含：

```text
identify_display_sec
long_press_ms
fault_clear_hold_ms
```

规则：

- 短按只进入识别显示，不改变运行状态。
- 运行中短按只显示状态，不停机、不启动、不切换水路。
- 长按进入维护确认流程。
- 本地按键不能直接启动灌溉。

## 从站安全配置包

同步到从站的安全配置至少包含：

- 配置版本和 CRC。
- 输入角色和有效电平。
- 流量计输入和边沿。
- 低液位输入、有效电平、确认时间。
- 自吸泵是否启用、启动/停止延时。
- 主阀是否启用、启动/停止延时。
- 启用的分支水路位图。
- 阀门 PWM 参数。
- 通信超时时间。
- 最大运行时间。
- 电压检测阈值。

不需要同步到从站的业务配置：

- 从站显示名称。
- 水源名称。
- 水路显示名称。
- Web 页面排序。
- 自动计划详情。
- 历史记录。

## 配置校验总规则

主控在保存配置前必须校验：

- 从站地址不冲突；疑似地址或总线冲突时，相关地址禁止运行。
- 每个启用从站有且只有一个流量计输入。
- 低液位输入不与流量计输入重复。
- 主阀不与分支水路重复。
- 至少启用一个分支水路。
- 自吸泵启用时输出资源有效。
- 所有时间、百分比、阈值在允许范围内。

从站在接收 `SET_SAFE_CONFIG` 时也必须重复关键校验，不能完全相信主控。

## 方案依据和边界

- 各配置项默认值、最小值、最大值和单位见 `16-参数默认值与校验范围.md`。
- 配置版本、同步状态和从站本地配置语义见 `12-从站安全配置语义与持久化边界.md` 与 `13-RS485详细Payload与主控重试规则.md`。
- 主控侧业务数据模型、生命周期和记录边界见 `15-主控数据模型与生命周期边界.md`。
- 主控和从站都必须能判断配置是否完整、是否过期、是否与当前运行请求匹配；具体存储介质、文件格式和断电保护算法属于实施阶段设计。
