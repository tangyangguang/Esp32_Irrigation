状态：当前阶段复评结论。本文在 `35-可选保护运行准入复评结论.md` 之后，继续检查计划跳过、失败、停止原因和增强能力提示是否存在概念混用。本文不写实施计划，不决定代码结构。

## 一、总体判断

当前方案的计划执行方向是正确的：

- 不同从站可以并行执行。
- 同一从站内部一次只运行一路。
- 同一从站冲突时，后触发计划项直接跳过并记录原因。
- 不做跨计划排队。
- 不做隐藏延后执行。
- 增强能力未校准、未学习、未启用不作为计划跳过原因。

本轮发现 4 个概念层问题，均已修正。当前仍不进入实施计划。

## 二、本轮修正的问题

### P1：跳过策略被写成可配置项

涉及文档：

- `05-Web交互模型.md`
- `07-Web API与数据结构.md`
- `21-计划调度与运行冲突规则.md`

问题描述：

- Web 页面职责中写了“跳过策略”。
- `Schedule` 数据中存在 `skip_policy`。
- `Schedule` 对象中存在 `skip_locked_zone`。
- 计划页面写了“是否跳过锁定水路”。

风险：

- 后续实现可能把跳过做成用户可配置策略。
- 用户可能误以为可以选择排队、延后、忽略锁定或继续执行。
- 与已确认的“冲突直接跳过并记录，不排队、不隐藏延后”不一致。

修正方式：

- 删除 `skip_policy` 和 `skip_locked_zone`。
- Web 页面不再配置“是否跳过锁定水路”。
- 明确当前产品固定采用：未启动项跳过并记录原因，不排队、不隐藏延后。

### P1：计划全部跳过与失败混用

涉及文档：

- `21-计划调度与运行冲突规则.md`
- `22-Web页面状态流与允许操作矩阵.md`
- `23-告警事件与运行记录生命周期.md`

问题描述：

- 原文把“没有任何水路成功启动或完成”归为 `failed`。
- 这会把“全部未启动即跳过”和“尝试启动/运行后失败”混在一起。

风险：

- Web 会把全部跳过显示成运行失败。
- 统计会把准入跳过当成设备运行故障。
- 用户难以区分是计划冲突、离线、锁定导致没启动，还是启动后真的出故障。

修正方式：

- 增加计划结果 `skipped`：全部应执行水路都未启动即跳过。
- 保留 `failed`：至少尝试启动或运行，但没有任何水路成功完成。
- Web 增加“计划全部跳过”显示状态，禁止显示为运行失败。

### P1：计划执行校验中“校准”表述过宽

涉及文档：

- `07-Web API与数据结构.md`

问题描述：

- 原文写“计划执行时再次校验在线、锁定、校准和配置同步状态”。
- “校准”可能被误读为流量计计量校准、标准流速学习或电压检测校准。

风险：

- 增强能力未校准可能被错误实现为计划跳过条件。

修正方式：

- 改为校验在线、锁定、配置同步和明确安全阻断状态。
- 增强能力未校准或未学习只生成保护不完整提示，不作为跳过原因。

### P2：错过执行补跑配置超出当前产品范围

涉及文档：

- `21-计划调度与运行冲突规则.md`

问题描述：

- 原文保留了“允许在短时间窗口内补跑”的可选策略。

风险：

- 引入断电恢复后自动补跑的额外行为，需要更多现场安全判断和用户预期解释。
- 当前产品目标不需要这个复杂度。

修正方式：

- 明确当前产品不提供补跑配置。
- 错过执行只记录事件，等待下一次符合重复规则的时间再执行。

## 三、统一后的计划结果语义

计划结果必须按以下语义使用：

| 结果 | 含义 |
|---|---|
| `completed` | 所有应执行水路都正常完成 |
| `partial` | 至少一路完成，且至少一路跳过或失败 |
| `skipped` | 全部应执行水路都未启动即跳过 |
| `failed` | 至少尝试启动或运行，但没有任何水路成功完成 |
| `cancelled` | 用户取消 |

规则：

- `skipped` 是准入或冲突结果，不是运行失败。
- `failed` 必须表示已经尝试启动或运行。
- `stop_reason` 只描述一次从站运行为什么停止，不描述计划整体跳过。
- 计划跳过原因必须记录在 `ScheduleRunRecord.skipped_items` 或事件中。
- 被跳过的水路不能生成从站停止原因，因为它没有启动。

## 四、当前仍需继续复评的方向

下一轮不进入实施计划，应继续检查：

- `Fault`、`Alarm`、`Warning`、`Event` 是否还有边界重叠。
- `LOCKED` 和 `FAULT` 的默认进入条件是否一致。
- `LOW_FLOW`、`LOW_LEVEL`、`STANDBY_LEAK` 这些既可能告警又可能阻断的场景，是否都写清楚触发、清除和运行准入影响。
- 本地按键维护确认、Web 清除故障、告警确认是否存在语义混用。
- 从站停止原因、主控运行结果、计划结果是否还有混用。
