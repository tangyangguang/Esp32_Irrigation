# 33 预留接口与安全配置边界复评结论

状态：当前阶段复评结论。本文在 `32-接口座边界复评结论.md` 之后，继续检查预留接口是否误进入 Web 必填配置或从站安全配置必填项，以及增强保护参数是否被误写成核心配置条件。本文不写实施计划，不决定代码结构。

## 一、总体判断

本轮发现 2 类残留问题，均已修正：

- “预留输入”容易被误解成从站所有预留接口都进入 Web 配置。
- 从站安全配置中部分表述容易把未启用的增强保护参数、计量校准状态写成运行准入条件。

当前统一口径：

- Web 输入配置只覆盖从站 `H4` 上的 `IN1..IN4`。
- `reserved` 只是 `IN1..IN4` 中某一路暂不参与当前灌溉逻辑的占位角色。
- 从站 `H6` 预留 I2C、`H8/H9` 预留 ADC、`SPARE_*` 不进入当前 Web 必填配置，也不进入当前从站安全配置必填项。
- 低液位、自吸泵、主阀、电压保护、低/高流量保护都是可选或增强能力；未启用时同步禁用状态，不导致配置无效。
- 流量计输入必配，但流量计计量校准不是 `synced` 或 `runtime_usable` 的前提。

当前仍不进入实施计划。

## 二、本轮修正的问题

### P1：`synced` 状态仍残留“已校准”准入条件

涉及文档：

- `12-从站安全配置语义与持久化边界.md`

问题描述：

- `synced` 原文写“是，还需满足在线、无故障、已校准”。
- 这里的“已校准”容易被理解为流量计计量校准完成，和当前核心原则冲突。

风险：

- 主控可能把同步状态、运行准入和计量增强能力混在一起，导致未校准时不能普通运行。

修正方式：

- 改为“在线、无故障、无明确安全阻断；流量计计量未校准不影响同步状态”。

### P1：增强保护参数容易被写成安全配置必填条件

涉及文档：

- `12-从站安全配置语义与持久化边界.md`
- `04-配置模型.md`
- `07-Web API与数据结构.md`

问题描述：

- 原文写“电压和流量保护参数在允许范围内”“电压检测阈值”等，未区分启用和未启用状态。
- 低液位、主阀、自吸泵的同步内容也需要明确为“启用时才包含对应参数”。

风险：

- 未启用电压保护、未学习标准流速、未配置低液位或主阀时，可能被误判为配置不完整。

修正方式：

- 改为“已启用的电压和流量保护参数在允许范围内；未启用的增强保护使用禁用状态，不导致配置无效”。
- 明确低液位、自吸泵、主阀、电压检测都同步启用状态；只有启用时才要求对应输入、输出、阈值或延时参数有效。

### P2：`reserved` 输入角色边界不够明确

涉及文档：

- `04-配置模型.md`
- `05-Web交互模型.md`
- `07-Web API与数据结构.md`

问题描述：

- 输入配置支持 `reserved`，但此前未明确它只作用于 `IN1..IN4`。
- 从站还有 `H6` I2C、`H8/H9` ADC、`SPARE_*` 等预留资源，如果不说明，容易被误纳入 Web 配置范围。

风险：

- Web 配置面会膨胀成通用 IO 配置器。
- 从站安全配置会包含当前不用的预留接口。

修正方式：

- 明确输入配置只覆盖 `IN1..IN4`。
- 明确 `reserved` 只是 `IN1..IN4` 的占位角色，不生成保护、不参与运行准入、不要求传感器参数。
- 明确 `H6/H8/H9/SPARE_*` 不进入当前 Web 输入配置和安全配置必填项。

## 三、当前必填与可选边界

| 项目 | 当前性质 | 是否影响运行准入 |
|---|---|---|
| 从站地址 | 必需 | 地址冲突或无效时禁止运行 |
| `IN1..IN4` 中 1 路流量计输入 | 必需 | 未配置时禁止运行 |
| 流量计计量校准 `pulses_per_liter` | 增强计量 | 未校准不禁止运行 |
| 低液位输入 | 可选保护 | 未启用不禁止运行 |
| 自吸泵干接点输出 | 可选联动 | 未启用不禁止运行 |
| 主阀 | 可选水路结构 | 未启用不禁止运行 |
| 分支水路至少 1 路 | 必需 | 未启用任何分支水路时禁止运行 |
| 电压保护 | 增强保护 | 未校准/未启用不禁止运行 |
| 标准流速低/高流量保护 | 增强保护 | 未学习不禁止运行 |
| `H6` I2C | 预留接口 | 不进入当前运行准入 |
| `H8/H9` ADC | 预留接口 | 不进入当前运行准入 |
| `SPARE_*` | 预留 MCU 引脚 | 不进入当前运行准入 |
