# 05 RS485 通信协议

## 协议目标

- 主控通过 RS485 管理一个或多个从站。
- 一个从站代表一个真实水源，最多 8 路阀输出，最多 1 个流量计。
- 协议优先保证稳定、安全和可维护性，不追求高吞吐。
- 从站不能依赖主控实时在线才能保证安全。
- 协议面向本产品固定硬件，不设计成通用寄存器平台。

## 通信模式

- 主控永远主动发起通信。
- 从站只响应主控请求，不主动发送。
- 地址来自 4 位拨码开关。
- 地址范围 `1..15`。
- 地址 `0` 保留。
- 默认波特率 `19200 bps`。
- 现场验证稳定后，可提供 `38400 bps` 配置。
- 不使用 `115200 bps` 作为默认控制波特率。

## 主控通信参数

| 参数 | 默认值 | 范围 | 单位 | 说明 |
|---|---:|---:|---|---|
| `request_timeout_ms` | 150 | 100..500 | ms | 普通请求超时 |
| `max_retries_idle` | 2 | 0..3 | 次 | 空闲从站查询 |
| `max_retries_running` | 1 | 0..2 | 次 | 运行中从站查询 |
| `offline_fail_count` | 3 | 2..5 | 次 | 主控判定离线 |
| `conflict_suspect_count` | 3 | 2..5 | 次 | 主控判定疑似地址冲突 |

运行中从站轮询必须优先于空闲从站轮询。通信参数用于主控调度和离线判断，不下发为从站安全配置；从站本地通信超时使用 `comm_timeout_ms`。

## 帧格式

```text
SOF        2 字节   固定 0xA5 0x5A
VER        1 字节   协议版本
ADDR       1 字节   从站地址，1..15
SEQ        1 字节   包序号
CMD        1 字节   命令
LEN        1 字节   Payload 长度
PAYLOAD    N 字节
CRC16      2 字节   从 VER 到 PAYLOAD 的 CRC
```

规则：

- `LEN` 限制为 `0..200`。
- `CRC16` 采用 Modbus CRC16。
- 多字节整数统一小端序。
- 单位必须写入字段名：`_ms`、`_sec`、`_mv`、`_ml`、`_ml_min`。
- 位图从低位开始编号，bit0 对应第 1 路资源。
- 保留字段发送方置 0，接收方忽略。

## 应答规则

主控只接受满足以下条件的应答：

1. SOF 正确。
2. CRC 正确。
3. `VER` 支持。
4. `ADDR` 等于请求地址。
5. `SEQ` 等于请求序号。
6. `CMD` 等于请求命令。
7. Payload 长度符合命令定义。

所有应答 Payload 以 `result_code uint8` 开头。

CRC 错误可以不应答，由主控超时重试。命令合法但业务拒绝时必须应答错误码。

## 命令集

| 编号 | 命令 | 用途 |
|---:|---|---|
| `0x01` | `PING` | 在线探测 |
| `0x02` | `GET_INFO` | 读取设备信息和硬件能力 |
| `0x03` | `GET_STATUS` | 读取运行状态 |
| `0x04` | `GET_CONFIG_SUMMARY` | 读取配置版本和 CRC |
| `0x10` | `SET_SAFE_CONFIG` | 写入从站安全配置 |
| `0x11` | `GET_SAFE_CONFIG` | 读取从站安全配置 |
| `0x20` | `START_ZONE` | 启动某一路水路 |
| `0x21` | `STOP` | 停止当前运行 |
| `0x22` | `CLEAR_FAULT` | 清除可恢复故障 |
| `0x30` | `FACTORY_RESET_CONFIG` | 清空从站本地配置 |

明确不支持：

- 多主控。
- 从站主动上报。
- 广播运行命令。
- JSON/文本协议。
- 通用寄存器映射。
- Web 任意透传 RS485 帧。

除特别说明外，查询类命令请求 Payload 为空。

## 结果码

| 编号 | 名称 | 含义 |
|---:|---|---|
| `0x00` | `OK` | 成功 |
| `0x01` | `ERR_BAD_CMD` | 不支持的命令 |
| `0x02` | `ERR_BAD_LEN` | Payload 长度错误 |
| `0x03` | `ERR_BAD_VALUE` | 字段值非法 |
| `0x04` | `ERR_BAD_CRC` | 配置块或数据块 CRC 不匹配 |
| `0x05` | `ERR_NOT_CONFIGURED` | 从站没有有效安全配置 |
| `0x06` | `ERR_CONFIG_MISMATCH` | 配置版本不一致 |
| `0x07` | `ERR_BUSY` | 从站正在运行 |
| `0x08` | `ERR_LOCKED` | 从站故障锁定或安全锁定 |
| `0x09` | `ERR_INVALID_ZONE` | 水路无效或未启用 |
| `0x0A` | `ERR_INPUT_CONFLICT` | 输入配置冲突 |
| `0x0B` | `ERR_OUTPUT_CONFLICT` | 输出配置冲突 |
| `0x0C` | `ERR_SAVE_FAILED` | 本地保存配置失败 |
| `0x0D` | `ERR_UNSAFE_STATE` | 当前状态不允许执行 |

## 状态、停止原因和故障位

从站状态码：

| 编号 | 状态 |
|---:|---|
| `0x00` | `BOOTING` |
| `0x01` | `UNCONFIGURED` |
| `0x02` | `IDLE` |
| `0x03` | `STARTING` |
| `0x04` | `RUNNING` |
| `0x05` | `STOPPING` |
| `0x06` | `FAULT` |
| `0x07` | `LOCKED` |
| `0x08` | `OFFLINE_LOCAL_SAFE` |

停止原因码：

| 编号 | 原因 |
|---:|---|
| `0x00` | `NONE` |
| `0x01` | `DURATION_DONE` |
| `0x02` | `USER_STOP` |
| `0x03` | `LOW_LEVEL` |
| `0x04` | `NO_FLOW_START` |
| `0x05` | `NO_FLOW_RUNNING` |
| `0x06` | `HIGH_FLOW` |
| `0x07` | `LOW_FLOW` |
| `0x08` | `COMM_TIMEOUT` |
| `0x09` | `RUN_TIMEOUT` |
| `0x0A` | `VOLTAGE_LOW` |
| `0x0B` | `VOLTAGE_HIGH` |
| `0x0C` | `CONFIG_INVALID` |
| `0x0D` | `WATCHDOG_RESET` |
| `0x0E` | `LOCAL_FAULT` |
| `0x0F` | `UNKNOWN` |

`fault_flags`：

| 位 | 含义 |
|---:|---|
| 0 | 配置无效 |
| 1 | 通信超时本地安全停机 |
| 2 | 低液位 |
| 3 | 无水 |
| 4 | 高流量 |
| 5 | 低流量 |
| 6 | 电压低 |
| 7 | 电压高 |
| 8 | 运行超时 |
| 9 | 输出被强制关闭 |
| 10 | 疑似地址冲突 |
| 11 | 本地配置保存或读取异常 |

无
```

```

用途：

- 在线探测。
- 空闲从站低成本轮询。

## `GET_INFO`

请求：

```text
无
```

应答：

```text
result_code        uint8
hw_type            uint8
hw_revision        uint8
fw_major           uint8
fw_minor           uint8
fw_patch           uint8
dipswitch_addr     uint8
capability_flags   uint32
device_uid         uint32
```

`device_uid` 可用于显示和辅助诊断，但当前地址冲突处理不依赖它。地址冲突按同地址响应异常、CRC 错误、响应矛盾和稳定性问题判断。

`capability_flags`：

| 位 | 含义 |
|---:|---|
| 0 | 支持 8 路阀输出 |
| 1 | 支持 4 路数字输入 |
| 2 | 支持 1 路流量计 |
| 3 | 支持低液位输入 |
| 4 | 支持 1 路干接点输出 |
| 5 | 支持系统电压 ADC |
| 6 | 支持本地安全配置持久化 |
| 7 | 支持阀门 PWM 保持 |

## `GET_STATUS`

请求：

```text
无
```

应答：

```text
result_code            uint8
state                  uint8
run_id                 uint32
active_zone            uint8
remaining_sec          uint16
last_stop_reason       uint8
fault_flags            uint32
input_level_bitmap     uint8
output_active_bitmap   uint8
pulse_count            uint32
flow_pulses_per_min    uint16
voltage_mv             uint16
voltage_status_flags   uint8
config_version         uint32
uptime_sec             uint32
```

规则：

- `active_zone=0` 表示当前没有活动水路。
- `pulse_count` 是当前运行或最近一次运行相关脉冲计数；空闲时可保持最近值。
- `flow_pulses_per_min` 是当前估算的每分钟脉冲数，不要求从站换算成水量。
- `voltage_mv=0` 表示当前电压值不可用。
- `output_active_bitmap` bit0 对应 VALVE1，bit7 对应 VALVE8；自吸泵干接点不放入该位图。

`voltage_status_flags`：

| 位 | 含义 |
|---:|---|
| 0 | ADC 电压值有效 |
| 1 | 电压保护已启用 |
| 2 | 已达到低压预警 |
| 3 | 已达到低压停机 |
| 4 | 已达到高压停机 |

## Safe Config Payload

```text
station_addr              uint8
input_role[4]             uint8[4]
input_active_level[4]     uint8[4]
input_debounce_ms[4]      uint16[4]
flow_input_index          uint8
flow_edge                 uint8
low_level_enabled         uint8
low_level_input_index     uint8
low_level_active_level    uint8
low_level_confirm_ms      uint16
pump_enabled              uint8
pump_start_delay_ms       uint16
pump_stop_delay_ms        uint16
master_valve_index        uint8
master_start_delay_ms     uint16
master_stop_delay_ms      uint16
branch_valve_bitmap       uint8
valve_pull_in_ms          uint16
valve_pwm_freq_hz         uint16
valve_hold_percent        uint8
comm_timeout_ms           uint16
max_run_sec               uint16
voltage_protection_enabled uint8
voltage_low_warn_mv       uint16
voltage_low_stop_mv       uint16
voltage_high_stop_mv      uint16
reserved                  uint8[8]
```

编码：

- `input_role`: 0=disabled, 1=flow_meter, 2=low_level, 3=reserved。
- `input_active_level`: 0=low, 1=high。
- `flow_input_index`: 1..4。
- `flow_edge`: 0=rising, 1=falling。
- `low_level_input_index`: 0=不启用，1..4。
- `master_valve_index`: 0=无主阀，1..8。
- `branch_valve_bitmap`: bit0 对应 VALVE1，bit7 对应 VALVE8。
- `valve_hold_percent`: 0..100。
- `voltage_protection_enabled` 只有在主控已完成电压校准后才能置 1；为 0 时从站只上报电压，不执行电压停机保护。

## `START_ZONE`

请求：

```text
run_id                    uint32
config_version            uint32
zone_index                uint8
duration_sec              uint16
expected_flow_ml_min      uint32
low_flow_percent          uint8
high_flow_percent         uint8
abnormal_confirm_sec      uint16
first_flow_timeout_sec    uint16
no_flow_timeout_sec       uint16
stabilize_sec             uint16
flags                     uint16
```

`expected_flow_ml_min=0` 表示该水路未学习标准流速，从站不做低/高流量判断。

`flags`：

| 位 | 含义 |
|---:|---|
| 0 | 启用低流量停机 |
| 1 | 启用高流量停机 |
| 2 | 无水停机，普通灌溉运行必须置 1 |
| 3 | 低液位停机；配置低液位时普通灌溉运行必须置 1 |

规则：

- 无水保护是核心安全能力，普通灌溉运行不得关闭。
- `expected_flow_ml_min=0` 时，从站不做低/高流量判断，但仍必须做无水判断。
- 校准、标准流速学习等维护运行如果需要特殊处理，必须在运行类型或主控记录中解释，不能影响普通灌溉默认安全策略。

应答：

```text
result_code             uint8
state                   uint8
run_id                  uint32
accepted_zone           uint8
reject_reason           uint8
```

## `CLEAR_FAULT`

请求：

```text
fault_mask              uint32
flags                   uint8
```

规则：

- 只能清除可恢复故障。
- 清除故障不能绕过配置无效、地址冲突、输出无法确认关闭等安全条件。
- 清除成功后不能自动启动运行。

应答：

```text
result_code             uint8
state                   uint8
fault_flags             uint32
```

## `FACTORY_RESET_CONFIG`

请求：

```text
confirm_code            uint32
```

规则：

- `confirm_code` 固定为 `0xA55A5AA5`，防止误触发。
- 只清空从站本地安全配置，不改变拨码地址。
- 从站处于 `STARTING`、`RUNNING`、`STOPPING` 时必须拒绝。
- 清空后从站进入 `UNCONFIGURED`，全部输出保持关闭。
- Web 或维护入口必须二次确认后才能触发。

应答：

```text
result_code             uint8
state                   uint8
config_valid            uint8
```

## 主控重试和离线

- 普通查询超时可重试。
- 运行控制命令重试必须使用同一 `SEQ` 或明确幂等处理。
- 运行中连续通信失败后，主控标记 `lost_contact`。
- 从站本地通信超时到达后进入 `OFFLINE_LOCAL_SAFE` 并关闭输出。
- 通信恢复后不自动继续原运行。

## 地址冲突

主控发现同一地址出现乱码、CRC 错误、应答不稳定、相同请求返回矛盾信息时，标记疑似地址冲突。

规则：

- 禁止该地址参与运行。
- Web 提示断开或逐个接入从站排查拨码地址。
- 从站本身通常不能可靠判断地址冲突。
