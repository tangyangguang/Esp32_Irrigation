进入实施计划前，应先按 `26-核心逻辑复评提示词.md` 对当前全部方案文档做基础复评，按 `29-核心逻辑三次复评结论.md` 末尾提示词继续检查最新运行逻辑残留口径，并按 `30-硬件源文件对照复评结论.md`、`31-硬件IO命名细化复评结论.md`、`32-接口座边界复评结论.md`、`33-预留接口与安全配置边界复评结论.md`、`34-Web配置范围非通用化复评结论.md`、`35-可选保护运行准入复评结论.md` 和 `36-计划跳过与结果语义复评结论.md` 末尾提示词检查硬件源文件一致性、IO 命名、接口座边界、安全配置边界、Web 配置范围、运行准入、增强保护降级原则、计划跳过和结果语义。只有方案层问题收敛后，才另行编写实施计划文档。
