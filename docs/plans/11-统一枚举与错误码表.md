## 使用规则

- 协议数值、固件枚举、Web/API 字符串和事件 token 必须使用同一语义。
- 协议层使用紧凑数值；代码和 API 保留稳定英文名称；页面显示中文文案。
- 不在不同文档里重新定义同一套枚举。
- 新增枚举值前必须先更新本文，再同步协议和代码。

## 从站状态

| 协议值 | 固件枚举/API 字符串 | 中文含义 | 输出要求 |
|---:|---|---|---|
| `0x00` | `BOOTING` | 上电或复位初始化 | 全部输出关闭 |
| `0x01` | `UNCONFIGURED` | 无有效安全配置 | 全部输出关闭 |
| `0x02` | `IDLE` | 空闲可运行 | 全部输出关闭 |
| `0x03` | `STARTING` | 正在启动泵、主阀或分支阀 | 只允许按启动步骤打开输出 |
| `0x04` | `RUNNING` | 正在运行水路 | 仅允许当前运行相关输出保持打开 |
| `0x05` | `STOPPING` | 正在关闭输出 | 按停止步骤关闭输出 |
| `0x06` | `FAULT` | 可恢复或待处理故障 | 全部输出关闭 |
| `0x07` | `LOCKED` | 安全锁定 | 全部输出关闭 |
| `0x08` | `OFFLINE_LOCAL_SAFE` | 通信超时后本地安全停机 | 全部输出关闭 |

## 停止原因

停止原因回答“本次从站运行为什么停止”。主控计划整体完成、计划跳过、用户确认告警等属于业务事件，不作为从站停止原因。

| 协议值 | 固件枚举/API 字符串 | 事件 reason token | 中文含义 | 默认后续状态 |
|---:|---|---|---|---|
| `0x00` | `NONE` | `none` | 无 | 保持当前状态 |
| `0x01` | `DURATION_DONE` | `duration_done` | 本次运行时长正常结束 | `IDLE` |
| `0x02` | `USER_STOP` | `user_stop` | 用户或主控主动停止 | `IDLE` |
| `0x03` | `LOW_LEVEL` | `low_level` | 低液位触发 | `FAULT` 或 `LOCKED` |
| `0x04` | `NO_FLOW_START` | `no_flow_start` | 启动后首次流量超时 | `LOCKED` |
| `0x05` | `NO_FLOW_RUNNING` | `no_flow_running` | 稳定运行中流量中断 | `LOCKED` |
| `0x06` | `HIGH_FLOW` | `high_flow` | 高流量保护 | `LOCKED` |
| `0x07` | `LOW_FLOW` | `low_flow` | 低流量策略要求停止 | `FAULT` 或 `IDLE` |
| `0x08` | `COMM_TIMEOUT` | `comm_timeout` | 通信超时，本地安全停机 | `OFFLINE_LOCAL_SAFE` |
| `0x09` | `RUN_TIMEOUT` | `run_timeout` | 超过从站允许的最大运行时间 | `FAULT` |
| `0x0A` | `VOLTAGE_LOW` | `voltage_low` | 系统电压过低 | `FAULT` 或 `LOCKED` |
| `0x0B` | `VOLTAGE_HIGH` | `voltage_high` | 系统电压过高 | `FAULT` 或 `LOCKED` |
| `0x0C` | `CONFIG_INVALID` | `config_invalid` | 运行中配置异常或丢失 | `UNCONFIGURED` 或 `FAULT` |
| `0x0D` | `WATCHDOG_RESET` | `watchdog_reset` | 从站看门狗复位导致停止 | `BOOTING` 后重新判定 |
| `0x0E` | `LOCAL_FAULT` | `local_fault` | 本地输出、存储或其他不可归类故障 | `FAULT` 或 `LOCKED` |
| `0x0F` | `UNKNOWN` | `unknown` | 未知原因 | `FAULT` |

## 地址和总线冲突

RS485 上两个从站拨到同一地址时，主控不保证能识别具体设备数量和身份。

主控应做到：

- 同一地址出现乱码、CRC 错误、应答不稳定、相同请求返回矛盾信息时，标记为 `station_address_conflict` 或疑似总线冲突。
- 标记后禁止该地址参与运行。
- Web 提示用户断开或逐个接入从站排查拨码地址。

从站通常不能可靠判断地址冲突，`FAULT_ADDRESS_CONFLICT` 主要由主控侧标记和展示。

## 业务事件 token

记录业务事件时：

- `source/type/reason/object` 使用机器可读英文 token。
- `reason` 优先使用本文停止原因的事件 token。
- 中文解释放在 Web 页面或 `text` 短说明中。
- 计划跳过、配置同步、故障清除等主控业务事件可以使用独立 `type/reason`，但不得和从站停止原因混用。
