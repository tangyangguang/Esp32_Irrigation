# 23 告警事件与运行记录生命周期

状态：成熟产品方案层文档。本文定义告警、故障、事件、运行记录和计划记录的业务边界、生命周期、清除规则和用户解释方式。不定义日志文件格式、数据库结构、存储介质或导出文件格式。

## 核心原则

- 告警回答“现在用户需要处理什么”。
- 故障回答“设备为什么不能运行或为什么停机”。
- 事件回答“过程中发生了什么可解释动作”。
- 运行记录回答“一次水路运行的结果”。
- 计划记录回答“一次计划触发的整体结果”。
- 清除告警不删除事件和运行记录。
- 清除故障不表示真实问题已经修好。
- 历史记录不能被当前状态覆盖。

## 概念边界

| 概念 | 生命周期 | 是否当前态 | 是否历史态 | 例子 |
|---|---|---|---|---|
| `Alarm` 告警 | 产生、确认、恢复、清除 | 是 | 可归档 | 低液位、待机漏水 |
| `Fault` 故障 | 触发、停机/锁定、清除 | 是 | 可记录 | 无水、高流量、配置无效 |
| `Event` 事件 | 发生即记录 | 否 | 是 | 配置同步成功、用户清除故障 |
| `RunRecord` 运行记录 | 运行结束后形成 | 否 | 是 | 手动运行 10 分钟完成 |
| `ScheduleRunRecord` 计划记录 | 一次计划触发后形成 | 否 | 是 | 计划部分完成 |

规则：

- 一个故障通常会产生一个告警和一个事件。
- 一次故障停机会结束一个 `RunSession` 并生成运行记录。
- 一个告警可以关联多个事件。
- 一个计划记录可以关联多个运行记录。

## 影响范围

告警和故障必须标明影响范围：

| 范围 | 含义 | 例子 |
|---|---|---|
| `system` | 影响主控或整体系统 | 主控时间无效、总线异常 |
| `station` | 影响一个从站/水源 | 从站离线、低液位、待机漏水 |
| `zone` | 影响一个分支水路 | 水路锁定、低流量 |
| `run` | 只影响一次运行 | 用户停止、运行超时 |
| `config` | 影响配置可用性 | 配置未同步、配置无效 |

规则：

- 从站级故障默认不影响其他从站。
- 水路级锁定默认只影响该水路。
- 系统级故障可能影响所有计划或所有运行，但必须明确原因。

## 告警生命周期

告警状态：

```text
raised -> active -> acknowledged -> condition_cleared -> cleared -> archived
```

含义：

| 状态 | 含义 |
|---|---|
| `raised` | 条件刚触发，尚未展示或处理 |
| `active` | 当前仍存在，需要用户处理 |
| `acknowledged` | 用户已看到并确认，但条件可能仍存在 |
| `condition_cleared` | 触发条件已恢复，但告警尚未由用户清除 |
| `cleared` | 用户或规则完成清除，当前不再阻止对应动作 |
| `archived` | 只作为历史保留 |

规则：

- 用户确认 `acknowledged` 不等于清除。
- 条件恢复 `condition_cleared` 不等于用户已经处理。
- 只有满足清除条件后，才能进入 `cleared`。
- 进入 `cleared` 后仍保留历史事件。

## 故障生命周期

故障状态：

```text
triggered -> protected -> waiting_clear -> cleared
```

含义：

| 状态 | 含义 |
|---|---|
| `triggered` | 故障条件成立 |
| `protected` | 已执行停机、锁定或禁止运行 |
| `waiting_clear` | 条件可能恢复，但需要用户或主控清除 |
| `cleared` | 已满足清除条件并清除 |

规则：

- 故障触发后必须先进入安全状态，再允许清除。
- 不可清除故障只能通过修复条件或重新同步配置解决。
- 清除故障后不能自动启动运行。

## 事件记录

事件是不可变历史记录。

事件类型：

| 类型 | 例子 |
|---|---|
| `run_started` | 手动或计划启动水路 |
| `run_stopped` | 正常停止或用户停止 |
| `run_faulted` | 本地保护停机 |
| `schedule_triggered` | 计划触发 |
| `schedule_skipped` | 计划项跳过 |
| `config_changed` | 用户保存配置 |
| `config_sync` | 配置同步成功或失败 |
| `fault_cleared` | 用户或维护动作清除故障 |
| `maintenance` | 扫描、识别、恢复配置 |
| `calibration` | 流量计校准和标准流速学习 |

事件必须包含：

```text
event_id
time
source
type
reason
scope
station_id
zone_index
run_id
schedule_run_id
message
```

规则：

- 事件不作为当前状态判断的唯一依据。
- 事件不能被清除故障删除。
- 事件用于解释为什么发生某个结果。

## 运行记录

运行记录在 `RunSession` 结束后形成。

运行记录必须包含：

```text
run_id
run_type
station_id
zone_index
started_at
ended_at
requested_duration_sec
actual_duration_sec
result
stop_reason
pulse_count
total_ml
flow_summary
config_version
had_lost_contact
disabled_protections
estimation_quality
created_by
```

`result`：

| 结果 | 含义 |
|---|---|
| `completed` | 按预期完成 |
| `stopped_by_user` | 用户停止 |
| `skipped` | 未启动即跳过 |
| `failed_to_start` | 启动失败 |
| `stopped_by_fault` | 故障或保护停机 |
| `lost_contact` | 最终状态无法确认或曾长期无法确认 |
| `cancelled` | 启动前取消 |

规则：

- 运行记录不能只写成功或失败。
- 如果运行中发生通信丢失，必须记录 `had_lost_contact`。
- 如果最终停止原因由从站上报，以从站停止原因为准。
- 校准和学习运行也要记录，但不计入普通灌溉完成率。
- 如果本次运行缺少增强保护，必须记录 `disabled_protections`，例如 `voltage_protection`、`low_high_flow_protection`。
- 如果流量计计量未校准，必须记录 `estimation_quality=unavailable` 或 `reference_only`，不能把估算水量显示为准确值。

## 计划记录

计划记录在一次 `ScheduleRun` 结束后形成。

计划记录必须包含：

```text
schedule_run_id
schedule_id
triggered_at
ended_at
result
station_results
skipped_items
failed_items
run_ids
```

`result`：

| 结果 | 含义 |
|---|---|
| `completed` | 全部应执行项完成 |
| `partial` | 部分完成，部分跳过或失败 |
| `skipped` | 全部应执行项未启动即跳过 |
| `failed` | 至少尝试启动或运行，但没有任何水路成功完成 |
| `cancelled` | 用户取消 |

规则：

- 计划部分完成必须明确显示。
- 计划全部跳过必须显示为 `skipped`，不能显示为 `failed`。
- 跳过项必须保留跳过原因。
- 删除计划不删除计划历史记录。

## 告警生成规则

以下情况生成当前告警：

| 条件 | 告警范围 | 是否禁止运行 |
|---|---|---|
| 低液位 | `station` | 是，禁止该从站 |
| 启动无水 | `station` 或 `zone` | 是，按锁定范围 |
| 运行中无水 | `station` 或 `zone` | 是，按锁定范围 |
| 高流量 | `zone` | 是 |
| 低流量 | `zone` | 按策略 |
| 待机漏水 | `station` | 默认禁止该从站新运行 |
| 通信丢失 | `station` | 是，直到确认状态 |
| 地址疑似冲突 | `system` 或 `station` | 是，禁止相关地址 |
| 配置未同步 | `config` | 是，禁止相关从站 |
| 配置无效 | `config` | 是 |
| 电压检测未校准 | `station` | 否，仅提示电压保护未启用 |
| 已启用电压保护并确认电压严重异常 | `station` | 是 |
| 主控时间无效 | `system` | 禁止自动计划 |

规则：

- 低/高流量告警只在流量计输入有效、目标水路标准流速已学习、对应保护已启用后生成；标准流速未学习时只记录“低/高流量保护未启用”，不得生成阻断性低/高流量告警。
- 电压检测未校准或电压保护未启用时，不生成阻断性电压异常告警，只生成电压保护未启用或未校准提示。

## 告警清除规则

| 告警 | 清除条件 |
|---|---|
| 低液位 | 液位恢复，或用户确认并允许清除 |
| 无水 | 用户确认检查水源、泵、阀、流量计后清除 |
| 高流量 | 用户确认检查管路、脱管、阀门和流量计 |
| 低流量 | 用户确认，或策略允许自动恢复 |
| 待机漏水 | 用户确认检查后，且当前无持续脉冲 |
| 通信丢失 | 从站重新上报安全状态后清除 |
| 地址疑似冲突 | 重新扫描后异常消失 |
| 配置未同步 | 同步成功 |
| 配置无效 | 重新同步有效配置 |
| 电压检测未校准 | 完成电压校准或用户确认暂不启用电压保护 |
| 已启用电压保护并确认电压严重异常 | 电压恢复并满足清除条件 |
| 主控时间无效 | 时间恢复有效 |

规则：

- 清除失败必须显示原因。
- 清除后不自动启动。
- 如果清除后条件仍然存在，应重新生成告警。

## 自动恢复和人工清除

默认分类：

| 类型 | 自动恢复 | 人工清除 |
|---|---|---|
| 配置未同步 | 同步成功后恢复 | 不适用 |
| 主控时间无效 | 时间恢复后恢复 | 不适用 |
| 低液位 | 条件恢复后可清 | 需要确认 |
| 低流量告警 | 可按策略自动恢复 | 可清 |
| 无水 | 否 | 需要确认 |
| 高流量 | 否 | 需要确认 |
| 待机漏水 | 否 | 需要确认 |
| 通信丢失 | 通信恢复后可清 | 需要确认 |
| 地址冲突 | 否 | 重新扫描确认 |

## 用户提示要求

每个当前告警必须显示：

- 名称。
- 影响范围。
- 当前是否阻止运行。
- 触发时间。
- 最近一次相关运行。
- 建议检查项。
- 是否可清除。
- 不能清除时的原因。

示例语义：

- “低液位触发，已停止该从站运行。请检查水源液位。”
- “配置已保存，但从站配置未同步，不能启动。”
- “主控已发送停止命令，正在等待从站确认停止。”
- “通信丢失，不能确认从站当前状态，等待从站恢复上报。”

## 记录保留边界

当前阶段只定义业务要求：

- 当前告警列表用于处理问题。
- 运行记录用于追溯灌溉结果。
- 事件记录用于解释操作和异常。
- 计划记录用于解释自动计划结果。

具体保存数量、保存天数、导出格式和存储介质属于实施阶段设计。

## 不支持的记录能力

当前产品不支持：

- 云端告警推送。
- 手机 App 消息。
- 复杂账号审计。
- 用户自定义脚本规则。
- 从站本地长期保存完整运行历史。
- 清除故障时删除历史证据。
