# 17 故障清除与本地维护策略

状态：成熟产品故障策略。本文定义故障触发、停机、恢复、清除、用户提示和记录方式。

关联文档：

## 总原则

- 故障清除不能绕过安全条件。
- 清除故障不删除历史记录。
- 从站本地按键只做识别和维护确认，不启动灌溉。
- 主控不能覆盖从站本地上报的真实停止原因。
- 某从站故障默认只影响该从站，不停止其他独立水源。

## CLEAR_FAULT 规则

从站执行 `CLEAR_FAULT` 前必须满足：

- 所有阀、主阀、自吸泵输出关闭。
- 配置有效。
- 命令地址、CRC、SEQ 正确。
- 低液位未触发，或清除的不是低液位故障。
- 电压恢复正常，或清除的不是电压故障。

不可通过普通 `CLEAR_FAULT` 清除：

- 配置无效。
- 存储错误。
- 疑似地址或总线冲突。
- 输出无法确认关闭。

这些问题只能通过重新同步配置、维护检修、重新扫描总线或断电检查处理。

## Web 提示和事件记录

Web 必须显示：

- 故障名称。
- 影响范围：从站级、水路级或系统级。
- 最近触发时间。
- 最近停止原因。
- 建议处理动作。
- 是否允许清除。

事件记录必须包含：

```text
source
type
reason
object
run_id
station_id
zone_index
fault_flags
voltage_mv
flow_rate_ml_min
text
```

低频业务事件进入业务事件记录；完整运行历史进入运行历史记录。具体记录介质和基础库 API 属于实施阶段设计。

## 本地按键

从站产品默认只使用 1 个本地维护/识别按键，另 1 个按键保留。

短按：

- 非运行中：进入识别显示，状态灯显示地址和通信状态。
- 运行中：只显示状态，不停止、不启动、不切换水路。

长按：

- 只允许清除可恢复故障或退出 `OFFLINE_LOCAL_SAFE`。
- 必须持续按住到故障灯快闪完成。
- 不允许清除配置无效、存储错误、地址冲突、输出不确认关闭等故障。

完整行为表：

| 场景 | 短按 | 长按 | 上电按住 |
|---|---|---|---|
| `BOOTING` | 忽略 | 忽略 | 进入识别显示，不改配置 |
| `UNCONFIGURED` | 显示地址和未配置状态 | 请求重新进入等待同步状态 | 不恢复出厂、不清配置 |
| `IDLE` | 识别从站，显示地址/通信状态 | 清除可恢复故障残留或退出维护提示 | 识别显示 |
| `STARTING` | 只显示正在启动 | 忽略 | 不适用 |
| `RUNNING` | 只显示正在运行 | 忽略 | 不适用 |
| `STOPPING` | 只显示正在停止 | 忽略 | 不适用 |
| `FAULT` | 显示故障状态 | 仅在清除条件满足时清除可恢复故障 | 识别显示 |
| `LOCKED` | 显示锁定状态 | 仅允许清除明确可恢复锁定 | 识别显示 |
| `OFFLINE_LOCAL_SAFE` | 显示通信丢失安全停机 | 通信恢复且输出关闭后可退出该状态 | 识别显示 |

本地按键禁止行为：

- 不启动灌溉。
- 不停止正在运行的灌溉。
- 不修改拨码地址。
- 不修改输入输出配置。
- 不恢复出厂配置。
- 不清除配置无效、存储错误、地址冲突或输出不确认关闭。

## 验收标准

- 每个故障都有可解释 Web 文案。
- 可清除故障在前置条件不满足时清除失败。
- 不可清除故障不能被 Web 或本地按键绕过。
- 清除后从站只能进入 `IDLE` 或保持安全状态，不能直接启动。
- 历史事件不因清除而删除。
