# 32 接口座边界复评结论

状态：当前阶段复评结论。本文在 `31-硬件IO命名细化复评结论.md` 之后，继续对主控板和从站板接口座做边界复评，明确当前产品哪些接口参与业务，哪些只作为电源、调试、烧录或预留接口。本文不写实施计划，不决定代码结构。

## 一、总体判断

当前 `01` 已覆盖主要芯片和 IO，但此前缺少接口座级边界说明。本轮已补充主控板 `CN1/H1/H2/H3/H4/U2/U3` 和从站板 `DC1/H1/H2/H3/H4/H5/H6/H7/H8/H9` 的用途边界。

补充后，当前边界为：

- 主控当前业务接口：12V 电源、H2 外接按键、U3 5V I2C、RS485。
- 主控当前预留/非业务接口：H1 3.3V I2C 引出、H3 模拟/NTC、H4 脉冲/传感、U2 串口调试。
- 从站当前业务接口：DC1 电源、H1 干接点输出、H4 输入、H5 RS485、8 路阀输出相关信号。
- 从站当前预留/非业务接口：H2 5V 引出、H3 GND 引出、H6 I2C 预留、H7 ISP/复位、H8/H9 预留 ADC。

当前仍不进入实施计划。

## 二、本轮修正的问题

### P1：接口座用途边界缺失

涉及文档：

- `01-新版硬件启用范围与系统边界.md`

问题描述：

- 文档已有芯片和 IO 表，但缺少连接器级说明。
- 主控 `H1/H3/H4/U2`、从站 `H6/H7/H8/H9` 这类接口如果只看网表，容易被误纳入当前产品软件范围。

风险：

- 把预留 I2C、ADC、传感输入、串口调试、ISP 口当成当前业务接口。
- 把外接端子板、阀输出接线形态误做成软件依赖。

修正方式：

- 增加“主控板接口座边界”表。
- 增加“从站板接口座边界”表。
- 明确阀输出外部端子板属于硬件连接边界，软件只控制网表中的阀驱动信号。

## 三、关键接口边界

### 主控板

| 接口 | 当前边界 |
|---|---|
| `CN1` | 12V 电源输入 |
| `H1` | 3.3V I2C/电源引出，不作为 LCD/SHT30 主接口 |
| `H2` | 外接 OK/ESC/UP/DOWN 按键 |
| `H3` | 预留模拟/NTC，不启用 |
| `H4` | 预留脉冲/传感，不启用 |
| `U2` | ESP32 串口/调试，不作为业务接口 |
| `U3` | 5V I2C，用于 LCD2004/SHT30 |

### 从站板

| 接口 | 当前边界 |
|---|---|
| `DC1` | 从站电源输入 |
| `H1` | 干接点输出，可选自吸泵/水泵继电器控制 |
| `H2` | 5V 引出/外部端子相关，不作为软件 IO |
| `H3` | GND 引出/外部端子相关，不作为软件 IO |
| `H4` | 4 路可配置数字输入 |
| `H5` | RS485 A/B |
| `H6` | 预留 I2C，不启用 |
| `H7` | STC ISP/复位，不作为业务接口 |
| `H8` | 预留 ADC1，不启用 |
| `H9` | 预留 ADC2，不启用 |

## 四、仍需实物确认的点

以下不是软件方案阻塞点，但不能在方案中写成已验证事实：

- 外部接口板如何把 8 路阀输出、公共电源、公共地和端子组合起来。
- `H2/H3` 在从站板上的实际外接端子用途和现场接线方式。
- 5V I2C 外接 LCD2004/SHT30 的线长、上拉、电平转换稳定性。
- `H4` 输入在干接点/有源脉冲、不同上拉焊接策略下的有效电平和抗干扰表现。

保守策略：

- 软件不依赖外部接口板的物理形态，只依赖从站安全配置中的输入、阀、主阀、自吸泵映射。
- 预留接口不进入 Web 配置的必需项。
- 实机测试前，输入有效电平、流量计边沿、低液位有效状态必须可配置。
