# 13 方案审查与明确结论

## 审查结论

当前方案的主线是成立的：主控只做 Web、调度、记录和 RS485 主站；从站对应真实水源；分支水路是计划、手动运行、记录和流量统计的最小对象。

需要收紧的是表达层，而不是重新推翻对象模型。当前文档中有几处命名不一致、入口不清和配置层级不清的问题；这些问题如果不先收掉，高保真原型会继续变复杂。

## 已修正的矛盾

### 文档入口

原问题：

- 当前文档放在 `docs/v5/`，但当前阶段不再需要用版本目录表达方案。

当前结论：

- 当前方案文档直接放在 `docs/` 下。
- `docs/README.md` 是文档入口。
- `docs/rough-prototype/index.html` 暂时保留为低保真结构参考。

### 设置和系统参数命名

原问题：

- 多数文档使用“系统参数”作为顶层页面。
- `10-水路校准与低保真原型准备.md` 仍写“设置”，容易让水源配置、水路配置和系统参数重新混到一个设置页里。

当前结论：

- 顶层页面名称统一为“系统参数”。
- 不再使用“设置”作为顶层页面名。
- 系统参数只放全局运行、通信和保护参数。

### 水源页是否允许配置

原问题：

- 文档一边说水源页不编辑硬件映射，一边又说水源级参数在水源配置页修改。
- 如果不说明“水源配置页”的层级，原型会变成隐藏的第八个顶层页面。

当前结论：

- 水源配置和水路配置不是顶层页面。
- 它们是水源详情和水路详情下的独立配置视图。
- 水源列表页只做查看和进入详情，不在列表下方直接铺配置表单。

### 无流量事件的阻断范围

原问题：

- “运行无流量”被列入阻断事件，但同时又写“不长期禁止该水路启动”。

当前结论：

- 无流量是本次任务异常停止事件。
- 它会解释为什么本次任务停止，但停止完成后不长期禁止再次启动。
- 长期影响启动的阻断只保留低液位、待机漏水、从站离线等当前仍有效的问题。

### 校准和浇水记录

原问题：

- 校准会实际打开水路运行，但文档需要明确它是否进入普通浇水记录。

当前结论：

- 校准运行不生成普通浇水记录。
- 校准成功、失败或取消只生成校准相关事件。
- 普通浇水记录只记录自动计划或手动浇水产生的真实水路任务。

## 复杂或过度设计风险

### RS485 通信参数展示层级

原问题：

- 之前把 RS485 通信速率、轮询周期、离线失败次数视为“工程调试项”，倾向于弱化或隐藏。
- 这个判断不适合当前项目。项目主要是个人使用，只要参数真实可配置，就应该按正确分组正常展示。

结论：

- RS485 通信速率、轮询周期、离线失败次数属于系统参数。
- 高保真原型应在系统参数页按“通信参数”分组正常展示和配置。
- 展示重点是默认值、单位、可选范围和修改后生效规则，不需要把它们藏起来。

### 阀 PWM 参数展示层级

原问题：

- 之前把 PWM 参数视为不应在原型中暴露的技术项。
- 实际上 PWM 是阀驱动的可配置安装参数，应该能配置，只是不应逐水路重复配置。

结论：

- PWM 参数按水源/从站统一配置，不逐水路配置。
- 高保真原型应在水源详情下的配置视图中正常展示和配置。
- 展示重点是吸合阶段、保持阶段、关闭阶段等参数的分组和默认值；不把 PWM 放到运行页或手动启动流程里。

### 统一周期模型需要用户友好表达

风险：

- “周期长度、周期起始日期、周期内执行日”作为底层模型没问题，但直接放给用户会显得抽象。

结论：

- 计划底层继续使用统一周期模型。
- 原型界面应优先提供“每天”“每周固定日”“浇几天停几天”等用户可理解入口，再映射到底层周期模型。

### 事件信息过多会淹没异常

风险：

- 如果配置变更、任务正常完成等信息事件和低液位、漏水、离线混在同一视觉层级，事件页会变成流水账。

结论：

- 事件页优先显示当前提示和阻断。
- 信息事件保留历史查询，但不进入当前异常数量，也不抢占首屏。

### SHT30、LCD2004 和本地按键不进入 Web 主流程

风险：

- SHT30 和 LCD2004 都是外接能力，按键本地操作也尚未抽取最小能力。
- 过早进入 Web 原型，会让当前主线从灌溉控制偏到外设展示。

结论：

- 当前 Web 原型不设计 LCD 菜单和本地按键流程。
- SHT30 最多作为后置的主控附近环境读数，不作为灌溉控制、计划或保护条件。

### 电压读数不要扩大成诊断系统

风险：

- 从站电压当前只确认“显示读数”，如果继续扩展告警、维护建议或停机，会超出已确认边界。

结论：

- 电压只显示，不告警、不保护、不停机。
- 高保真原型里只做低权重状态信息。

## 明确后的方案基准

### 顶层页面

高保真原型继续按 7 个顶层页面推进：

1. 总览
2. 运行
3. 计划
4. 水源
5. 记录
6. 事件
7. 系统参数

不新增诊断、维护、统计、天气、通知、OTA、LCD 顶层页面。

### 配置作用域

- 系统参数：全局运行、全局保护、RS485 通信和离线判定。
- 水源配置：从站/水源名称、主阀、水泵、流量计输入、低液位输入、阀 PWM 参数。
- 水路配置：水路名称、启用状态、参考流量和校准状态。

### 运行规则

- 同一水源同一时刻只运行一路。
- 不同水源可以并行运行。
- 手动运行不排队、不抢占。
- 自动计划触发时，水源不可用则跳过该水源本次步骤，生成事件，不补跑。

### 保护规则

- 低液位、待机漏水、从站离线会阻断新任务。
- 运行无流量导致本次任务异常停止，但不长期禁止再次启动。
- 低流量和过流量默认提示；如果配置为停机，只影响本次任务。

### 记录和事件

- 浇水记录只记录真实启动过的自动或手动水路任务。
- 跳过只生成事件，不生成浇水记录。
- 异常停止既补全浇水记录，也生成事件。
- 校准不生成普通浇水记录，只生成校准事件。

### 校准

- 校准从水路详情进入，不做顶层页面。
- 校准默认采集稳定流量 60 秒。
- 正常浇水记录中的稳定流速可以被采用为该水路参考流量。

## 高保真原型前需要确认

- 计划编辑页采用哪些用户友好的周期入口。
- 是否需要单独做移动端适配；如果不确认，先按桌面 Web 管理界面设计。
